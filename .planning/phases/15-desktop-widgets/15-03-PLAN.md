}

Component `WidgetContextMenu({widgetId, children}: {widgetId: string; children: ReactNode})`:
```typescript
export function WidgetContextMenu({widgetId, children}: {widgetId: string; children: ReactNode}) {
  const handleRemove = () => {
    removeDesktopWidget(widgetId)
  }

  return (
    <ContextMenu>
      <ContextMenuTrigger asChild>
        {children}
      </ContextMenuTrigger>
      <ContextMenuContent>
        <ContextMenuItem className="text-red-500 focus:text-red-500" onSelect={handleRemove}>
          <TbTrash className="mr-2 h-4 w-4" /> Widget'i Kaldir
        </ContextMenuItem>
      </ContextMenuContent>
    </ContextMenu>
  )
}
```

This follows the exact same pattern as the folder delete menu item in desktop-folder.tsx. Only one menu item: "Widget'i Kaldir" (Turkish, per CONTEXT.md). Red text + trash icon, same as folder delete pattern.

**IMPORTANT about ContextMenuTrigger**: Use `asChild` on the trigger so the context menu wraps around the widget div without adding an extra wrapper element that could break the grid layout. The children will be the motion.div that contains the WidgetRenderer.
