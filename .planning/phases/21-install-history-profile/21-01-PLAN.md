    3. Create a fire-and-forget `reportEvent` helper inside the hook (per CONTEXT.md decision -- fire-and-forget, don't block UI):
       ```typescript
       const reportEvent = useCallback((appId: string, action: 'install' | 'uninstall') => {
         const { apiKey, instanceName } = optionsRef.current
         if (!apiKey) return
         fetch('https://apps.livinity.io/api/install-event', {
           method: 'POST',
           headers: {
             'Content-Type': 'application/json',
             'X-Api-Key': apiKey,
           },
           body: JSON.stringify({
             app_id: appId,
             action,
             instance_name: instanceName,
           }),
         }).catch(() => {}) // Fire-and-forget: silently ignore errors
       }, [])
       ```

    4. In `handleInstall`, after the successful `trpcClient.apps.install.mutate({appId})` call (before `sendToIframe`), add:
       ```typescript

       ```

    5. In `handleUninstall`, after the successful `trpcClient.apps.uninstall.mutate({appId})` call (before `sendToIframe`), add:
       ```typescript
       reportEvent(appId, 'uninstall')
       ```

    Important: Place `reportEvent` calls inside the `try` blocks only (on success), NOT in catch blocks. Failed installs/uninstalls should not be reported. (Per HIST-01/HIST-04)

    **1. Create GET /api/user/history endpoint** at `platform/web/src/app/api/user/history/route.ts`:

    - Auth via `validateApiKey(req)` from `@/lib/api-auth` (same pattern as user/apps and user/profile)
    - Query install_history table joined with apps for the authenticated user, ordered by created_at DESC, limited to 50 rows:
      ```sql
      SELECT ih.id, ih.app_id, ih.action, ih.instance_name, ih.created_at,
             a.name as app_name, a.icon_url
      FROM install_history ih
      JOIN apps a ON a.id = ih.app_id
      WHERE ih.user_id = $1
      ORDER BY ih.created_at DESC
      LIMIT 50
      ```
    - Use raw SQL via `pool` (same pattern as user/apps and user/profile -- per Phase 17 decision about raw SQL for complex queries)
    - Return JSON: `{ events: Array<{ id, app_id, app_name, icon_url, action, instance_name, created_at }> }`
