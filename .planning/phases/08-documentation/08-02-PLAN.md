  <action>
Create CONTRIBUTING.md at project root with the following sections:

1. **Introduction**
   - Thank contributors
   - Explain this guide covers setup and contribution process

2. **Development Setup**
   - Prerequisites:
     - Node.js 22+
     - pnpm 9+ (for livos/)
     - npm 10+ (for nexus/)
     - Docker and Docker Compose
     - Redis (or Docker container)

   - Clone and setup:
     ```bash
     git clone https://github.com/utopusc/livinity-io.git
     cd livinity-io
     ```

   - LivOS setup (pnpm workspace):
     ```bash
     cd livos
     pnpm install
     cp .env.example .env
     # Edit .env with your values
     pnpm dev
     ```

   - Nexus setup (npm workspace):
     ```bash
     cd nexus
     npm install
     cp .env.example .env
     # Edit .env with your values
     npm run dev
     ```

   - Start dependencies:
     ```bash
     # Redis (required)
     docker run -d --name redis -p 6379:6379 redis:7

     # PostgreSQL (optional, for persistence)
     docker run -d --name postgres -p 5432:5432 \
       -e POSTGRES_PASSWORD=postgres postgres:15
     ```

3. **Project Structure**
   - Explain monorepo layout:
     ```
     livinity-io/
     ├── livos/                 # Main platform (pnpm)
     │   ├── packages/
     │   │   ├── livinityd/    # Backend daemon
     │   │   │   └── source/
     │   │   │       ├── modules/   # Feature modules
     │   │   │       └── utils/     # Shared utilities
     │   │   ├── ui/           # React frontend
     │   │   │   └── src/
     │   │   │       ├── routes/    # Page components
     │   │   │       └── modules/   # Feature modules
     │   │   └── config/       # @livos/config package
     │   └── skills/           # AI skill definitions
     │
     ├── nexus/                # AI Agent (npm)
     │   ├── packages/
     │   │   ├── core/         # Agent brain, daemon, tools
     │   │   ├── memory/       # Embedding service
     │   │   └── mcp-server/   # Claude/Cursor integration
     │   └── skills/           # Nexus skills
     │
     └── _archive/             # Deprecated code (do not modify)
     ```

4. **Code Style**
   - TypeScript strict mode
   - Use @livos/config for paths and domains (no hardcoding)
   - Error handling: Use formatErrorMessage() or getErrorMessage() helpers
   - No `any` types - use proper typing or `unknown`
   - Imports: Use node: prefix for Node.js built-ins (e.g., `import path from 'node:path'`)

5. **Testing**
   - Run tests:
     ```bash
     # livos
     cd livos && pnpm test

     # nexus
     cd nexus && npm test
     ```
   - Write tests for new features
   - Ensure existing tests pass before PR

6. **Pull Request Process**
   - Branch naming: `feature/description`, `fix/description`, `docs/description`
   - Commit messages: Use conventional commits
     - `feat:` new feature
     - `fix:` bug fix
     - `docs:` documentation
     - `refactor:` code refactoring
     - `test:` adding tests
   - PR checklist:
     - [ ] Tests pass
     - [ ] No TypeScript errors
     - [ ] Code follows style guidelines
     - [ ] Documentation updated if needed
   - PR will be reviewed within 48 hours

7. **Issue Reporting**
   - Use GitHub Issues
   - Include:
     - LivOS version
     - Node.js version
     - Steps to reproduce
     - Expected vs actual behavior
     - Logs if applicable

8. **Code of Conduct**
   - Reference CODE_OF_CONDUCT.md
   - "Be kind and respectful"

9. **Questions?**
   - Open a GitHub Discussion
   - Join Discord (placeholder link)
