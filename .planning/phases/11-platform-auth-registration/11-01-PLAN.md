**Update platform/relay/src/schema.sql** — Add auth columns to users table and sessions table:

After the existing `CREATE TABLE IF NOT EXISTS users` block, add:
```sql
-- Auth columns (added in Phase 11)
ALTER TABLE users ADD COLUMN IF NOT EXISTS email VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS password_hash VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS email_verified BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN IF NOT EXISTS email_verification_token VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS email_verification_expires TIMESTAMPTZ;
ALTER TABLE users ADD COLUMN IF NOT EXISTS password_reset_token VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS password_reset_expires TIMESTAMPTZ;

-- Make email unique (idempotent: check if constraint exists)
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_constraint WHERE conname = 'users_email_key') THEN
    ALTER TABLE users ADD CONSTRAINT users_email_key UNIQUE (email);
  END IF;
END $$;

-- Sessions table
CREATE TABLE IF NOT EXISTS sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  token VARCHAR(255) UNIQUE NOT NULL,
  expires_at TIMESTAMPTZ NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  ip_address VARCHAR(45),
  user_agent TEXT
);
CREATE INDEX IF NOT EXISTS idx_sessions_token ON sessions(token);
CREATE INDEX IF NOT EXISTS idx_sessions_user_id ON sessions(user_id);
```

**Create platform/web/src/lib/auth.ts:**
Core auth utilities:
- `hashPassword(password: string): Promise<string>` — bcrypt hash with 10 rounds
- `verifyPassword(password: string, hash: string): Promise<boolean>` — bcrypt compare
- `createSession(userId: string, req?: { ip?: string; headers?: { get(name: string): string | null } }): Promise<string>` — generate nanoid token, insert into sessions with 30d expiry, return token
- `getSession(token: string): Promise<{ userId: string; username: string; email: string; emailVerified: boolean } | null>` — query sessions JOIN users, check expiry, return user info or null
- `deleteSession(token: string): Promise<void>` — delete from sessions
- `validateUsername(username: string): { valid: boolean; error?: string }` — 3-30 chars, ^[a-z0-9][a-z0-9-]*[a-z0-9]$ (or single char), no reserved words
- `RESERVED_USERNAMES: Set<string>` — admin, www, api, app, relay, status, help, support, billing, dashboard, login, register, signup, signin, auth, account, settings, profile, mail, ftp, ssh, root, test, demo
- `SESSION_COOKIE_NAME = 'liv_session'`
- `SESSION_MAX_AGE = 30 * 24 * 60 * 60` (30 days in seconds)

**Create platform/web/src/lib/email.ts:**
- `sendVerificationEmail(to: string, token: string): Promise<void>` — uses Resend SDK
- `sendPasswordResetEmail(to: string, token: string): Promise<void>` — uses Resend SDK
- Base URL from `process.env.NEXT_PUBLIC_BASE_URL || 'https://livinity.io'`
- Resend API key from `process.env.RESEND_API_KEY`
- From address: `noreply@livinity.io`
- If no RESEND_API_KEY, log the email to console instead of sending (dev mode)
