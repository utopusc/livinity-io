**API key retrieval (getClient method):**
- Read from Redis key `nexus:config:anthropic_api_key` first
- Fallback to `process.env.ANTHROPIC_API_KEY`
- Cache the client instance, re-create only if key changes
- If no key available, throw Error('No Anthropic API key configured')

**chat() method:**
- Get tier model name from CLAUDE_MODELS (default to 'sonnet')
- Use `prepareForProvider(messages, 'claude')` to convert messages to Claude format
- Call `client.messages.create({ model, max_tokens, system: systemPrompt, messages: claudeMessages })`
- Extract text from response content blocks (concatenate all text blocks)
- Return `{ text, inputTokens: response.usage.input_tokens, outputTokens: response.usage.output_tokens, provider: 'claude', model }`
- Default max_tokens: 4096 (NOT 2048 -- Claude tool responses need more space, as noted in Pitfall #11)

**chatStream() method:**
- This is the critical method for the streaming experience
- Use `client.messages.stream({ model, max_tokens, system: systemPrompt, messages: claudeMessages })`
- The SDK's `.stream()` helper returns a `MessageStream` with event emitters
- Create an async generator that:
  1. Listens to `.on('text', (text) => ...)` events and yields `{ text, done: false }`
  2. After the stream completes (`.finalMessage()`), yields `{ text: '', done: true }`
  3. Captures usage from `stream.finalMessage()` for the `getUsage()` closure
- Return `{ stream: generator, getUsage, provider: 'claude', model }`

IMPORTANT for the async generator pattern with the SDK's event-based stream:
- Use the `stream[Symbol.asyncIterator]()` or iterate with `for await (const event of stream)` and check event types
- Alternatively, use `stream.on('text', callback)` with a queue pattern
- The simplest approach: use the stream's async iterator which yields `MessageStreamEvent` objects, check for `event.type === 'content_block_delta'` with `event.delta.type === 'text_delta'`, extract `event.delta.text`
- After iteration, call `const finalMessage = await stream.finalMessage()` to get usage

**think() method:**
- Simple single-turn completion
- Create a single user message with the prompt
- Call `chat()` with the prompt wrapped as a single-message conversation
- Return just the text string

**isAvailable() method:**
- Try to read API key from Redis then env
- Return true if a key exists (don't make an API call, just check key presence)
- Wrap in try/catch, return false on any error

**getModels() method:**
- Return the CLAUDE_MODELS record

**Error handling in chatStream:**
- Wrap the stream iteration in try/catch
- On error, yield a final `{ text: '', done: true }` chunk so the consumer doesn't hang
- Log the error with `logger.error('ClaudeProvider.chatStream error', { error: err.message })`
- Re-throw after yielding the done signal so retry logic in the caller can catch it

**Retry handling:**
- Do NOT implement retry inside ClaudeProvider -- the Brain wrapper and AgentLoop already have retry logic
- ClaudeProvider should throw on errors and let callers handle retries
- Use the existing `retryAsync` and `isRetryableError` infrastructure from `../infra/retry.js` and `../infra/errors.js`
