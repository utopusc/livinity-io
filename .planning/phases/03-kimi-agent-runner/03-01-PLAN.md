**Step 1: Check current Python version**
Run `python3 --version` to see what's installed. If already 3.12+, skip Python installation.

**Step 2: Install Python 3.12 (if needed)**
On Ubuntu/Debian (which the server runs):
```bash
apt update && apt install -y python3.12 python3.12-venv python3.12-dev
# If python3 still points to old version:
update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.12 1
```
If the system repo doesn't have 3.12, add the deadsnakes PPA:
```bash
apt install -y software-properties-common
add-apt-repository -y ppa:deadsnakes/ppa
apt update && apt install -y python3.12 python3.12-venv python3.12-dev
```

**Step 3: Install uv**
```bash
curl -LsSf https://astral.sh/uv/install.sh | bash
# Source profile to make uv available
source /root/.bashrc
# Or if using .cargo/bin:
export PATH="/root/.local/bin:$PATH"
uv --version
```

**Step 4: Install Kimi CLI**
The recommended installation method from research:
```bash
# Option A: via uv
uv tool install kimi-code

# Option B: via official installer
curl -LsSf https://code.kimi.com/install.sh | bash
```
After installation, verify:
```bash
kimi --version
```

**Step 5: Ensure PATH is available to PM2 processes**
PM2 processes need `kimi` on PATH. The nexus-core process sources `/root/.profile` on start.
Check where kimi was installed (`which kimi`) and ensure that directory is in the PATH that PM2 sees.
If kimi is in `/root/.local/bin/`, ensure that's in `/root/.profile` or `/root/.bashrc`:
```bash
echo 'export PATH="/root/.local/bin:$PATH"' >> /root/.profile
# Also ensure uv tools path is included
echo 'export PATH="/root/.local/share/uv/tools/bin:$PATH"' >> /root/.profile
```
Then restart PM2 processes so they pick up the new PATH:
```bash
source /root/.profile
pm2 restart nexus-core
```

**Step 6: Verify kimi is accessible from PM2 context**
```bash
# Run kimi from the PM2 environment
source /root/.profile && kimi --version
```

IMPORTANT: Remember SSH stdout appears in stderr on Windows. Use the pattern:
```bash
ssh ... 'command > /tmp/out.txt 2>&1' && scp ... root@server:/tmp/out.txt local.txt
```
to capture output if needed.

AVOID: Do not attempt to configure kimi credentials or login here -- that is Phase 2's concern. This task is purely about installation.
