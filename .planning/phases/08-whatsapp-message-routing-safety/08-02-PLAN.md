<objective>
Consolidate all WhatsApp message routing through ChannelManager and remove legacy ad-hoc code from daemon.ts.

Purpose: Currently daemon.ts has two parallel WhatsApp delivery paths: the new ChannelManager/Baileys path (inbound) and the legacy wa_outbox Redis polling path (outbound). This creates confusion and bypasses the rate limiter added in Plan 01. Unifying everything through ChannelManager means outbound WhatsApp messages flow through WhatsAppProvider.sendMessage() which is rate-limited.

Output: daemon.ts with all WhatsApp routing through ChannelManager, zero references to wa_outbox/wa_pending, and three legacy methods deleted.
</objective>
