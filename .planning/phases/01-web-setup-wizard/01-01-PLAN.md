  <action>
    **agent/package.json**: Add dependencies: `express` and `open` (for auto-opening browser). Add devDependency: `@types/express`.

    **agent/src/setup-server.ts**: Create the local HTTP server module. Exports a single function:

    ```typescript
    export async function startSetupServer(): Promise<{ port: number; close: () => void }>
    ```

    Implementation:
    1. Import express, path, open, and os.hostname
    2. Create Express app
    3. Determine the setup-ui dist path. Use `path.join(path.dirname(new URL(import.meta.url).pathname), '..', 'setup-ui', 'dist')` for dev mode. For SEA/bundled mode, use `path.join(process.cwd(), 'setup-ui')` as fallback. Check both paths and use the first that exists. On Windows, strip leading slash from URL pathname (e.g., `/C:/...` becomes `C:/...`).
    4. Serve static files: `app.use(express.static(distPath))`
    5. Implement 3 API endpoints (JSON responses):

    **GET /api/status** — Returns current setup state:
    ```json
    { "deviceName": "<os.hostname()>", "status": "awaiting_setup" | "connecting" | "polling" | "success" | "error", "agentVersion": "0.1.0" }
    ```
    The status starts as "awaiting_setup". This is a module-level variable that other endpoints mutate.

    **POST /api/start-setup** — Stub that returns:
    ```json
    { "started": true }
    ```
    (Will be wired to deviceFlowSetup in Plan 02. For now, just set status to "connecting" and return success.)

    **GET /api/poll-status** — Returns current status + optional fields:
    ```json
    { "status": "awaiting_setup" | "connecting" | "polling" | "success" | "error", "userCode": null, "verificationUri": null, "deviceName": null, "errorMessage": null }
    ```
    (Will be populated by the OAuth flow in Plan 02.)

    6. Fallback route: serve index.html for SPA routing (`app.get('*', (req, res) => res.sendFile(path.join(distPath, 'index.html')))`)

    7. Start server on port 19191. If EADDRINUSE, try 19192, 19193, up to 19199. Store the actual port.

    8. After server starts, call `open(`http://localhost:${port}`)` to auto-open browser.

    9. Return `{ port, close: () => server.close() }` so the caller can shut down the server later.

    Add `express.json()` middleware for POST body parsing.

    Run `cd agent && npm install && npx tsc --noEmit` to verify types.
