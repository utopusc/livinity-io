    **agent/src/setup-server.ts** -- Replace the POST /api/start-setup and GET /api/poll-status stubs with real OAuth flow integration.

    The challenge: `deviceFlowSetup()` in auth.ts is a blocking function (loops polling until success/error). We need the server to run it asynchronously while the SPA polls for status. Do NOT modify auth.ts. Instead, implement the OAuth device flow steps directly in the server endpoints using fetch against PLATFORM_URL (same pattern as auth.ts but non-blocking):

    Add module-level mutable state:
    ```typescript
    let setupStatus: 'awaiting_setup' | 'connecting' | 'polling' | 'success' | 'error' = 'awaiting_setup';
    let userCode: string | null = null;
    let verificationUri: string | null = null;
    let connectedDeviceName: string | null = null;
    let errorMessage: string | null = null;
    let setupPromiseResolve: ((creds: CredentialsData) => void) | null = null;
    ```

    **POST /api/start-setup** implementation:
    1. Accept optional `{ deviceName }` from body (default to os.hostname())
    2. Set status = 'connecting'
    3. Kick off an async function (do NOT await it) that:
       a. POSTs to `${PLATFORM_URL}/api/device/register` with `{ deviceName, platform: process.platform, agentVersion: AGENT_VERSION }`
       b. On success: set userCode, verificationUri from response, set status = 'polling'
       c. Start a polling loop: POST `${PLATFORM_URL}/api/device/token` with `{ device_code }` every `interval` seconds
       d. On authorization_pending: continue polling
       e. On success (200): decode JWT payload (reuse the base64url decode pattern from auth.ts), create CredentialsData, call writeCredentials(), set connectedDeviceName, set status = 'success', resolve setupPromiseResolve
       f. On expired_token or timeout: set errorMessage, set status = 'error'
       g. On any error: set errorMessage, set status = 'error'
    4. Return `{ started: true }`

    **GET /api/poll-status** implementation:
    Return the current state:
    ```json
    {
      "status": "<setupStatus>",
      "userCode": "<userCode or null>",
      "verificationUri": "<verificationUri or null>",
      "deviceName": "<connectedDeviceName or null>",
      "errorMessage": "<errorMessage or null>"
    }
    ```

    **POST /api/retry** -- New endpoint:
    1. Reset all state to initial (awaiting_setup)
    2. Return `{ reset: true }`

    Modify `startSetupServer` to return a Promise that resolves with credentials when setup completes:
    ```typescript
    export async function startSetupServer(): Promise<{ port: number; close: () => void; waitForSetup: () => Promise<CredentialsData> }>
    ```
    The `waitForSetup()` returns a promise that resolves when setupPromiseResolve is called (i.e., OAuth flow succeeds).

    **agent/setup-ui/src/App.tsx** -- Wire the SPA to real API endpoints:

    1. On mount: `GET /api/status` to get deviceName, set screen based on returned status
    2. When user clicks "Connect Your Account" (onConnect callback):
       - `POST /api/start-setup` with `{ deviceName }`
       - Switch to 'connecting' screen
       - Start polling `GET /api/poll-status` every 2 seconds using setInterval
    3. Poll handler logic:
       - status === 'polling' AND userCode present: update userCode + verificationUri in state (ConnectingScreen shows the code)
       - status === 'success': switch to 'success' screen, stop polling, set 5-second timeout to call `window.close()` (per CONTEXT.md: auto-close after success). If window.close() is blocked by browser, show "You can close this tab" text.
       - status === 'error': switch to 'error' screen with errorMessage, stop polling
    4. When user clicks "Try Again" (onRetry callback):
       - `POST /api/retry`
       - Reset to 'welcome' screen

    Clean up setInterval on unmount via useEffect cleanup.
