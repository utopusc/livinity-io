   ```

   ```

2. In docker.ts, update manageContainer switch to handle new operations:
   - 'kill': call container.kill() -- SIGKILL. Protected container check: allow kill (same as restart -- it's an emergency stop, not removal).
   - 'pause': call container.pause(). Protected container check: block pause on protected containers (they must stay running).
   - 'unpause': call container.unpause(). No protection needed (resuming is always safe).
   - Update the success message for these new operations: kill -> "killed", pause -> "paused", unpause -> "resumed"
   - Update the protection check: add 'pause' to the list alongside 'stop' and 'remove' (if isProtectedContainer and operation is 'stop', 'remove', or 'pause', throw error).

3. In docker.ts, add a new exported function `bulkManageContainers`:
   ```typescript
   export async function bulkManageContainers(
     names: string[],
     operation: ContainerOperation,
     force?: boolean,
   ): Promise<Array<{name: string; success: boolean; message: string}>>
   ```
   - Run all operations in parallel using Promise.allSettled
   - For each name, call manageContainer(name, operation, force)
   - Map settled results to {name, success, message} -- on rejection use error.message
   - Do NOT skip protected containers here -- let manageContainer's own protection throw (caller sees per-container errors)

4. In routes.ts, update the manageContainer route's operation z.enum to include 'kill', 'pause', 'unpause':
   ```
   operation: z.enum(['start', 'stop', 'restart', 'remove', 'kill', 'pause', 'unpause']),
   ```
   - Keep existing confirmName requirement only for 'remove' operation (unchanged)

5. In routes.ts, add a new `bulkManageContainers` mutation after the manageContainer route:
   ```typescript
   bulkManageContainers: adminProcedure
     .input(z.object({
       names: z.array(z.string().min(1).max(255)).min(1).max(50),
       operation: z.enum(['start', 'stop', 'restart', 'remove', 'kill', 'pause', 'unpause']),
       force: z.boolean().optional().default(false),
     }))
     .mutation(async ({input}) => {
       return await bulkManageContainers(input.names, input.operation, input.force)
     })
   ```
   - Import bulkManageContainers from docker.ts
   - Wrap in try/catch with TRPCError like other routes

6. In common.ts, add 'docker.bulkManageContainers' to httpOnlyPaths array (after 'docker.manageContainer').
