**Tool format translation (CORE-02) -- two helper functions inside the file:**
1. `translateToolDefinition(tool: ClaudeToolDefinition): OpenAIFunctionTool` -- Converts Anthropic format to OpenAI format:
   - Input: `{ name, description, input_schema: { type: 'object', properties, required } }`
   - Output: `{ type: 'function', function: { name, description, parameters: { type: 'object', properties, required } } }`
   - This is a straightforward field rename: `input_schema` -> `parameters`

2. `parseToolArguments(args: string | Record<string, unknown>): Record<string, unknown>` -- Handles CORE-03:
   - If `args` is already an object, return it directly
   - If `args` is a string, `JSON.parse()` it
   - On parse failure, log warning via logger and return `{}` (don't crash the agent loop)

**OpenAI-compatible types (define at top of file, not exported):**
```typescript
interface OpenAIFunctionTool {
  type: 'function';
  function: {
    name: string;
    description: string;
    parameters: {
      type: 'object';
      properties: Record<string, unknown>;
      required: string[];
    };
  };
}

interface OpenAIChatMessage {
  role: 'system' | 'user' | 'assistant' | 'tool';
  content?: string | null;
  tool_calls?: Array<{
    id: string;
    type: 'function';
    function: { name: string; arguments: string };
  }>;
  tool_call_id?: string;
}

interface OpenAIStreamChunk {
  id: string;
  choices: Array<{
    delta: {
      content?: string | null;
      tool_calls?: Array<{
        index: number;
        id?: string;
        type?: 'function';
        function?: { name?: string; arguments?: string };
      }>;
    };
    finish_reason: string | null;
  }>;
  usage?: { prompt_tokens: number; completion_tokens: number; total_tokens: number };
}

interface OpenAIChatResponse {
  id: string;
  choices: Array<{
    message: {
      content: string | null;
      tool_calls?: Array<{
        id: string;
        type: 'function';
        function: { name: string; arguments: string };
      }>;
    };
    finish_reason: string;
  }>;
  usage: { prompt_tokens: number; completion_tokens: number; total_tokens: number };
}
```

**`chat()` method:**
- Resolve model from tier using `getModelForTier(tier)` (reads Redis override or falls back to KIMI_MODELS constant)
- Build messages array: system message + user/assistant messages via `prepareForProvider(messages, 'kimi')`
- If `options.rawMessages` provided, use those directly (same pattern as ClaudeProvider)
- If `options.tools` provided, translate each via `translateToolDefinition()`
- Make `fetch()` POST to `${BASE_URL}/chat/completions` with:
  - Headers: `Authorization: Bearer ${apiKey}`, `Content-Type: application/json`
  - Body: `{ model, messages, tools?, stream: false }`
- Parse response as `OpenAIChatResponse`
- Extract text from `choices[0].message.content`
- Extract tool calls: map each `tool_calls[]` item to `ToolUseBlock` format:
  - `type: 'tool_use'`
  - `id: tc.id`
  - `name: tc.function.name`
  - `input: parseToolArguments(tc.function.arguments)`
- Map `finish_reason` to stopReason: `'stop'` -> `'end_turn'`, `'tool_calls'` -> `'tool_use'`, `'length'` -> `'max_tokens'`
- Return `ProviderChatResult` with usage from `response.usage`

**`chatStream()` method (CORE-05):**
- Same setup as `chat()` but with `stream: true` in the body
- Make `fetch()` POST, get `response.body` as ReadableStream
- Create async generator that:
  1. Reads the SSE stream line by line (split on `\n`)
  2. For lines starting with `data: `, strip prefix
  3. If line is `[DONE]`, yield final chunk `{ text: '', done: true, stopReason }` and return
  4. Parse JSON as `OpenAIStreamChunk`
  5. For text deltas (`choices[0].delta.content`), yield `{ text: delta, done: false }`
  6. For tool call deltas (`choices[0].delta.tool_calls`):
     - Accumulate tool call chunks by index (id comes in first chunk, name in first chunk, arguments streamed across chunks)
     - When `finish_reason` becomes `'tool_calls'` or `'stop'`, emit completed tool calls as `ToolUseBlock` via the `toolUse` field
  7. Track usage: if `chunk.usage` present (some providers send usage in last chunk), capture it
- For usage tracking: also try parsing the final `[DONE]` preceding chunk for usage, or fall back to 0 if not available
- Return `ProviderStreamResult` with the generator, `getUsage` closure, provider `'kimi'`, and resolved model string

**Tool call accumulation in streaming (important detail):**
Track an array of in-progress tool calls during streaming:
```typescript
const pendingToolCalls: Map<number, { id: string; name: string; args: string }> = new Map();
```
- On delta with `tool_calls[i].id`, create new entry at index `i`
- On delta with `tool_calls[i].function.name`, set name on entry `i`
- On delta with `tool_calls[i].function.arguments`, append to args on entry `i`
- When `finish_reason === 'tool_calls'`, emit each pending tool call as a `ToolUseBlock` chunk, then clear
