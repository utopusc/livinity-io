1. **Update `server.on('upgrade')` in index.ts:**

The current WebSocket upgrade handler (starting around line 499) checks for /tunnel/connect, /device/connect, then falls through to subdomain-based user WebSocket proxying. Add custom domain WS routing after the subdomain check fails.

Import `lookupCustomDomain` from `./custom-domains.js` at the top of index.ts.

Modify the upgrade handler to add custom domain lookup:

```typescript
server.on('upgrade', async (req, socket, head) => {
  const url = req.url ?? '';

  if (url.startsWith('/tunnel/connect')) {
    tunnelWss.handleUpgrade(req, socket, head, (ws) => {
      onTunnelConnect(ws);
    });
    return;
  }

  if (url.startsWith('/device/connect')) {
    deviceWss.handleUpgrade(req, socket, head, (ws) => {
      onDeviceConnect(ws);
    });
    return;
  }

  // User subdomain WebSocket proxying
  const { username, appName } = parseSubdomain(req.headers.host);
  if (username) {
    const tunnel = registry.get(username);
    if (tunnel && tunnel.ws.readyState === 1) {
      handleWsUpgrade(req, socket, head, tunnel);
      return;
    }
  }

  // --- NEW: Custom domain WebSocket proxying (DOM-04) ---
  if (!username) {
    const hostname = req.headers.host?.split(':')[0]?.toLowerCase();
    if (hostname) {
      const customDomain = await lookupCustomDomain(pool, redis, hostname);
      if (customDomain) {
        const tunnel = registry.get(customDomain.username);
        if (tunnel && tunnel.ws.readyState === 1) {
          handleWsUpgrade(req, socket, head, tunnel, null);
          return;
        }
      }
    }
  }

  // No tunnel or not connected -- reject upgrade
  socket.destroy();
});
```

Note: The upgrade callback needs to become async. This is safe because Node.js `server.on('upgrade')` does not expect a return value.
