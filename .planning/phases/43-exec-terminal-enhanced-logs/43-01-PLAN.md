  <action>
    Create `docker-exec-socket.ts` as a new WebSocket handler for container exec. Follow the exact pattern from `terminal-socket.ts` but use dockerode instead of node-pty:

    1. Export a function `createDockerExecHandler` with signature:
       ```
       ({logger}: {logger: ReturnType<typeof createLogger>}) =>
         async (ws: WebSocket, request: http.IncomingMessage) => { ... }
       ```

    2. Parse query params from request.url:
       - `container` (required) -- container name
       - `shell` (optional, default 'bash') -- one of 'bash', 'sh', 'ash'
       - `user` (optional, default '') -- container user to exec as

    3. Validate container param exists, reject with ws.close(1008, 'Missing container') if not.

    4. Use the dockerode singleton (import from `./docker.ts` or create local one):
       ```typescript
       import Dockerode from 'dockerode'
       const docker = new Dockerode()
       ```

    5. Create exec instance:
       ```typescript
       const container = docker.getContainer(containerName)
       const exec = await container.exec({
           Cmd: [shell],
           AttachStdin: true,
           AttachStdout: true,
           AttachStderr: true,
           Tty: true,
           User: user || undefined,
       })
       ```

    6. Start exec with hijack mode to get raw duplex stream:
       ```typescript
       const stream = await exec.start({hijack: true, stdin: true, Tty: true})
       ```

    7. Bidirectional piping:
       - stream.on('data', (chunk) => { if ws is open, ws.send(chunk) })
       - ws.on('message', (data) => { stream.write(data) })

    8. Handle resize messages from client. Client sends JSON: `{"type":"resize","cols":N,"rows":N}`.
       In ws.on('message'), try JSON.parse first. If it has type==='resize', call:
       ```typescript
       exec.resize({h: msg.rows, w: msg.cols})
       ```
       Otherwise treat as terminal input and write to stream.

    9. Cleanup: ws.on('close', () => stream.destroy()). Also stream.on('end', () => ws.close()).

    10. Wrap entire handler in try/catch. On error: logger.error, ws.close(1011, error.message).

    Then in `server/index.ts`:
    - Import createDockerExecHandler from '../docker/docker-exec-socket.js'
    - After the existing `/terminal` mountWebSocketServer call (around line 637), add:
      ```typescript
      this.mountWebSocketServer('/ws/docker-exec', (wss) => {
          const logger = this.logger.createChildLogger('docker-exec')
          wss.on('connection', createDockerExecHandler({logger}))
      })
      ```
    - Note: Auth is already handled by the upgrade handler (lines 559-573) which validates the token from query params before allowing the connection. No additional auth needed in the handler itself.
  </action>
