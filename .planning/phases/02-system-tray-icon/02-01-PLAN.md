2. Create `agent/src/tray.ts` with these exports:

   **`startTray(callbacks): Promise<void>`** — Initializes the system tray icon.
   - `callbacks` parameter: `{ onDisconnect: () => void, onQuit: () => void, onOpenSetup: () => void }`
   - Creates a SysTray instance with:
     - Icon: base64-encoded 16x16 PNG (green dot by default). Use programmatically-generated PNG buffers for the 3 states (connected=green, connecting=yellow, disconnected=red/gray). Generate minimal valid PNG buffers in code — a 16x16 solid colored circle on transparent background. Use Node.js Buffer to construct raw PNG bytes (IHDR + IDAT with zlib deflate + IEND). This avoids needing external image files that break in SEA mode.
     - Alternatively (simpler and more robust): Create 3 tiny 16x16 PNGs as base64 string constants hardcoded in the file. The simplest approach: use a single-pixel colored PNG scaled up, or a minimal hand-crafted PNG. The key requirement is that icons are EMBEDDED as base64 constants — not loaded from disk.
     - Title: empty string (icon only in tray, no text label)
     - Tooltip: `"Livinity Agent — Connected"` / `"Connecting..."` / `"Disconnected"`
     - Menu items array:
       ```
       statusItem:   { title: "Status: Connected", tooltip: "Connection status", enabled: false, checked: false }
       separator:    SysTray.separator
       openSetup:    { title: "Open Setup", tooltip: "Open setup wizard in browser", enabled: true, checked: false }
       disconnect:   { title: "Disconnect", tooltip: "Disconnect from relay", enabled: true, checked: false }
       separator:    SysTray.separator
       quit:         { title: "Quit", tooltip: "Stop the agent", enabled: true, checked: false }
       ```
   - Attach onClick handler that checks `action.item` identity and calls the appropriate callback
   - Await `systray.ready()` before returning
   - Store the SysTray instance and menu item references in module-level variables for `updateTrayStatus` to use

   **`updateTrayStatus(status: 'connected' | 'connecting' | 'disconnected' | 'error'): void`** — Updates the tray icon and status text.
   - Maps status to icon (green/yellow/red base64 PNG) and label text
   - Calls `systray.sendAction({ type: 'update-item', item: statusItem })` to update the status line text
   - Calls `systray.sendAction({ type: 'update-menu', item: { ...menu, icon: newIcon, tooltip: newTooltip } })` to change the icon
   - For 'error' status, use the red/disconnected icon with text "Status: Error"
   - If systray instance is null (not yet started), silently return

   **`killTray(): void`** — Calls `systray.kill(false)` for clean shutdown.

   **Icon generation approach:** Create a helper function `generateColoredIcon(r, g, b): string` that returns a base64-encoded 16x16 PNG with a colored circle. Use the simplest valid PNG structure:
   - PNG signature (8 bytes)
   - IHDR chunk (13 bytes data: width=16, height=16, bit_depth=8, color_type=6 RGBA)
   - IDAT chunk (zlib-compressed raw RGBA pixel data — 16 rows of 16 RGBA pixels, draw a circle)
   - IEND chunk
   - Use `import { deflateSync } from 'node:zlib'` for compression
   - Pre-generate the 3 icons at module load time as constants: `ICON_CONNECTED` (green #22c55e), `ICON_CONNECTING` (yellow #eab308), `ICON_DISCONNECTED` (red #ef4444)
   - For Windows: systray2 accepts PNG on all platforms (it converts internally)

2. **Modify `agent/src/cli.ts`** startCommand() to initialize the tray:

   Add import at top:
   ```typescript
   import { startTray, updateTrayStatus, killTray } from './tray.js';
   ```

   After the ConnectionManager is created (`const manager = new ConnectionManager(...)`) but before `manager.connect()`:

   a. Pass `onStatusChange: updateTrayStatus` in the ConnectionManager options:
   ```typescript
   const manager = new ConnectionManager({
     credentials,
     onStatusChange: (status) => updateTrayStatus(status),
   });
   ```

   b. Start the tray BEFORE connecting, with callbacks:
   ```typescript
   // Initialize system tray (non-blocking -- continues even if tray fails)
   try {
     await startTray({
       onDisconnect: () => {
         console.log('[agent] Disconnect requested from tray');
         manager.disconnect();
       },
       onQuit: () => {
         console.log('[agent] Quit requested from tray');
         manager.disconnect();
         killTray();
         process.exit(0);
       },
       onOpenSetup: async () => {
         console.log('[agent] Opening setup wizard from tray');
         try {
           const openModule = await import('open');
           const openFn = openModule.default;
           // Start setup server and open browser
           const { startSetupServer } = await import('./setup-server.js');
           const server = await startSetupServer();
           console.log(`Setup wizard opened at http://localhost:${server.port}`);
           // Don't await waitForSetup -- let it run in background
         } catch (err) {
           console.error('[agent] Failed to open setup:', err);
         }
       },
     });
   } catch (err) {
     // Tray failure is non-fatal -- agent works without tray (headless servers, SSH)
     console.warn('[agent] System tray not available:', (err as Error).message);
   }
   ```

   c. Update the shutdown handler to also kill the tray:
   ```typescript
   const shutdown = () => {
     console.log('\n[agent] Shutting down...');
     manager.disconnect();
     killTray();
     process.exit(0);
   };
   ```

   Important: The tray initialization must be wrapped in try/catch because headless Linux servers, SSH sessions, and CI environments have no display — systray2 will fail, and that is OK. The agent must continue running without a tray in those cases.
