## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `RunStore` ← caller (LivAgentRunner / SSE handler) | runId strings flow in from authenticated callers; treated as opaque keys, but composed into Redis key paths. Risk: key injection via crafted runId. |
| `RunStore` ← Redis | Untrusted JSON in stored values would only matter if a different process wrote to `liv:agent_run:*` keys. In v31 this is single-writer (livinityd in-process). |
| Pub/Sub channel | Channel name composed from runId. Same key-injection risk class as keys. |
