    2. Create `agent/src/blocklist.ts`:
       - Import `fs`, `path`, `getCredentialsDir` from `./state.js`, `CONFIG_FILE` from `./config.js`
       - Export `DEFAULT_BLOCKLIST` as an array of objects `{ pattern: string, description: string }`:
         ```typescript
         // Unix dangerous commands
         { pattern: 'rm\\s+-[^\\s]*r[^\\s]*f[^\\s]*\\s+/', description: 'Recursive force delete from root' },
         { pattern: 'rm\\s+-[^\\s]*f[^\\s]*r[^\\s]*\\s+/', description: 'Recursive force delete from root' },
         { pattern: 'mkfs\\.', description: 'Format filesystem' },
         { pattern: 'dd\\s+if=', description: 'Raw disk write' },
         { pattern: ':\\(\\)\\{\\s*:\\|:\\&\\s*\\};:', description: 'Fork bomb' },
         { pattern: '>(\\s*)/dev/sda', description: 'Write to raw disk device' },
         { pattern: 'chmod\\s+-R\\s+777\\s+/', description: 'Recursive chmod 777 on root' },
         { pattern: 'chown\\s+-R\\s+.*\\s+/', description: 'Recursive chown on root' },
         // Shutdown/reboot (standalone commands, not substrings)
         { pattern: '(?:^|[;&|]\\s*)shutdown(?:\\s|$)', description: 'System shutdown' },
         { pattern: '(?:^|[;&|]\\s*)reboot(?:\\s|$)', description: 'System reboot' },
         { pattern: '(?:^|[;&|]\\s*)halt(?:\\s|$)', description: 'System halt' },
         { pattern: '(?:^|[;&|]\\s*)poweroff(?:\\s|$)', description: 'System poweroff' },
         { pattern: '(?:^|[;&|]\\s*)init\\s+0', description: 'Init runlevel 0 (shutdown)' },
         // Windows dangerous commands
         { pattern: 'format\\s+[a-zA-Z]:', description: 'Format drive' },
         { pattern: 'del\\s+/[sS]\\s+/[qQ]\\s+[cC]:\\\\', description: 'Recursive silent delete C drive' },
         { pattern: 'reg\\s+delete\\s+.*HKLM', description: 'Registry delete HKLM' },
         { pattern: 'reg\\s+delete\\s+.*HKEY_LOCAL_MACHINE', description: 'Registry delete HKEY_LOCAL_MACHINE' },
         { pattern: 'Remove-Item\\s+-Recurse\\s+-Force\\s+[Cc]:\\\\', description: 'PowerShell recursive force delete C drive' },
         { pattern: 'Stop-Computer', description: 'PowerShell shutdown' },
         { pattern: 'Restart-Computer', description: 'PowerShell restart' },
         ```
       - Export interface `BlocklistConfig { blocklist: Array<{ pattern: string; description: string }> }`
       - Export function `loadBlocklist(): Array<{ pattern: string; description: string; regex: RegExp }>`:
         - Try to read `config.json` from `getCredentialsDir()`
         - If file exists and has a `blocklist` array, use that. Otherwise use DEFAULT_BLOCKLIST.
         - Compile each pattern to a RegExp with 'i' flag (case insensitive)
         - Wrap in try/catch: if config file is malformed, log warning and fall back to DEFAULT_BLOCKLIST
         - Cache the compiled result in a module-level variable. Reload only if config file mtime changed (track mtime).
       - Export function `isCommandBlocked(command: string): { blocked: boolean; pattern?: string; description?: string }`:
         - Call `loadBlocklist()` to get current rules
         - Test command against each regex
         - Return first match: `{ blocked: true, pattern: rule.pattern, description: rule.description }`
         - No match: `{ blocked: false }`

    3. Update `agent/src/tools/shell.ts`:

       - After the command validation check (line ~51 `if (!command ...)`), add blocklist check:

         ```typescript
         const blockCheck = isCommandBlocked(command);
         if (blockCheck.blocked) {
           return {
             success: false,
             output: '',
             error: `Command blocked by security policy: ${blockCheck.description} (pattern: ${blockCheck.pattern})`,
           };
         }
         ```
       - This MUST be before the `detectShell()` and `spawn()` calls so blocked commands never execute
