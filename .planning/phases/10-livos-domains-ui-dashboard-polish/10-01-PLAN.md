Add three new tRPC routes to the existing `platform` router in `routes.ts`:

1. **listCustomDomains** (query): Read domains from PostgreSQL `custom_domains` table via `getPool()`. Return array of `{id, domain, appMapping: Record<string, string>, status, syncedAt, createdAt}`. Fallback to Redis `livos:custom_domains` key if PG pool is null. Import `getPool` from `../database/index.js`.

2. **updateAppMapping** (mutation): Input `{domain: string, prefix: string, appSlug: string | null}`. If appSlug is null, remove the prefix from the mapping; otherwise set it. Update both:
   - PostgreSQL: `UPDATE custom_domains SET app_mapping = $2 WHERE domain = $1` (read current, modify, write back)
   - Redis: Read `livos:custom_domain:{domain}`, parse JSON, update appMapping field, write back. Then call `rebuildDomainCache()` equivalent -- read all per-domain keys, rebuild `livos:custom_domains` list.
   - Also notify the platform via tunnel message so relay picks up the mapping change: send a `domain_sync` message with action `update` through the tunnel WebSocket. Access the tunnel client via `ctx.livinityd.tunnelClient` and call its `send()` method if connected.

3. **removeCustomDomain** (mutation): Input `{domain: string}`. Delete from PostgreSQL, delete Redis key `livos:custom_domain:{domain}`, rebuild domain list cache, and send a tunnel message to notify the platform of removal. Access tunnel client via `ctx.livinityd.tunnelClient`.

For the Redis operations, access Redis via `ctx.livinityd.ai.redis` (same pattern as existing routes in this file).

In `common.ts`, add these to httpOnlyPaths array at the end of the domain platform section:
```

'domain.platform.updateAppMapping',
'domain.platform.removeCustomDomain',
```
