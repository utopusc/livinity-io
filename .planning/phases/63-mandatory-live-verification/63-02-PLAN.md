ONE single SSH invocation to Mini PC, batching 5 verifications:

```bash
ssh -i C:/Users/hello/Desktop/Projects/contabo/pem/minipc \
    -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null \
    bruce@10.69.31.68 \
    "echo '=== A: 4 services ==='; \
     systemctl is-active livos liv-core liv-worker liv-memory; \
     echo '=== B: update-history latest ==='; \
     LATEST=\$(ls -t /opt/livos/data/update-history/*.json 2>/dev/null | head -1); \
     echo \"latest_file=\$LATEST\"; \
     if [ -n \"\$LATEST\" ]; then cat \"\$LATEST\"; fi; \
     echo '=== C: pnpm-store @nexus+core dirs ==='; \
     ls -d /opt/livos/node_modules/.pnpm/@nexus+core* 2>/dev/null; \
     echo '=== D: livinityd dist resolution ==='; \
     readlink -f /opt/livos/node_modules/@nexus/core 2>/dev/null; \
     ls -la /opt/livos/node_modules/@nexus/core/dist/index.js 2>/dev/null; \
     echo '=== E: memory dist ==='; \
     ls -la /opt/nexus/packages/memory/dist/index.js 2>/dev/null || echo 'NO_MEMORY_DIST'; \
     echo '=== F: sacred file SHA on server ==='; \
     git hash-object /opt/livos/nexus/packages/core/src/sdk-agent-runner.ts 2>/dev/null || echo 'NO_GIT_ON_SERVER'; \
     # Fallback if no git: run sha1 manually \
     if ! command -v git >/dev/null 2>&1; then \
       openssl dgst -sha1 /opt/livos/nexus/packages/core/src/sdk-agent-runner.ts; \
     fi; \
     echo '=== G: timestamp ==='; \
     date -u +%Y-%m-%dT%H:%M:%SZ; \
     echo '=== END ==='" \
    | tee .planning/phases/63-mandatory-live-verification/63-postdeploy.log
```

Note: `git hash-object` is a SHA-1 hash of the blob (with `blob N\0` prefix), so it differs from a raw `sha1sum`. To make a comparable check on a non-git host, use:
```
printf 'blob %d\0' $(stat -c%s /opt/livos/nexus/packages/core/src/sdk-agent-runner.ts) | cat - /opt/livos/nexus/packages/core/src/sdk-agent-runner.ts | sha1sum | awk '{print $1}'
```
If the orchestrator-side check (Task 3 in Wave 0) already passed, the on-server file came from the same git source via `update.sh` rsync — so a content-equality check (file size + first/last 40 chars) is sufficient on-server.
