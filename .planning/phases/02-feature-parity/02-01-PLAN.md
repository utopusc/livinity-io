Enable full feature parity between Claude and Kimi in the agent loop: native tool calling, streaming, vision/multimodal image handling, and model tier routing.

Purpose: The agent loop currently hardcodes native tool calling to Kimi only (`useNativeTools = activeProvider === 'kimi'`). Output: Agent loop correctly uses native tool calling for both providers, with provider-appropriate image format conversion.

Claude requires Anthropic format:

Replace the image mapping block (lines ~672-682) with provider-aware conversion:

```typescript
// Build content: if tool returned images, create multimodal content array
let resultContent: string | Array<{ type: string; [k: string]: unknown }> = resultText;
if (toolResult.images && toolResult.images.length > 0) {
  resultContent = [
    { type: 'text', text: resultText },
    ...toolResult.images.map(img => {
      if (activeProvider === 'claude') {
        return {
          type: 'image' as const,
          source: { type: 'base64' as const, media_type: img.mimeType, data: img.base64 },
        };
      }
      return {
        type: 'image_url' as const,
        image_url: { url: `data:${img.mimeType};base64,${img.base64}` },
      };
    }),
  ];
}
```

This ensures FEAT-03 (vision/multimodal) works correctly -- when Claude is the active provider, images in tool_result blocks use Anthropic's native image format. Kimi still gets OpenAI `image_url` format.
