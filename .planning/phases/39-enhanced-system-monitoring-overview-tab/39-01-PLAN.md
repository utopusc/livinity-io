**types.ts** -- Define these interfaces:
```typescript
export interface NetworkStat {
  iface: string       // interface name e.g. "eth0"
  rxBytes: number     // total bytes received
  txBytes: number     // total bytes transmitted
  rxSec: number | null // bytes/sec received (null on first call)
  txSec: number | null // bytes/sec transmitted (null on first call)
}

export interface DiskIO {
  rIO: number         // total read operations
  wIO: number         // total write operations
  rIOSec: number | null // read bytes/sec (null on first call)
  wIOSec: number | null // write bytes/sec (null on first call)
}

export interface ProcessInfo {
  pid: number
  name: string
  cpu: number         // percentage 0-100
  memory: number      // percentage 0-100
  state: string       // e.g. "running", "sleeping"
}
```

**monitoring.ts** -- Three domain functions using systeminformation (import as `import systemInformation from 'systeminformation'`, same as system.ts):

1. `getNetworkStats()`: Call `systemInformation.networkStats()`. Map result to `NetworkStat[]`. Filter out `lo` (loopback) interface. For each interface return `{iface, rxBytes: stat.rx_bytes, txBytes: stat.tx_bytes, rxSec: stat.rx_sec, txSec: stat.tx_sec}`. The `rx_sec`/`tx_sec` values are null on first call per systeminformation docs -- pass through as-is.

2. `getDiskIO()`: Call `systemInformation.disksIO()`. Return `{rIO: result.rIO, wIO: result.wIO, rIOSec: result.rIO_sec, wIOSec: result.wIO_sec}`. The `rIO_sec`/`wIO_sec` values are null on first call -- pass through as-is. NOTE: `disksIO()` returns a single aggregate object, not an array.

3. `getProcesses(sortBy: 'cpu' | 'memory' = 'cpu')`: Call `systemInformation.processes()`. From `result.list`, sort by the chosen field descending, take top 20. Map to `ProcessInfo` with `{pid: p.pid, name: p.name, cpu: p.cpu, memory: p.mem, state: p.state}`. Note: systeminformation uses `mem` not `memory` for the field name.

**routes.ts** -- tRPC router with `privateProcedure` (read-only monitoring, not admin-only per CONTEXT.md decision):
- `networkStats`: query, no input, calls `getNetworkStats()`
- `diskIO`: query, no input, calls `getDiskIO()`
- `processes`: query, input `z.object({ sortBy: z.enum(['cpu', 'memory']).optional().default('cpu') })`, calls `getProcesses(input.sortBy)`

Import `privateProcedure` and `router` from `../server/trpc/trpc.js`. Follow the same error handling pattern as docker/routes.ts (try/catch with TRPCError INTERNAL_SERVER_ERROR).

**index.ts** -- Add monitoring import and registration:
- Add `import monitoring from '../../monitoring/routes.js'`
- Add `monitoring` to the appRouter object

No httpOnlyPaths needed -- these are read-only queries, not mutations.
