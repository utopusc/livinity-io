Fix horizontal overflow issues in message rendering. CRITICAL: Do NOT change desktop behavior or visual appearance -- only add width constraints that prevent overflow.

**A. streaming-message.tsx -- constrain code blocks (CHAT-02):**

1. The `<pre>` element for code blocks (line ~107) uses inline style `overflow: 'auto'` but has NO max-width. On mobile, a wide code block expands its parent, pushing the chat area wider than the viewport.

Add `maxWidth: '100%'` to the code block `<pre>` style object:
```jsx
<pre style={{margin: '8px 0', overflow: 'auto', borderRadius: '8px', border: '1px solid #e2e8f0', background: '#f8fafc', padding: '12px', maxWidth: '100%'}}>
```

2. The streaming (typewriter) render div (line ~71) has `whiteSpace: 'pre-wrap', wordBreak: 'break-word'` which is correct for text wrapping. But add `overflow: 'hidden'` to prevent any chance of horizontal expansion:
```jsx
<div style={{...textStyle, whiteSpace: 'pre-wrap', wordBreak: 'break-word', overflow: 'hidden'}}>
```

3. The markdown render wrapper div (line ~86) has just `style={textStyle}`. Add `overflow: 'hidden'` to prevent code blocks from pushing it wider:
```jsx
<div style={{...textStyle, overflow: 'hidden'}}>
```

**B. chat-messages.tsx -- constrain message containers (CHAT-02):**

1. `AssistantMessage` (line ~435-468): The outer div has `max-w-[90%]` but no overflow constraint. Add `overflow-hidden` to prevent code blocks inside from pushing it wider:
```jsx
<div className='max-w-[90%] overflow-hidden border-l-2 border-violet-500/30 pl-4'>
```

Wait -- `overflow-hidden` would clip the tool call expand animations. Instead use `min-w-0` which is the flexbox way to allow shrinking below content size:
```jsx
<div className='min-w-0 max-w-[90%] border-l-2 border-violet-500/30 pl-4'>
```

Actually, the issue is that `max-w-[90%]` alone doesn't prevent children from pushing wider. The solution is to add `overflow-x-auto` to the inner content but `overflow-hidden` on the message wrapper would break animations. 

The correct approach: Add `w-0 min-w-0 max-w-[90%] flex-1` -- but this only works inside a flex container. The parent `.space-y-4` is not flex.

Simplest correct fix: Add `overflow-hidden` ONLY on the content area, not the animation wrapper. Restructure AssistantMessage slightly:

Change AssistantMessage's inner div from:
```jsx
<div className='max-w-[90%] border-l-2 border-violet-500/30 pl-4'>
```
to:
```jsx
<div className='max-w-[90%] border-l-2 border-violet-500/30 pl-4' style={{overflowWrap: 'break-word', wordBreak: 'break-word'}}>
```

This ensures long unbroken strings wrap. The code block overflow is handled by the `<pre>` having `overflow: auto` + `maxWidth: 100%` from streaming-message.tsx fixes above.

2. `UserMessage` (line ~417-424): The blue bubble has `max-w-[85%]` and the text has `whitespace-pre-wrap` but no word-break. Add `break-words` to prevent long URLs/strings from overflowing:
```jsx
<p className='whitespace-pre-wrap break-words text-sm'>{message.content}</p>
```
Note: Tailwind's `break-words` maps to `overflow-wrap: break-word`.

3. Tool output `<pre>` elements in `ToolOutput` component (lines 121, 147, 168): These already have `overflow-auto` and `whitespace-pre-wrap` which is good. But add `max-w-full` via a style to ensure they respect parent width:
Add to ALL three `<pre>` elements in ToolOutput the style: add className `max-w-full` alongside existing classes.

For the shell output pre (line ~121):
```jsx
<pre className='max-h-60 max-w-full overflow-auto whitespace-pre-wrap rounded bg-surface-2 p-2 font-mono text-xs text-text-primary'>
```

For the file read pre (line ~147):
```jsx
<pre className='max-h-80 max-w-full overflow-auto whitespace-pre-wrap rounded bg-surface-2 p-2 font-mono text-xs text-text-primary'>
```

For the default pre (line ~168):
```jsx
<pre className='max-h-60 max-w-full overflow-auto whitespace-pre-wrap text-xs text-text-secondary'>
```

4. The `renderToolInput` default JSON pre (line ~102) also needs `max-w-full`:
```jsx
<pre className='max-w-full overflow-x-auto whitespace-pre-wrap text-xs text-text-secondary'>
```

5. In the parent scroll container in index.tsx (already handled by plan 01's scope -- but for reference), the `max-w-3xl` constrains the message list width, which is the outermost width boundary.

Make tool call cards more compact on mobile with clear tap targets (CHAT-03). CRITICAL: Do NOT change desktop behavior.

The current `AgentToolCallDisplay` (line ~328-411) already has a good compact design: one-line header with icon + tool name + summary + chevron, and expandable output on click. This is ALREADY a compact-summary + tap-to-expand pattern, which satisfies CHAT-03's core requirement.

However, the touch target for expanding is the entire header row which is only ~24px tall (text-xs + py-0.5). On mobile, this is too small to tap reliably.

**1. Import useIsMobile:**
Add to the imports at the top of chat-messages.tsx:
```typescript
import {useIsMobile} from '@/hooks/use-is-mobile'
```

**2. Make tool call header a bigger touch target on mobile:**
In `AgentToolCallDisplay`, add `const isMobile = useIsMobile()` at the start of the function.

Change the header button's padding to be larger on mobile. Current classes (line ~357):
```
className='group flex w-full items-center gap-1.5 text-left text-xs hover:bg-surface-1/50 rounded px-1 py-0.5 -mx-1'
```

Change to use responsive padding:
```
className={cn(
  'group flex w-full items-center gap-1.5 text-left text-xs hover:bg-surface-1/50 rounded px-1 -mx-1',
  isMobile ? 'py-2' : 'py-0.5'
)}
```

The `py-2` on mobile gives 8px + 8px = 16px vertical padding plus the text height (~16px) = ~32px minimum, closer to a tappable target. Combined with the `my-1` on the outer div, the effective touch area is ~40px.

**3. Ensure expanded tool output is constrained on mobile:**
The expanded output section (line ~383-409) has `ml-5 pl-3` which indents content. On very narrow screens, this could leave very little room for output. Add a mobile-specific reduction:

Change:
```jsx
<div className='ml-5 mt-0.5 border-l-2 border-surface-2 pl-3 pb-1'>
```
to:
```jsx
<div className={cn('mt-0.5 border-l-2 border-surface-2 pb-1', isMobile ? 'ml-2 pl-2' : 'ml-5 pl-3')}>
```

This reduces left margin from 20px to 8px and padding from 12px to 8px on mobile, giving ~24px more horizontal space for tool output content.

**4. Truncate tool summary more aggressively on mobile:**
The `toolInputSummary` function (line ~307) truncates commands at 80 chars. On mobile, this is too long. Add an optional parameter:

Change `toolInputSummary` signature to accept maxLen:
```typescript
function toolInputSummary(toolCall: ChatToolCall, maxLen = 80): string {
```

And use it in the truncation:
```typescript
return cmd.length > maxLen ? cmd.slice(0, maxLen - 3) + '...' : cmd
```

Then in AgentToolCallDisplay, call it with a shorter limit on mobile:
```typescript
const summary = toolInputSummary(toolCall, isMobile ? 40 : 80)
```

This keeps the one-line summary truly one-line on narrow screens.
