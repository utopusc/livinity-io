<objective>
Fix the DPI-induced coordinate mismatch in the screenshot-to-click pipeline. Install sharp for actual image resize, rewrite toolScreenshot() to resize from physical pixels to a target resolution, fix toScreenX/toScreenY to map AI coordinates to logical screen space (what robotjs expects), and update the AI system prompt to explicitly document the coordinate space.

Purpose: On HiDPI displays (125%, 150%, 200%), the AI's click coordinates currently miss their targets because screenshots are sent at full physical resolution while claiming to be at a smaller target resolution. This makes computer use unusable on any non-100% DPI display.

Output: Working screenshot pipeline where the image is actually resized, coordinates map correctly through the full chain (AI -> resized image space -> logical screen space -> robotjs), and the AI prompt is accurate.
</objective>

**Step 3: Rewrite toolScreenshot() method**

Replace the entire `toolScreenshot()` method (lines 469-525) with a new implementation that:

1. Captures screenshot via `node-screenshots` (same as before)
2. Gets physical dimensions from `primary.width()` and `primary.height()` -- these are PHYSICAL pixels
3. Gets `scaleFactor` from `primary.scaleFactor()` (e.g., 1.5 for 150% DPI)
4. Computes logical dimensions: `logicalW = Math.round(physicalW / scaleFactor)`, `logicalH = Math.round(physicalH / scaleFactor)`
5. Finds best Anthropic target resolution by aspect ratio (keep existing SCALE_TARGETS logic, but compare against LOGICAL dimensions not physical)
6. Uses sharp to ACTUALLY resize the captured image from physical to target dimensions
7. Stores `this.logicalScreenW`, `this.logicalScreenH`, `this.aiTargetW`, `this.aiTargetH` for coordinate mapping
8. Returns metadata with `displayWidth: targetW` and `displayHeight: targetH` matching the ACTUAL resized image dimensions

The method must be async (sharp resize is async). Update the method signature to `private async toolScreenshot(): Promise<any>`.

Key implementation:
```typescript
private async toolScreenshot(): Promise<any> {
  try {
    const ns = require('node-screenshots');
    const sharp = require('sharp');
    const monitors = ns.Monitor.all();
    const primary = monitors.find((m: any) => m.isPrimary()) || monitors[0];
    if (!primary) return { error: 'No display found' };

    const physicalW = primary.width();   // Physical pixels (e.g., 2560)
    const physicalH = primary.height();  // Physical pixels (e.g., 1440)
    const scaleFactor = primary.scaleFactor(); // e.g., 1.5

    // Logical dimensions = what robotjs uses for mouse coordinates
    const logicalW = Math.round(physicalW / scaleFactor);
    const logicalH = Math.round(physicalH / scaleFactor);

    // Find best Anthropic target by aspect ratio of logical dimensions
    const ratio = logicalW / logicalH;
    let targetW = logicalW;
    let targetH = logicalH;
    for (const t of AgentCore.SCALE_TARGETS) {
      if (Math.abs(t.ratio - ratio) < 0.02 && t.w <= logicalW) {
        targetW = t.w;
        targetH = t.h;
        break;
      }
    }

    // Store for coordinate mapping
    this.logicalScreenW = logicalW;
    this.logicalScreenH = logicalH;
    this.aiTargetW = targetW;
    this.aiTargetH = targetH;
    // Keep legacy fields for backward compat
    this.screenWidth = logicalW;
    this.screenHeight = logicalH;

    const image = primary.captureImageSync();
    const rawBuffer = Buffer.from(image.toRawSync());

    // Use sharp to ACTUALLY resize the image from physical to target resolution
    const resizedJpeg = await sharp(rawBuffer, {
      raw: { width: physicalW, height: physicalH, channels: 4 }
    })
      .resize(targetW, targetH, { fit: 'fill' })
      .jpeg({ quality: 80 })
      .toBuffer();

    const base64 = resizedJpeg.toString('base64');

    return {
      success: true,
      output: `Screenshot captured: ${physicalW}x${physicalH} physical, ${logicalW}x${logicalH} logical, resized to ${targetW}x${targetH} for AI. Coordinate space: 0,0 to ${targetW},${targetH}. Scale factor: ${scaleFactor}.`,
      data: {
        width: logicalW, height: logicalH,
        displayWidth: targetW, displayHeight: targetH,
        physicalWidth: physicalW, physicalHeight: physicalH,
        scaleFactor: scaleFactor,
        monitorX: primary.x(), monitorY: primary.y(),
        size: resizedJpeg.length,
      },
      images: [{ base64, mimeType: 'image/jpeg' }],
    };
  } catch (e: any) { return { error: `Screenshot failed: ${e.message}` }; }
}
```

NOTE: The `data.width` and `data.height` now report LOGICAL dimensions (not physical). `data.displayWidth` and `data.displayHeight` report the actual resized image dimensions. `data.physicalWidth` and `data.physicalHeight` are new fields for debugging.

IMPORTANT: `image.toRawSync()` returns BGRA raw pixel data from node-screenshots. Sharp's `raw` input expects `channels: 4` for RGBA. node-screenshots uses BGRA byte order. If colors appear wrong in the JPEG, the executor should check whether the channels need swapping. However, for coordinate accuracy (the goal of this phase), color correctness is secondary -- if BGRA/RGBA causes a blue tint, that is cosmetic and does not affect coordinate mapping. If `toRawSync()` is not available, use `image.toJpegSync()` and pass the JPEG buffer to sharp directly (sharp can read JPEG input without raw options):

```typescript
// Alternative if toRawSync() is not available:
const jpegInput = Buffer.from(image.toJpegSync());
const resizedJpeg = await sharp(jpegInput)
  .resize(targetW, targetH, { fit: 'fill' })
  .jpeg({ quality: 80 })
  .toBuffer();
```

**Step 4: Rewrite toScreenX/toScreenY (DPI-03)**

Replace the `toScreenX` and `toScreenY` methods (lines 557-569) with correct coordinate mapping.

The mapping chain is:
- AI returns coordinates in resized-image space (targetW x targetH)
- We need logical screen coordinates for robotjs (logicalW x logicalH)
- Formula: `screenX = aiX * (logicalW / targetW)`

```typescript
// Map AI coordinates (in resized screenshot space) to logical screen coordinates (for robotjs)
private toScreenX(x: number): number {
  if (this.aiTargetW > 0 && this.logicalScreenW > 0) {
    return Math.round(x * (this.logicalScreenW / this.aiTargetW));
  }
  return Math.round(x);
}
private toScreenY(y: number): number {
  if (this.aiTargetH > 0 && this.logicalScreenH > 0) {
    return Math.round(y * (this.logicalScreenH / this.aiTargetH));
  }
  return Math.round(y);
}
```

This is the critical fix: instead of dividing by `screenScaleX` (which mapped to PHYSICAL space), we now multiply by the ratio of logical-to-AI dimensions, which maps to LOGICAL space -- exactly what robotjs expects.

Remove the `screenScaleX` and `screenScaleY` fields since they are replaced by the new `aiTargetW`/`aiTargetH` and `logicalScreenW`/`logicalScreenH` fields. Also remove `this.screenScaleX` and `this.screenScaleY` assignments from toolScreenshot. The `data.scaleX` and `data.scaleY` fields in the return value should be removed since they exposed the old broken scale factors.
