  <action>
1. Run `cd agent && npm install systeminformation` to add the dependency.

2. Create `agent/src/tools/processes.ts`:
   - Export async function `executeProcesses(params: Record<string, unknown>)` returning `Promise<ProcessResult>`.
   - ProcessResult follows the established shape: `{ success: boolean; output: string; error?: string; data?: unknown }`.
   - Parameters: `sortBy` (optional, 'cpu' | 'memory', default 'cpu'), `limit` (optional number, default 20).
   - Use `import si from 'systeminformation'` then call `si.processes()`.
   - From the result `data.list`, map each process to `{ pid: p.pid, name: p.name, cpu: p.cpu, memory: p.mem, user: p.user }`.
   - Sort by `sortBy` field descending, then slice to `limit`.
   - Set `output` to a formatted text table: `PID  NAME  CPU%  MEM%  USER\n{rows}`.
   - Set `data` to the array of process objects.
   - Wrap in try/catch; on error return `{ success: false, output: '', error: err.message }`.

3. Create `agent/src/tools/system-info.ts`:
   - Export async function `executeSystemInfo(params: Record<string, unknown>)` returning `Promise<SystemInfoResult>`.
   - SystemInfoResult follows the same shape as ProcessResult.
   - No parameters needed.
   - Call in parallel via Promise.all: `si.osInfo()`, `si.cpu()`, `si.mem()`, `si.fsSize()`, `si.networkInterfaces()`, `si.time()`.
   - Build structured result object:
     ```
     {
       os: { platform: osInfo.platform, distro: osInfo.distro, release: osInfo.release, arch: osInfo.arch, hostname: osInfo.hostname },
       cpu: { manufacturer: cpu.manufacturer, brand: cpu.brand, cores: cpu.cores, speed: cpu.speed },
       memory: { total: mem.total, used: mem.used, free: mem.free, percent: +(mem.used / mem.total * 100).toFixed(1) },
       disks: fsSize.map(d => ({ fs: d.fs, size: d.size, used: d.used, available: d.available, mount: d.mount })),
       network: networkInterfaces.filter(n => !n.internal).map(n => ({ iface: n.iface, ip4: n.ip4, mac: n.mac })),
       uptime: time.uptime
     }
     ```
   - Set `output` to a human-readable multiline summary (OS line, CPU line, Memory line, Disk lines, Network lines, Uptime).
   - Set `data` to the full structured object.
   - Wrap in try/catch; on error return `{ success: false, output: '', error: err.message }`.
  </action>
