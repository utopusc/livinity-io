   - Add import at the top with the other plugin imports:

   - Add `tailwindSafeArea` to the plugins array as the LAST entry (after tailwindRadix):

This enables the following Tailwind utilities for use in future mobile components:
- `pt-safe` -- padding-top: env(safe-area-inset-top)
- `pb-safe` -- padding-bottom: env(safe-area-inset-bottom)
- `pl-safe` / `pr-safe` -- left/right safe area padding
- `h-screen-safe` -- 100vh minus top and bottom safe areas
- `min-h-screen-safe` -- min-height version
- `pb-safe-offset-2` -- safe-area-inset-bottom + 0.5rem (composable offsets)

These utilities resolve to 0px on non-notched devices and desktop browsers, so they are safe to add without affecting existing UI.

  <action>
Update `livos/packages/ui/src/index.css` with these additions:

1. **Add safe area CSS custom properties** inside the existing `html { }` block (after `--wallpaper-blur: 12px;`):
   ```css
   html {
     --font-inter: 'Inter';
     --font-jakarta: 'Space Grotesk';
     --sheet-top: 2vh;
     --wallpaper-blur: 12px;
     /* Safe area insets for notched devices (viewport-fit=cover required in index.html) */
     --safe-area-top: env(safe-area-inset-top, 0px);
     --safe-area-right: env(safe-area-inset-right, 0px);
     --safe-area-bottom: env(safe-area-inset-bottom, 0px);
     --safe-area-left: env(safe-area-inset-left, 0px);
   }
   ```

2. **Add overscroll-behavior and touch-action** to the existing `html, body { }` block:
   ```css
   html,
   body {
     text-rendering: geometricPrecision;
     @apply -tracking-3 text-text-primary;
     overscroll-behavior: none;
     -webkit-touch-callout: none;
   }
   ```
   - `overscroll-behavior: none` prevents iOS rubber-band bounce on the outer shell (Pitfall 7 mitigation)
   - `-webkit-touch-callout: none` prevents long-press callout menu on touch targets

3. **Add standalone mode CSS detection** after the existing `html { }` block:
   ```css
   /* PWA standalone mode adjustments */
   @media (display-mode: standalone) {
     html {
       --sheet-top: 0vh;
     }
   }
   ```
   This sets `--sheet-top` to 0 in standalone mode because the Sheet layout's 2vh gap (used to peek the wallpaper behind the sheet on desktop) is not needed in PWA mode.

4. **Add touch-action: manipulation** to the existing `input, button { }` block:
   ```css
   input,
   button {
     font-size: inherit;
     text-rendering: inherit;
     touch-action: manipulation;
   }
   ```
   This eliminates the 300ms tap delay on mobile without disabling pinch-to-zoom (Pitfall 14 partial mitigation).

Do NOT modify any other existing CSS rules. The additions are purely additive and have no effect on desktop browsers (safe area vars resolve to 0px, overscroll-behavior: none has no visible effect, display-mode: standalone media query never matches on desktop).
