## Objective

Update all CSS in `index.css` that uses dark-theme-specific colors (white-based scrollbars, dividers, glows, gradients) to light-theme-appropriate values.

## Tasks

<task id="1">
**Update scrollbar colors for light theme**

Replace white-based scrollbar colors with dark-on-light values:

```css
/* Global scrollbar styling */
::-webkit-scrollbar {
  width: 11px;
  background: transparent;
}

::-webkit-scrollbar-track {
  background: transparent;
}

/* Show track background on hover */
::-webkit-scrollbar:hover {
  background: rgba(0, 0, 0, 0.02);
}

::-webkit-scrollbar-thumb {
  background-color: rgba(0, 0, 0, 0.12);
  background-clip: padding-box;
  border: 4px solid transparent;
  border-radius: 20px;
}

::-webkit-scrollbar-thumb:hover {
  background-color: rgba(0, 0, 0, 0.25);
}
```
</task>

<task id="2">
**Update divider styling**

Change `livinity-divide-y` from white-based to dark-based gradient:

```css
.livinity-divide-y > * + *::before {
  content: '';
  @apply absolute inset-x-0 top-0 h-[1px] w-full bg-gradient-to-r from-transparent via-black/5 to-transparent;
}
```
</task>

<task id="3">
**Update gradient animation**

Change `livinity-bouncing-gradient` from white to dark gradient:

```css
.livinity-bouncing-gradient {
  background: linear-gradient(to right, transparent, rgba(0, 0, 0, 0.08), transparent);
  background-size: 200% 100%;
  animation: bounce-gradient 2s ease infinite;
}
```
</task>

<task id="4">
**Update onboarding blob/glow keyframes for light theme**

Update the logo glow pulse to use brand-colored or dark shadows instead of white:

```css
@keyframes logo-glow-pulse {
  0%, 100% { filter: drop-shadow(0 0 20px rgba(0, 0, 0, 0.05)); }
  50% { filter: drop-shadow(0 0 40px rgba(0, 0, 0, 0.10)); }
}
```

Update blob drift gradients — these are fine as-is since they're positioned background effects. No change needed for blob-drift-1/2/3 keyframes (they just translate/scale, no colors).
</task>

<task id="5">
**Update base text color application**

The `html, body` rule already uses `@apply text-text-primary` which will pick up the new token values from Plan 01-01. No change needed, but verify it's still correct.

Keep `::selection` as `@apply bg-brand text-white` — white text on brand selection highlight is correct even in light theme.
</task>
