<objective>
Create the IntentRouter module that classifies user messages and selects relevant capabilities from the unified registry using keyword/trigger matching with TF-IDF-like scoring, confidence thresholds, context budget management, and Redis caching. Wire it into the agent session so tool loading becomes intent-driven instead of static.

Purpose: Replace the current approach of loading ALL tools into every agent session with intelligent, intent-based capability selection. This keeps the context window lean (under 30% for tools) and improves response quality by only presenting relevant tools to the AI.

Output: `intent-router.ts` with IntentRouter class, modified `agent-session.ts` using intent-based tool selection, `lib.ts` with IntentRouter exports, modified `ws-agent.ts` with IntentRouter instantiation and injection into AgentSessionManager.

Create `nexus/packages/core/src/intent-router.ts` with the following implementation:

**Exports:** `IntentRouter` class, `IntentRouterDeps` interface, `IntentResult` interface, `ScoredCapability` type.

**IntentRouterDeps interface — note brain is OPTIONAL to support livinityd (no Brain instance available):**
```typescript
export interface IntentRouterDeps {
  redis: Redis;
  getCapabilities: () => CapabilityManifest[] | Promise<CapabilityManifest[]>;
  brain?: { think(opts: { prompt: string; tier?: string; maxTokens?: number }): Promise<string> };
}
```

Key design note: `getCapabilities` is a function (not a CapabilityRegistry instance) because IntentRouter runs in two contexts:
- In nexus `index.ts`: wraps `capabilityRegistry.list({ status: 'active' })`
- In livinityd `ws-agent.ts`: wraps an HTTP fetch to `GET /api/capabilities`
This abstraction avoids requiring CapabilityRegistry's heavy deps (SkillLoader, McpClientManager, etc.) in livinityd.

**IntentResult interface:**
```typescript
export interface IntentResult {
  capabilities: ScoredCapability[];
  fromCache: boolean;
  totalContextCost: number;
}
```

**ScoredCapability type:**
```typescript
export type ScoredCapability = CapabilityManifest & { _score: number };
```

**CORE_TOOL_NAMES constant (per D-decision: core tools always loaded):**
```typescript
const CORE_TOOL_NAMES = ['shell', 'files_read', 'files_write', 'files_list', 'sysinfo', 'docker_list', 'docker_manage', 'docker_exec'];
```

**DEFAULT_CONFIDENCE_THRESHOLD = 0.3**
**DEFAULT_CACHE_TTL_SECONDS = 3600** (1 hour)
**MAX_CAPABILITIES = 15**

**Context window sizes by model tier (approximate token counts):**
```typescript
const CONTEXT_WINDOWS: Record<string, number> = {
  opus: 200000,
  sonnet: 200000,
  haiku: 200000,
  flash: 200000,
};
const CONTEXT_BUDGET_RATIO = 0.3;
```

**Constructor:** Takes `IntentRouterDeps`. Stores deps privately.

**Method: `resolveCapabilities(message: string, tier?: string): Promise<IntentResult>`**

This is the main entry point. Steps:

1. **Cache check:** Normalize message (lowercase, strip punctuation via `message.replace(/[^\w\s]/g, '')`, split into words, sort, join with space, compute MD5 hash). Cache key = `nexus:intent:${hash}`. If Redis has a cached value, parse it, set `fromCache: true`, and return early.

2. **Get all active capabilities:** Call `await this.deps.getCapabilities()` (supports both sync and async returns).

3. **Tokenize message:** Split normalized message into word tokens. Also create a set of these tokens for O(1) lookup.

4. **Score each capability:** For each capability, compute a score using `scoreCapability(tokens: string[], tokenSet: Set<string>, cap: CapabilityManifest): number`:
   - For each trigger in `cap.triggers`: if any token exactly matches a trigger (case-insensitive), add 1.0 per match.
   - For each semantic_tag in `cap.semantic_tags`: if any token exactly matches a tag word (split multi-word tags by space/hyphen), add 0.5 per match.
   - For each token: if `cap.name.toLowerCase()` includes the token OR `cap.description.toLowerCase()` includes the token, add 0.3 (once per token that hits, not per character match).
   - Normalize: divide total score by max(1, tokens.length) to get a 0-1ish range. Cap at 1.0.

5. **Filter by threshold:** Read threshold from Redis key `nexus:config:intent_threshold` (fallback to DEFAULT_CONFIDENCE_THRESHOLD = 0.3). Filter out capabilities with score below threshold.

6. **LLM fallback:** If zero capabilities pass the threshold AND message has 3+ words AND `this.deps.brain` exists (not null/undefined), call `this.deps.brain.think()` with flash tier:
   - Prompt: `"Given this user message: \"${message}\"\n\nWhich of these capability categories are relevant? Return a JSON array of matching IDs.\n\nCapabilities:\n${capList}"`
   - Where `capList` is each capability's `id: description` (first 50 capabilities only to keep prompt small).
   - Parse the response as JSON array of capability IDs. For each returned ID, find it in the full list and assign score = 0.5.
   - maxTokens: 300
   - If `this.deps.brain` is null/undefined, skip this step entirely (livinityd context has no Brain).

7. **Sort by score descending.**

8. **Apply context budget:** Compute `budgetTokens = CONTEXT_WINDOWS[tier || 'sonnet'] * CONTEXT_BUDGET_RATIO`. Iterate scored capabilities in order. Accumulate `context_cost`. Stop adding when the next capability would exceed budget OR count reaches MAX_CAPABILITIES (15).

9. **Always include core tools:** After budget selection, check if core tool capabilities are present. For each CORE_TOOL_NAME, check if `tool:${name}` is already in the selected list. If not, find it in the full capability list and append it with `_score: 0` (these are always-on, don't count against budget).

10. **Cache result:** Serialize the result (capabilities array with _score, totalContextCost) and store in Redis with `EX ${cacheTTL}`. Log the operation.

11. **Return** `{ capabilities: selected, fromCache: false, totalContextCost }`.

**Method: `private normalizeAndHash(message: string): string`**
- Lowercase, strip punctuation (`/[^\w\s]/g`), split on whitespace, filter empty, sort, join with space.
- MD5 hash using `import { createHash } from 'node:crypto'`.
- Return hash string.

**Method: `private scoreCapability(tokens: string[], tokenSet: Set<string>, cap: CapabilityManifest): number`**
- Implement the TF-IDF-like scoring per step 4 above.
- Return the normalized score capped at 1.0.

**Method: `getToolNamesFromCapabilities(capabilities: ScoredCapability[]): string[]`**
- Public utility. For each capability, collect `provides_tools` arrays. Flatten and deduplicate.
- Return unique tool names. This is used by agent-session to build the filtered tool set.

**Logging:** Use `logger.info` for resolve results (capability count, fromCache, totalContextCost, timing). Use `logger.warn` for LLM fallback triggers. Use `logger.error` for cache/LLM failures (with graceful degradation -- never throw, always return best-effort result).

**Error handling:** All Redis and Brain calls wrapped in try/catch. If cache read fails, proceed without cache. If cache write fails, log and continue. If LLM fallback fails, log and return empty result (core tools will still be included). If brain is null/undefined, skip LLM fallback gracefully.

// Intent-based tool selection: use IntentRouter to select relevant tools
let sdkTools: ReturnType<typeof buildSdkTools> = [];
if (this.intentRouter) {
  try {
    const intentResult = await this.intentRouter.resolveCapabilities(prompt, tier);
    const intentToolNames = this.intentRouter.getToolNamesFromCapabilities(intentResult.capabilities);
    // Create a scoped registry containing only intent-matched tools
    const scopedRegistry = new ToolRegistry();
    for (const toolName of intentToolNames) {
      const tool = this.toolRegistry.get(toolName);
      if (tool) scopedRegistry.register(tool);
    }
    // Also apply tool policy (deny list etc.) on top of intent selection
    sdkTools = buildSdkTools(scopedRegistry, toolPolicy);
    logger.info('AgentSessionManager: intent-based tool selection', {
      userId,
      intentCapabilities: intentResult.capabilities.length,
      intentTools: intentToolNames.length,
      fromCache: intentResult.fromCache,
      totalContextCost: intentResult.totalContextCost,
      sdkToolCount: sdkTools.length,
    });
  } catch (err: any) {
    // Fallback to full tool set if intent routing fails
    logger.error('AgentSessionManager: intent routing failed, using full tool set', { error: err.message });
    sdkTools = this.toolRegistry ? buildSdkTools(this.toolRegistry, toolPolicy) : [];
  }
} else {
  // No intent router — use full tool set (backward compatible)
  sdkTools = this.toolRegistry ? buildSdkTools(this.toolRegistry, toolPolicy) : [];
}
