**1. Add `getApiKey` tRPC route** in `livos/packages/livinityd/source/modules/platform/routes.ts`:

Add a new `privateProcedure` query called `getApiKey` that reads the full API key from Redis (`livos:platform:api_key`) and returns `{ apiKey: string | null }`. This is needed because the existing `getStatus` route intentionally only returns `hasApiKey` and a prefix. The iframe needs the full key as a query parameter.

```typescript
getApiKey: privateProcedure.query(async ({ctx}) => {
    const redis = ctx.livinityd.ai.redis
    const apiKey = await redis.get(`${REDIS_PREFIX}api_key`)
    return { apiKey: apiKey ?? null }
}),
```

**2. Replace app-store-content.tsx** entirely with an iframe-based component:

The current `app-store-content.tsx` renders a full React app store (WindowRouterProvider, AvailableAppsProvider, route matching, discover/category/app pages). Replace ALL of this with:

- A component that fetches the API key via `trpcReact.platform.getApiKey.useQuery()` and the hostname via `trpcReact.domain.getStatus.useQuery()`
- Renders a single `<iframe>` pointing to `https://livinity.io/store?token=${apiKey}&instance=${hostname}`
- The iframe should be `width: 100%, height: 100%, border: none` to fill the window
- Show a Loading spinner while API key/domain data loads
- Show a helpful message if no API key is configured ("Connect to Livinity platform to access the App Store" or similar)
- Import and use the `useAppStoreBridge` hook (created in Task 2) passing the iframe ref so it can send messages TO the iframe
- Use `useRef<HTMLIFrameElement>` for the iframe reference

The component should look roughly like:
```tsx
import {useRef} from 'react'
import {Loading} from '@/components/ui/loading'
import {trpcReact} from '@/trpc/trpc'
import {useAppStoreBridge} from '@/hooks/use-app-store-bridge'

export default function AppStoreWindowContent() {
    const iframeRef = useRef<HTMLIFrameElement>(null)
    const apiKeyQ = trpcReact.platform.getApiKey.useQuery()
    const domainQ = trpcReact.domain.getStatus.useQuery()
    useAppStoreBridge(iframeRef)

    if (apiKeyQ.isLoading || domainQ.isLoading) return <Loading />
    const apiKey = apiKeyQ.data?.apiKey
    if (!apiKey) return <NoApiKeyMessage />
    const hostname = domainQ.data?.domain || window.location.hostname
    const storeUrl = `https://livinity.io/store?token=${encodeURIComponent(apiKey)}&instance=${encodeURIComponent(hostname)}`

    return (
        <iframe
            ref={iframeRef}
            src={storeUrl}
            style={{width: '100%', height: '100%', border: 'none'}}
            allow="clipboard-write"
            title="App Store"
        />
    )
}
```

The `NoApiKeyMessage` should be a simple centered div with text explaining the user needs to connect their LivOS instance to the Livinity platform.

Create a new hook `useAppStoreBridge(iframeRef: React.RefObject<HTMLIFrameElement | null>)` that:

**1. Listens for postMessage from the iframe:**

```typescript
useEffect(() => {
    function handleMessage(event: MessageEvent) {
        // Origin validation: only accept from livinity.io
        if (!isAllowedOrigin(event.origin)) return
        const data = event.data as StoreToLivOSMessage
        if (!data || typeof data.type !== 'string') return

        switch (data.type) {
            case 'ready':
                sendStatusToIframe()
                break
            case 'install':
                handleInstall(data.appId)
                break
            case 'uninstall':
                handleUninstall(data.appId)
                break
            case 'open':
                handleOpen(data.appId)
                break
        }
    }
    window.addEventListener('message', handleMessage)
    return () => window.removeEventListener('message', handleMessage)
}, [dependencies])
```

**2. Origin validation function:**
```typescript
function isAllowedOrigin(origin: string): boolean {
    if (origin === 'https://livinity.io') return true
    if (origin === 'https://apps.livinity.io') return true
    if (/^https:\/\/[a-z0-9-]+\.livinity\.io$/.test(origin)) return true
    // Allow localhost in development
    if (import.meta.env.DEV && /^https?:\/\/(localhost|127\.0\.0\.1)(:\d+)?$/.test(origin)) return true
    return false
}
```

**3. Define the StoreToLivOSMessage and LivOSToStoreMessage types inline** (duplicate from store types.ts since LivOS UI cannot import from the platform/web package):
```typescript
type StoreToLivOSMessage =
    | { type: 'ready' }
    | { type: 'install'; appId: string; composeUrl: string }
    | { type: 'uninstall'; appId: string }
    | { type: 'open'; appId: string }

type AppStatusEntry = { id: string; status: 'running' | 'stopped' | 'not_installed' }

type LivOSToStoreMessage =
    | { type: 'status'; apps: AppStatusEntry[] }
    | { type: 'installed'; appId: string; success: boolean; error?: string }
    | { type: 'uninstalled'; appId: string; success: boolean }
```

**4. handleInstall(appId):**
- Call `trpcClient.apps.install.mutate({appId})` (imperative, not hook -- this runs inside an event handler)
- On success: send `{ type: 'installed', appId, success: true }` to iframe via `iframeRef.current?.contentWindow?.postMessage(msg, '*')`
- On error: send `{ type: 'installed', appId, success: false, error: err.message }` to iframe
- After result, also call `sendStatusToIframe()` to send updated full app list
- Also invalidate tRPC queries: `trpcReact.useUtils()` -- get utils ref outside the handler (in hook body), call `utils.apps.list.invalidate()` and `utils.apps.state.invalidate()` after install

**5. handleUninstall(appId):**
- Call `trpcClient.apps.uninstall.mutate({appId})`
- On success: send `{ type: 'uninstalled', appId, success: true }` to iframe
- On error: send `{ type: 'uninstalled', appId, success: false }` to iframe
- After result, call `sendStatusToIframe()` and invalidate queries

**6. handleOpen(appId):**
- Open the app in a new browser tab using `window.open()` pointing to the app's subdomain
- Get the domain from `trpcReact.domain.getStatus.useQuery()` data
- URL: `https://${appId}.${domain}` (or just open on the local port if no domain configured)
- Can also use `window.location.origin` to construct a URL that opens the app within LivOS

**7. sendStatusToIframe():**
- Fetch current installed apps via `trpcClient.apps.list.query()` (imperative)
- Map each app to `{ id: app.id, status: mapState(app.state) }` where `mapState` converts the livinityd state to the bridge status:
  - 'running' | 'ready' -> 'running'
  - 'stopped' | 'stopping' -> 'stopped'
  - everything else -> 'not_installed'
- Send `{ type: 'status', apps: [...] }` to iframe via postMessage

**8. Import trpcClient from '@/trpc/trpc'** for imperative mutation/query calls. Use `trpcReact.useUtils()` for cache invalidation (call in hook body, use ref in handlers).

The hook signature:
```typescript
export function useAppStoreBridge(iframeRef: React.RefObject<HTMLIFrameElement | null>): void
```

It does not return anything -- it just sets up the side effects. The iframe ref is needed to send messages back to the iframe.

Use `useCallback` for handler functions and include proper deps. Use `useRef` to hold latest values of things that change (like the utils ref) to avoid stale closures in the message event listener.
