**Step A: Add registerCapability() and unregisterCapability() to CapabilityRegistry**

In `nexus/packages/core/src/capability-registry.ts`, add two public methods after the existing `search()` method:

```typescript
/**
 * Register a single capability manifest.
 * Adds to in-memory cache and persists to Redis.
 * Used by MarketplaceMcp after installing marketplace capabilities.
 */
async registerCapability(manifest: CapabilityManifest): Promise<void> {
  this.cache.set(manifest.id, manifest);
  const [type, ...nameParts] = manifest.id.split(':');
  const name = nameParts.join(':');
  await this.deps.redis.set(
    `${CapabilityRegistry.REDIS_PREFIX}${type}:${name}`,
    JSON.stringify(manifest),
  );
  logger.info('CapabilityRegistry: capability registered', { id: manifest.id, type: manifest.type });
}

/**
 * Unregister a capability by ID.
 * Removes from in-memory cache and Redis.
 */
async unregisterCapability(id: string): Promise<boolean> {
  const existed = this.cache.delete(id);
  if (existed) {
    const [type, ...nameParts] = id.split(':');
    const name = nameParts.join(':');
    await this.deps.redis.del(`${CapabilityRegistry.REDIS_PREFIX}${type}:${name}`);
    logger.info('CapabilityRegistry: capability unregistered', { id });
  }
  return existed;
}
```

**Step B: Create marketplace-mcp.ts**

Create `nexus/packages/core/src/marketplace-mcp.ts` with a `MarketplaceMcp` class.

The class has one public method `registerTools()` that registers all 5 tools in the ToolRegistry.

**Marketplace index pattern:** Fetch `marketplace/index.json` from GitHub (same pattern as SkillRegistryClient's catalog fetch). The index.json contains an array of `MarketplaceEntry` objects:
```typescript
interface MarketplaceEntry {
  name: string;
  type: CapabilityType;
  version: string;
  description: string;
  author?: string;
  tags: string[];
  triggers: string[];
  provides_tools: string[];
  requires: string[];
  conflicts: string[];
  context_cost: number;
  tier: 'flash' | 'sonnet' | 'opus' | 'any';
  path: string;  // e.g. "marketplace/skills/server-health"
}
```

**Caching:** Cache the fetched index in Redis at key `nexus:marketplace:index` with a 1-hour TTL. On cache hit, parse and return. On cache miss or expiry, fetch from GitHub.

**GitHub fetch:** Use the default registry URL `https://github.com/utopusc/livinity-skills`. Fetch `marketplace/index.json` via `https://raw.githubusercontent.com/utopusc/livinity-skills/main/marketplace/index.json`. If the file doesn't exist (404), return empty array (marketplace not yet populated) -- not an error.

**Tool 1: livinity_search** (per MKT-01)
- Parameters: `query` (string, required) -- keyword or tag to search, `type` (string, optional, enum: tool/skill/mcp/hook/agent) -- filter by capability type
- Execute: Fetch marketplace index, filter entries where `name`, `description`, or `tags` contain the query (case-insensitive). If type filter provided, apply it. Return JSON array of matching entries (max 20).
- Output: JSON string of matches with name, type, version, description, tags.

**Tool 2: livinity_install** (per MKT-02, MKT-03)
- Parameters: `name` (string, required) -- capability name from marketplace
- Execute:
  1. Fetch marketplace index, find entry by name. Return error if not found.
  2. **Validate manifest:** Check required fields (name, type, description, version) are present and non-empty. Return error if invalid.
  3. **Conflict detection:** Check entry's `conflicts` array against installed capabilities in CapabilityRegistry. If any conflicts found, return error listing them.
  4. **Check not already installed:** Check if `capabilityRegistry.get('{type}:{name}')` already exists with source='marketplace'. Return error if so.
  5. **Create CapabilityManifest** from the MarketplaceEntry fields, setting `source: 'marketplace'`, `status: 'active'`, `registered_at: Date.now()`, `last_used_at: 0`.
  6. **Register:** Call `capabilityRegistry.registerCapability(manifest)`.
  7. **Store install metadata** in Redis at `nexus:marketplace:installed:{name}` with JSON `{name, type, version, installedAt}`.
  8. Return success message with the capability ID.

**Tool 3: livinity_uninstall** (per MKT-01)
- Parameters: `name` (string, required), `type` (string, required, enum: tool/skill/mcp/hook/agent)
- Execute:
  1. Build ID as `{type}:{name}`.
  2. Check the capability exists in CapabilityRegistry and has `source: 'marketplace'`. Return error if not found or if source is not 'marketplace' (cannot uninstall builtins).
  3. Call `capabilityRegistry.unregisterCapability(id)`.
  4. Remove Redis metadata at `nexus:marketplace:installed:{name}`.
  5. Return success message.

**Tool 4: livinity_recommend** (per MKT-01)
- Parameters: none required (optional: `limit` number, default 5)
- Execute:
  1. Fetch marketplace index.
  2. Get all installed capabilities from capabilityRegistry.list().
  3. Compute tag overlap: for each marketplace entry NOT already installed, count how many of its tags overlap with tags of installed capabilities. Score = overlap count.
  4. Sort by score descending, return top `limit` entries.
  5. If no installed capabilities have tags, return the first `limit` entries from marketplace as "popular" defaults.
  6. Output: JSON array of recommended entries with score.

**Tool 5: livinity_list** (per MKT-01)
- Parameters: `source` (string, optional, enum: marketplace/all, default: all)
- Execute:
  1. If source='marketplace': list capabilities where `source === 'marketplace'` from CapabilityRegistry.
  2. If source='all': list ALL capabilities from CapabilityRegistry.
  3. Return JSON array with id, name, type, description, status, source.

All tools follow the existing Tool interface pattern: `{ name, description, parameters: ToolParameter[], execute }` returning `ToolResult`.

Error handling: Each tool wraps its logic in try/catch and returns `{ success: false, output: '', error: message }` on failure.
