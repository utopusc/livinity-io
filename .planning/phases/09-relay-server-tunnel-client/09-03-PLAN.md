**ReconnectionManager** (private class or inline logic):
- `attempt: number` starts at 0
- `baseDelay: 1000` (1 second)
- `maxDelay: 60_000` (60 seconds)
- `maxJitter: 1000` (1 second)
- `getNextDelay(): number` -- exponentialDelay = min(baseDelay * 2^attempt, maxDelay), jitter = random * min(maxJitter, exponentialDelay), increment attempt, return exponentialDelay + jitter
- `reset(): void` -- set attempt to 0
- `get consecutiveFailures(): number` -- return attempt

**TunnelClient class:**

Constructor takes `{ redis: Redis, relayUrl?: string }`:
- `redis` -- livinityd's existing Redis connection
- `relayUrl` -- defaults to 'wss://relay.livinity.io' (configurable for dev)

Properties:
- `private ws: WebSocket | null`
- `private sessionId: string | null`
- `private assignedUrl: string | null`
- `private reconnectTimer: NodeJS.Timeout | null`
- `private reconnection: ReconnectionManager`
- `private status: 'idle' | 'connecting' | 'connected' | 'disconnected' | 'error'`
- `private localWsSockets: Map<string, WebSocket>` -- for WS proxying (populated in Plan 04)

**Methods:**

`async start(): Promise<void>`:
- Check Redis key `livos:platform:api_key` -- if not set, set status to 'idle', return
- Call `connect()`

`async connect(): Promise<void>`:
- Get API key from Redis `livos:platform:api_key`
- If no API key, return
- Set status to 'connecting', update Redis `livos:platform:status`
- Get stored session ID from Redis `livos:platform:session_id` (for reconnection)
- Create WebSocket: `new WebSocket(this.relayUrl + '/tunnel/connect')`
- On 'open': send TunnelAuth message `{ type: 'auth', apiKey, sessionId: storedSessionId || undefined }`
- On 'message': parse JSON, route by type:
  - 'connected' -> store sessionId and assignedUrl in Redis, set status='connected', update Redis status, call reconnection.reset(), log
  - 'auth_error' -> set status='error', update Redis status, log error, do NOT reconnect (bad API key)
  - 'http_request' -> call handleHttpRequest(msg)
  - 'ws_upgrade' -> no-op stub for now (Plan 04 adds implementation), log warning
  - 'ping' -> send pong: `{ type: 'pong', ts: msg.ts }`
  - 'relay_shutdown' -> log, expect connection close, reconnect
  - 'quota_exceeded' -> log warning, update Redis status
  - 'ws_frame' -> no-op stub (Plan 04)
  - 'ws_close' -> no-op stub (Plan 04)
- On 'close': if status was 'connected' or 'connecting', schedule reconnect. Set status='disconnected', update Redis.
- On 'error': log, (close handler will fire next and trigger reconnect)

`private scheduleReconnect(): void`:
- If status is 'error' (bad API key), do not reconnect
- Clear any existing reconnectTimer
- Get delay from reconnection.getNextDelay()
- Log: `Reconnecting in ${delay}ms (attempt ${reconnection.consecutiveFailures})`
- Set reconnectTimer = setTimeout(() => this.connect(), delay)

`private async handleHttpRequest(msg: TunnelRequest): Promise<void>`:
- Create local HTTP request to `127.0.0.1:8080`:
  - method: msg.method
  - path: msg.path
  - headers: copy msg.headers, replace `host` with `127.0.0.1:8080`
  - Add `X-Forwarded-For` and `X-Forwarded-Proto: https` headers
- Collect response body chunks
- On local response end: build TunnelResponse { type: 'http_response', id: msg.id, status, headers, body (base64 or null) }
- Send through tunnel ws
- On local request error: send TunnelResponse with status 502, body "Local server error: ${err.message}" (base64)
- If msg.body: write Buffer.from(msg.body, 'base64') to local request before ending

`async disconnect(): Promise<void>`:
- Clear reconnectTimer
- Close WebSocket if open
- Set status='idle', clear Redis keys (status, session_id, url)
- Set ws to null

`async stop(): Promise<void>`:
- Call disconnect()

`getStatus(): { status: string; url: string | null; sessionId: string | null }`:
- Return current status, assignedUrl, sessionId (synchronous, for tRPC)

**Redis keys used** (prefix `livos:platform:`):
- `api_key` -- stored API key string
- `enabled` -- '1' or '0'
- `status` -- 'idle' | 'connecting' | 'connected' | 'disconnected' | 'error'
- `url` -- assigned URL (e.g., 'testuser.livinity.io')
- `session_id` -- session ID for reconnection

**Import notes:**
- Use `import WebSocket from 'ws'` -- ws uses default export in livinityd's CJS/ESM hybrid setup. Check existing livinityd code for the correct import pattern (look at how other modules import ws).
- Use `import http from 'node:http'` for the local request
- Use `import { Redis } from 'ioredis'` for the type annotation

Export the `TunnelClient` class.
