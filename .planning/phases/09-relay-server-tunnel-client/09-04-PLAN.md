**ws-proxy.ts** -- WebSocket upgrade proxying through the tunnel:

Maintain a module-level Map: `pendingUpgrades: Map<string, PendingWsUpgrade>` where:
```typescript
interface PendingWsUpgrade {
  req: http.IncomingMessage
  socket: import('stream').Duplex
  head: Buffer
  timeout: NodeJS.Timeout
  tunnel: TunnelConnection
}
```

Export `handleWsUpgrade(req, socket, head, tunnel: TunnelConnection): void`:
1. Generate wsId with nanoid()
2. Build TunnelWsUpgrade message: { type: 'ws_upgrade', id: wsId, path: req.url, headers: req.headers, targetApp: (parse from subdomain if present) }
3. Set timeout: config.WS_UPGRADE_TIMEOUT_MS -- on expiry, delete from pendingUpgrades, destroy socket
4. Store in pendingUpgrades Map
5. Send message through tunnel.ws

Export `handleWsReady(msg: TunnelWsReady, tunnel: TunnelConnection): void`:
1. Get pending upgrade from Map by msg.id
2. If not found, return (timed out)
3. Clear timeout, delete from pendingUpgrades
4. Create a WebSocketServer({ noServer: true }) -- one per upgrade (use `new (await import('ws')).WebSocketServer({ noServer: true })` or import at module level)
5. Call wss.handleUpgrade(pending.req, pending.socket, pending.head, (browserWs) => { ... })
6. Inside the callback:
   - Store browserWs in tunnel.addBrowserSocket(msg.id, browserWs)
   - browserWs.on('message', (data, isBinary) => { send TunnelWsFrame through tunnel: { type: 'ws_frame', id: msg.id, data: Buffer.from(data).toString('base64'), binary: isBinary } })
   - browserWs.on('close', (code, reason) => { send TunnelWsClose through tunnel, remove from tunnel.removeBrowserSocket(msg.id) })
   - browserWs.on('error', () => { send TunnelWsClose through tunnel, remove from tunnel.removeBrowserSocket(msg.id) })

Export `handleWsFrame(msg: TunnelWsFrame, tunnel: TunnelConnection): void`:
1. Get browserWs from tunnel.getBrowserSocket(msg.id)
2. If not found or not OPEN, return
3. Decode data: Buffer.from(msg.data, 'base64')
4. Send to browser: browserWs.send(decoded, { binary: msg.binary })

Export `handleWsClose(msg: TunnelWsClose, tunnel: TunnelConnection): void`:
1. Get browserWs from tunnel.getBrowserSocket(msg.id)
2. If found, close it with msg.code and msg.reason
3. Remove from tunnel.removeBrowserSocket(msg.id)

Export `handleWsError(msg: TunnelWsError, tunnel: TunnelConnection): void`:
1. Get pending upgrade from pendingUpgrades by msg.id
2. If found, clear timeout, destroy socket, delete from Map
3. Also check tunnel.getBrowserSocket(msg.id) and close it if found

**tunnel-client.ts** -- Implement WS upgrade forwarding (replace the no-op stubs from Plan 03):

Add a `localWsSockets: Map<string, WebSocket>` property (should already exist from Plan 03).

Implement `handleWsUpgrade(msg: TunnelWsUpgrade)`:
1. Build target URL: `ws://127.0.0.1:8080${msg.path}`
2. Add forwarded headers to the WS connection options (ws library supports `headers` option)
3. Create local WebSocket: `new WebSocket(targetUrl, { headers: { ...msg.headers, host: '127.0.0.1:8080' } })`
4. On local WS 'open': send TunnelWsReady through tunnel `{ type: 'ws_ready', id: msg.id }`
5. Store in localWsSockets Map
6. On local WS 'message' (data, isBinary): send TunnelWsFrame through tunnel `{ type: 'ws_frame', id: msg.id, data: Buffer.from(data).toString('base64'), binary: isBinary }`
7. On local WS 'close' (code, reason): send TunnelWsClose through tunnel, remove from Map
8. On local WS 'error': send TunnelWsError through tunnel `{ type: 'ws_error', id: msg.id, error: err.message }`, remove from Map

Implement `handleWsFrame(msg: TunnelWsFrame)`:
1. Get local WS from localWsSockets by msg.id
2. If not found or not OPEN, return
3. Decode: Buffer.from(msg.data, 'base64')
4. Send to local WS: localWs.send(decoded, { binary: msg.binary })

Implement `handleWsClose(msg: TunnelWsClose)`:
1. Get local WS from localWsSockets by msg.id
2. If found, close with msg.code and msg.reason
3. Remove from Map

Update the message router in the 'message' handler to call these methods:
- 'ws_upgrade' -> this.handleWsUpgrade(msg)
- 'ws_frame' -> this.handleWsFrame(msg)
- 'ws_close' -> this.handleWsClose(msg)

On tunnel disconnect (in the close handler or disconnect method): close ALL local WebSockets in localWsSockets Map and clear it.

**tunnel-registry.ts** -- Add session reconnection buffering (RELAY-07):

Add to TunnelConnection class:
- `private reconnecting: boolean = false`
- `private requestBuffer: Array<{ req: http.IncomingMessage; res: http.ServerResponse }> = []`
- `private reconnectTimeout: NodeJS.Timeout | null = null`
- `private static MAX_BUFFER_REQUESTS = 100` (from config)

Add method `enterReconnectMode(): void`:
1. Set reconnecting = true
2. Set reconnectTimeout = setTimeout(() => this.destroy('reconnect timeout'), config.RECONNECT_BUFFER_MS)
3. Do NOT destroy the WebSocket (it's already closed)
4. Do NOT clear pendingRequests (let them timeout naturally)

Add method `isReconnecting(): boolean` -- return reconnecting

Add method `bufferRequest(req, res): boolean`:
1. If buffer is full (>= MAX_BUFFER_REQUESTS), return false
2. Push { req, res } to requestBuffer
3. Return true

Add method `resumeSession(newWs: WebSocket): void`:
1. Set this.ws to newWs (needs to be made settable -- change readonly to private with a setter, or use a method)
2. Clear reconnectTimeout
3. Set reconnecting = false
4. Restart heartbeat (clear old interval, start new)
5. Flush buffered requests: for each { req, res } in requestBuffer, call the proxy function to send through the new tunnel
6. Clear requestBuffer
7. Set up message handler on new ws (same as initial connection)

Update TunnelRegistry.register():
- When registering a connection for a username that already has a RECONNECTING tunnel:
  - If the new connection provides a sessionId matching the existing session: call existing.resumeSession(newWs)

Update TunnelRegistry.unregister() or add a method that's called when tunnel disconnects:
- Instead of immediately destroying, check if reconnection buffering should be used
- Call enterReconnectMode() instead of destroy()
- The caller (index.ts onTunnelConnect close handler) should call `registry.markDisconnected(username)` which enters reconnect mode
