<objective>
Close the ownership loop end-to-end for OWN-02 and OWN-03:

1. **OWN-02 (registration binding):** Confirm and reinforce that the existing OAuth Device Grant approval flow binds devices to the authenticated user. The `approve` endpoint already returns 401 on missing session, and `createDeviceRecord` (hardened in Plan 11-01) rejects missing userId. This plan adds explicit grep-visible comments marking the invariant and adds the missing user-facing `GET /api/devices` REST endpoint.

2. **OWN-03 (owner-filtered listing):**
   - Modify livinityd's tRPC `devices.list` to filter the Redis cache by `ctx.currentUser.id`.
   - Extend the relay's `device_connected` tunnel event to carry `userId` so LivOS can populate the cache without a Postgres hop.
   - Add `userId` to the Redis `livos:devices:*` JSON blob and in-memory `ConnectedDevice` state.
   - Add a `getDevicesForUser(userId)` method to DeviceBridge.
   - Enforce ownership on tRPC `devices.rename`/`devices.remove` (preview of Phase 12 defense in depth).
   - Create `platform/web/src/app/api/devices/route.ts` exporting a GET handler that returns only the session user's devices.

Purpose: Phase 11 guarantees the invariant "a user only ever sees their own devices". This is the foundation that Phase 12's authorizeDeviceAccess helper will consume, Phase 14's session binding will verify, and Phase 16's admin override will bypass only through explicit admin endpoints.

Output: Modified relay protocol + forwarding, modified LivOS tunnel client + device bridge + tRPC routes, new platform REST endpoint.
</objective>

<architecture_note>
Device flow (what exists today, post Plan 11-01):

1. Agent binary calls POST platform/web/src/app/api/device/register/route.ts (no auth) → creates device_grants row (user_id NULL).
2. User visits livinity.io/device in browser, enters user_code → POST platform/web/src/app/api/device/approve/route.ts (requires session cookie, returns 401 if absent) → UPDATE device_grants SET user_id = session.userId, status = 'approved'.
3. Agent polls POST platform/web/src/app/api/device/token/route.ts with device_code → once approved, calls createDeviceRecord(grant.userId, ...) → INSERT INTO devices (user_id, ...) → returns signed device JWT.
4. Agent connects to relay `/device/connect` → relay reads token.userId + token.deviceId → deviceRegistry.register(userId, deviceId, ...) → relay forwards `device_connected` tunnel event to the user's LivOS tunnel (but WITHOUT the userId field today).
5. LivOS tunnel-client.ts handles `device_connected` → DeviceBridge.onDeviceConnected(event) → sets Redis `livos:devices:{deviceId}` with deviceId/deviceName/platform/tools/connectedAt (NO userId today) + registers proxy tools in Nexus.
6. tRPC devices.list (adminProcedure) returns ALL devices from Redis (no filtering).

After this plan:
- Step 4 tunnel event will carry userId.
- Step 5 will store userId in Redis + in-memory cache.
- Step 6 will be privateProcedure + filter by ctx.currentUser.id via getDevicesForUser().
- NEW: GET platform/web/src/app/api/devices/route.ts returns the caller's devices direct from Postgres.
</architecture_note>

import { NextRequest, NextResponse } from 'next/server';
import { getSession, SESSION_COOKIE_NAME } from '@/lib/auth';
import pool from '@/lib/db';

/**
 * GET /api/devices
 *
 * Phase 11 OWN-03: Returns the list of devices owned by the authenticated
 * session user. Unauthenticated requests get 401. Admin users see ONLY their
 * own devices — admin-wide cross-user listing is Phase 16 (GET /api/admin/devices).
 */
export async function GET(req: NextRequest) {
  try {
    const sessionToken = req.cookies.get(SESSION_COOKIE_NAME)?.value;
    if (!sessionToken) {
      return NextResponse.json({ error: 'Authentication required' }, { status: 401 });
    }

    const session = await getSession(sessionToken);
    if (!session) {
      return NextResponse.json({ error: 'Invalid session' }, { status: 401 });
    }

    // Phase 11 OWN-03: WHERE user_id = session.userId enforces per-request ownership
    // at the REST layer (defense in depth — tRPC already filters on the livinityd side).
    const result = await pool.query<{
      device_id: string;
      device_name: string;
      platform: string;
      created_at: Date;
      last_seen: Date | null;
      revoked: boolean;
    }>(
      `SELECT device_id, device_name, platform, created_at, last_seen, revoked
       FROM devices
       WHERE user_id = $1 AND revoked = false
       ORDER BY created_at DESC`,
      [session.userId],
    );

    return NextResponse.json({
      devices: result.rows.map((row) => ({
        deviceId: row.device_id,
        deviceName: row.device_name,
        platform: row.platform,
        createdAt: row.created_at.toISOString(),
        lastSeen: row.last_seen ? row.last_seen.toISOString() : null,
        revoked: row.revoked,
      })),
    });
  } catch (err) {
    console.error('[devices] List error:', err);
    return NextResponse.json({ error: 'Internal server error' }, { status: 500 });
  }
}
