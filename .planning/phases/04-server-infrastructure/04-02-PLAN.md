3. Update apps.ts nativeAppSubdomains mapping (in the same rebuildCaddy method modified in Task 2) to pass the `streaming` flag. The mapping should become:
```typescript
const nativeAppSubdomains = this.nativeInstances.map((app) => {
    return {
        subdomain: app.subdomain,
        port: app.proxyPort,
        streaming: app.id === 'desktop-stream',
    }
})
```

This means apps.ts is also modified in this task (adding the `streaming` property to the mapping object).

**Why stream_close_delay 5m:**
Per PITFALLS.md Pitfall 1, every `caddy reload` kills active WebSocket connections. `stream_close_delay 5m` gives the client 5 minutes to reconnect gracefully after a config change, rather than instant termination. This is critical because LivOS calls `caddy reload` whenever apps are installed or domain config changes.

**Why stream_timeout 24h:**
Prevents truly indefinite connections, but allows long desktop sessions (up to 24 hours). Users who leave the desktop viewer open all day won't be forcibly disconnected.
