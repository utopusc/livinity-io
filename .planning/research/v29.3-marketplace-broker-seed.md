## Goal (one sentence)

Marketplace AI uygulamaları (MiroFish, Dify, RAGFlow, CrewAI agent'ları gibi) — kullanıcının Livinity UI'da yaptığı **Claude OAuth login**'i kullanarak, kendi API key'lerine ihtiyaç duymadan, **ToS-uyumlu per-user multi-user** desteğiyle Claude'a erişebilsin.

## Why now

- Livinity zaten kullanıcı için Claude Code OAuth flow'u implementli (`nexus/packages/core/src/providers/claude.ts:381-502`, public client_id `9d1c250a-e61b-44d9-88ed-5944d1962f5e` + PKCE).
- Marketplace'e MiroFish gibi `LLM_BASE_URL` üzerinden OpenAI-compatible endpoint kabul eden uygulamalar eklemek isteniyor.
- Bu uygulamalardan API key istemek yerine Claude subscription'ı kullanabilirsek **kullanıcı deneyimi dramatik olarak iyileşir** (BYOK yerine "zaten login'sin").

## Background research (already done)

### Anthropic ToS — kritik nokta

Anthropic 4 Nisan 2026'da resmi olarak yasakladı:

> "Anthropic does not permit third-party developers to offer Claude.ai login or to **route requests through Free, Pro, or Max plan credentials on behalf of their users**."

Anahtar kelime: **çoğul "users"**. Yani bir SaaS proxy'nin çoklu kullanıcının trafiğini başka kullanıcının aboneliği üzerinden Anthropic'e geçirmesi yasak. Bu OpenClaw'ı kapattı.

**Livinity'nin durumu farklı:**
- ✅ Self-hosted (kullanıcının kendi Mini PC'si, kendi donanımı)
- ✅ Kullanıcının kendi Claude aboneliği
- ✅ Resmi `claude login` OAuth akışı (Anthropic'in onayladığı yol)
- ✅ Agent SDK = Anthropic'in zaten subscription kullanılması için yaptığı SDK

Anthropic için tek-kullanıcı self-hosted Mini PC, claude CLI'ın laptop'ta çalışmasından ayırt edilemez. Banlanmıyor.

**Ama** — multi-user'da tek admin aboneliğine fan-out: bu **yasak desen**. Trafik hacminden tespit edilebilir. Çözüm: per-user OAuth.

### Mevcut Livinity Claude entegrasyonu (codebase analizi)

İki rota var, `claude_auth_method` Redis key'iyle seçiliyor:

- **Rota A (`api-key` modu):** `nexus/packages/core/src/providers/claude.ts:121-272` — `@anthropic-ai/sdk` ile `client.messages.create()` direkt HTTP. Subscription token'ı burada değil, BYOK API key kullanılıyor. ✅ Meşru.
- **Rota B (`sdk-subscription` modu):** `nexus/packages/core/src/sdk-agent-runner.ts:309-324` — `@anthropic-ai/claude-agent-sdk`'nın `query()` fonksiyonu, altında gerçek `claude` CLI subprocess. Anthropic için "Claude Code çalışıyor" görünür. ✅ Meşru.

### ⚠️ Bilinen risk (claude.ts:82-119)

`ClaudeProvider.getClient()` — Redis'te API key yoksa ama `~/.claude/.credentials.json` varsa OAuth token'ı `authToken` olarak `@anthropic-ai/sdk`'ya geçiriyor:

```typescript
const token = creds.claudeAiOauth?.accessToken;
if (token) {
  this.client = new Anthropic({ authToken: token });
}
```

Bu **OAuth subscription token + raw `/v1/messages` direkt çağrı**. Anthropic'in fingerprint detection'ının (Nisan 2026 OpenClaw banı) tam hedefi. Subscription kullanıcısı `chat()` çağrısı yaparsa (agent loop yerine direkt mesajlaşma), bu fallback tetiklenir.

**Bu v29.3'ün ilk işi** — kapat veya sadece Agent SDK rotasına yönlendir.

## Architecture (proposed)

### Multi-user OAuth model (ToS-compliant)

Her kullanıcı kendi `claude login`'unu yapar, kendi `~/.claude/.credentials.json`'u kendi user HOME'unda olur. Livinity her user için ayrı `SdkAgentRunner` sürecini o user'ın HOME'uyla başlatır.

```
User A → SdkAgentRunner (HOME=/home/user-a) → ~user-a/.claude/.credentials.json → Anthropic (User A's subscription)
User B → SdkAgentRunner (HOME=/home/user-b) → ~user-b/.claude/.credentials.json → Anthropic (User B's subscription)
```

Anthropic için: "İki ayrı kullanıcı kendi laptop'larında Claude Code çalıştırıyor" görünümü.

### Marketplace broker

Marketplace app → Livinity Broker (`/v1/messages` Anthropic-format + `/v1/chat/completions` OpenAI-compat) → `SdkAgentRunner.run()` (current user's OAuth) → Anthropic.

```
Marketplace app (örn. MiroFish container)
  │ ANTHROPIC_BASE_URL=http://livinity-broker:8080
  │ LLM_BASE_URL=http://livinity-broker:8080/v1
  ▼
Livinity Broker (Mini PC, lokal)
  │ Anthropic Messages API formatında istek alır
  │ Container header'ından user_id'yi tespit eder
  │ ↓
  │ SdkAgentRunner.run(prompt) — o user'ın HOME'uyla
  │ (kullanıcının kendi `claude login` OAuth'unu kullanır)
  │ ↓
  ▼
Anthropic API (o user'ın subscription'ından düşer)
```

**Kritik:** Broker raw HTTP forward yapmaz; her zaman Agent SDK `query()` üzerinden gider. Anthropic için "kullanıcı Claude Code/Agent SDK kullanıyor" demektir.

### Marketplace app uyumluluğu

| App tipi | Yönlendirme | Kod değişikliği |
|----------|-------------|-----------------|
| OpenAI-compat (MiroFish, CrewAI, çoğu agent) | `LLM_BASE_URL` env var | Hayır |
| Resmi `@anthropic-ai/sdk` kullananlar | `ANTHROPIC_BASE_URL` (SDK v0.25+ destekler) | Hayır |
| LibreChat | `ANTHROPIC_REVERSE_PROXY` | Hayır |
| Dify, RAGFlow | Admin UI'dan base_url + container start otomasyonu | Yapılabilir |

Tahmini **%80-90** marketplace AI app'ine sıfır kod değişikliği ile bağlanırız.

## Constraints / non-negotiables (LOCKED)

- **D-TOS-01:** Multi-user'da TEK admin aboneliği fan-out YASAK. Her user kendi OAuth'unu yapacak.
- **D-TOS-02:** Broker raw HTTP forward yapmayacak. Her request Agent SDK `query()` üzerinden gidecek.
- **D-RISK-01:** `claude.ts` OAuth fallback (raw `@anthropic-ai/sdk` + subscription token) v29.3'ün ilk işinde kapatılacak.
- **D-CONTAINER-01:** Marketplace app'leri Docker container'larda çalışıyor → broker hostname `livinity-broker` internal DNS olur, env var injection trivial.
- **D-NO-SERVER4:** Server4 OFF-LIMITS (proje memory hard rule'u, 2026-04-27).

## Draft requirements (refine in `/gsd-new-milestone`)

### Category: Risk Fix (FR-RISK)
- **FR-RISK-01:** `ClaudeProvider.getClient()` OAuth fallback (claude.ts:99-115) kaldır VEYA sadece "subscription kullanıcılarını her zaman SdkAgentRunner'a yönlendir" guard'ı ekle. Subscription token'ı asla `@anthropic-ai/sdk`'ya direkt geçmeyecek.

### Category: Per-User OAuth (FR-AUTH)
- **FR-AUTH-01:** Multi-user mode'da her user için bağımsız `claude login` flow. Her user'ın kendi `~/.claude/.credentials.json`'u kendi user HOME'unda saklanır.
- **FR-AUTH-02:** Settings > AI Integrations > "Connect my Claude account" butonu (per-user). Login durumu UI'da görünür (her user için).
- **FR-AUTH-03:** SdkAgentRunner spawn'ı `HOME=/home/<user>` ile yapılır (cross-user OAuth credential leak engellenir).
- **FR-AUTH-04:** Admin "single subscription fan-out" mode'u açıkça opt-in olur, ToS uyarısı gösterilir, hacim limitlenir (ör. günlük 100 msg).

### Category: Anthropic-format Broker (FR-BROKER-A)
- **FR-BROKER-A-01:** `POST /v1/messages` HTTP endpoint (livinityd üzerinde, container network'te exposed).
- **FR-BROKER-A-02:** Streaming desteği (SSE) — Anthropic Messages API streaming şemasıyla uyumlu.
- **FR-BROKER-A-03:** Request → SdkAgentRunner.run() çevrimi — Anthropic Messages format'ındaki messages array'ini Agent SDK `prompt` + `options` formatına çevirir.
- **FR-BROKER-A-04:** Container request'lerini user'a bağlama — container'ın owner user_id'sinden HOME'unu çözer.

### Category: OpenAI-compat Broker (FR-BROKER-O)
- **FR-BROKER-O-01:** `POST /v1/chat/completions` HTTP endpoint — OpenAI Chat Completions format.
- **FR-BROKER-O-02:** OpenAI ↔ Anthropic format çevirisi (mesaj history, tool calls, streaming).
- **FR-BROKER-O-03:** Model adı mapping: `gpt-4`, `claude-sonnet-4-6` vb. → broker daima `claude-sonnet-4-6` Anthropic model'ine forward.
- **FR-BROKER-O-04:** Optional fallback to LiteLLM sidecar — eğer çeviri katmanını sıfırdan yazmak yerine LiteLLM'i broker arkasına koymak istenirse, container'da paketleyip kullan.

### Category: Marketplace Integration (FR-MARKET)
- **FR-MARKET-01:** Marketplace app install template'ine env var auto-injection: `ANTHROPIC_BASE_URL`, `ANTHROPIC_REVERSE_PROXY`, `LLM_BASE_URL` (broker hostname `http://livinity-broker:8080`).
- **FR-MARKET-02:** App manifest'te `requires_ai_provider: true` flag → broker env var'ları otomatik enjekte.
- **FR-MARKET-03:** Per-app Anthropic key transport mode flag: `subscription` (broker üzerinden) | `byok` (kullanıcı kendi key'ini girer) | `auto` (subscription varsa onu kullan, yoksa BYOK iste).

### Category: Usage Dashboard (FR-DASH)
- **FR-DASH-01:** Per-user usage stats: kaç token, hangi app, aylık total.
- **FR-DASH-02:** Per-app usage stats: hangi user, hangi model, kaç istek.
- **FR-DASH-03:** Subscription rate limit görünürlüğü (Pro 200/day, Max 5x); kullanıcı yaklaştığında uyarı.

## Open questions (resolve in `/gsd-discuss-phase`)

1. **Hangi user'a forward?** Container'ın tag'inden mi (label `livos.user_id=...`), gateway middleware'den mi (mevcut `apps.ts` middleware'i container'ı user'a zaten bağlıyor), yoksa ayrı bir auth header mı? (Önerim: gateway middleware mevcut, onu broker call site'da reuse et.)
2. **Tek pod mu broker, per-user mi?** Tek broker process olur (lightweight HTTP wrapper) ama her request kendi user'ının HOME'uyla SdkAgentRunner spawn eder.
3. **OpenAI format çevirisi:** sıfırdan mı yazalım, yoksa LiteLLM container'ını sidecar olarak mı kullanalım? (Önerim: LiteLLM kanıtlanmış, daha az iş.)
4. **Hangi marketplace app'leri ilk seçilsin?** (MiroFish, Dify, RAGFlow, CrewAI agent template — hangisi anchor olsun?)
5. **Subscription rate limit handling:** Pro 200/day dolduğunda broker 429 mu döner, yoksa BYOK fallback mı?
6. **Token observability:** Anthropic Messages API response'unda `usage.input_tokens` + `usage.output_tokens` döner. Broker bunları her request'te tutar mı (DB tablosu)?
7. **Phase sayısı:** 4-6 faz arası tahmin (risk fix + auth refactor + Anthropic broker + OpenAI compat + marketplace integration + dashboard). `/gsd-new-milestone` discuss'ta netleşir.
