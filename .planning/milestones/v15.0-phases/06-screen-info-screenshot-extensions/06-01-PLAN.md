**1. Create `agent/src/tools/screen-info.ts`** (SCREEN-01)

Create a new tool file following the same lazy-load pattern as screenshot.ts. Use node-screenshots (NOT robotjs getScreenSize) because it provides much richer data: multi-monitor, scaling factor, rotation, window info.

Lazy loading pattern: reuse the same async `import('node-screenshots')` pattern from screenshot.ts (NOT the synchronous require() from mouse.ts -- node-screenshots is ESM).

```typescript
let Monitor: any = null;
let WindowClass: any = null;
let loadError: string | null = null;
let loaded = false;

async function ensureLoaded(): Promise<void> {
  if (loaded) return;
  loaded = true;
  try {
    const mod = await import('node-screenshots');
    Monitor = mod.Monitor;
    WindowClass = mod.Window;
  } catch (err: unknown) {
    loadError = err instanceof Error ? err.message : String(err);
  }
}
```

Export `executeScreenInfo(params)` that returns:
- `data.displays`: Array of display objects, each with: `{ id, name, x, y, width, height, scaleFactor, rotation, frequency, isPrimary, isBuiltin }`
- `data.displayCount`: number of displays
- `data.primaryDisplay`: the primary display object (or first if none primary)
- `data.activeWindow`: `{ title, appName, x, y, width, height, pid }` of the focused window, or `null` if none focused
- `output`: Human-readable summary like "2 displays: Primary 1920x1080 @1.0x, Secondary 2560x1440 @1.25x | Active window: 'My Document - Chrome' (1200x800 at 100,50)"

Call Monitor.all() to get displays. Call WindowClass.all() to get windows, find the one where isFocused() is true.

All Monitor/Window methods are method calls (e.g., `monitor.width()` not `monitor.width`).

Wrap everything in try/catch. If node-screenshots fails to load, return `{ success: false, output: '', error: 'Screen info unavailable: ' + loadError }`.

**2. Extend `agent/src/tools/screenshot.ts`** (SCREEN-02)

Modify the existing `executeScreenshot` function to return richer metadata in the `data` field. Currently it returns `{ width, height, size }`. Extend to also include:

- `scaleFactor`: from `monitor.scaleFactor()` -- critical for the AI to map image pixels to physical screen coordinates
- `monitorX`: from `monitor.x()` -- monitor's position in virtual screen space
- `monitorY`: from `monitor.y()` -- monitor's position in virtual screen space
- `monitorWidth`: from `monitor.width()` -- monitor's pixel dimensions
- `monitorHeight`: from `monitor.height()` -- monitor's pixel dimensions
- `isPrimary`: from `monitor.isPrimary()`
- `rotation`: from `monitor.rotation()`

The `width` and `height` in data are the captured IMAGE dimensions (from `image.width`, `image.height`). The `monitorWidth` and `monitorHeight` are the MONITOR dimensions. These may differ on high-DPI displays where the image captures at physical resolution.

Also import Window from node-screenshots and include `activeWindow` in data:
- `activeWindow`: `{ title, appName, x, y, width, height }` from the focused Window, or `null`

Update the ensureLoaded function to also capture Window:
```typescript
let WindowClass: any = null;
// in ensureLoaded:
WindowClass = mod.Window;
```

Update the output string to include scale factor:
```
`Screenshot captured: ${image.width}x${image.height} @${monitor.scaleFactor()}x (${jpegBuffer.length} bytes JPEG)`
```

The data object becomes:
```typescript
data: {
  width: image.width,
  height: image.height,
  size: jpegBuffer.length,
  scaleFactor: monitor.scaleFactor(),
  monitorX: monitor.x(),
  monitorY: monitor.y(),
  monitorWidth: monitor.width(),
  monitorHeight: monitor.height(),
  isPrimary: monitor.isPrimary(),
  rotation: monitor.rotation(),
  activeWindow: focusedWindow ? {
    title: focusedWindow.title(),
    appName: focusedWindow.appName(),
    x: focusedWindow.x(),
    y: focusedWindow.y(),
    width: focusedWindow.width(),
    height: focusedWindow.height(),
  } : null,
}
```

Get focusedWindow by calling `WindowClass.all()` and finding `w.isFocused() === true`. Wrap in try/catch and default to null if window enumeration fails (some platforms may not support it).
