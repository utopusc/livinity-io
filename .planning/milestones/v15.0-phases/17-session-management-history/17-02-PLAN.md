**Add conversationId tracking to the hook:**

3. Add a `conversationIdRef` (useRef) to track the current conversation ID. This ref is updated when sendMessage is called and when loadConversation is called. Using a ref (not state) because the WebSocket send needs the latest value without re-renders.
   ```typescript
   const conversationIdRef = useRef<string | null>(null)
   ```

4. Modify `sendMessage` to accept an optional `conversationId` parameter and include it in the WebSocket `start` payload:
   ```typescript
   const sendMessage = useCallback(
     (prompt: string, model?: string, conversationId?: string) => {
       const ws = wsRef.current
       if (!ws || ws.readyState !== WebSocket.OPEN) return

       resetBuffer()

       // Track the active conversation
       if (conversationId) conversationIdRef.current = conversationId

       const userMsg: ChatMessage = {
         id: `msg_${Date.now()}_user`,
         role: 'user',
         content: prompt,
         timestamp: Date.now(),
       }
       dispatch({type: 'ADD_USER_MESSAGE', message: userMsg})
       dispatch({type: 'START_ASSISTANT_MESSAGE', id: `msg_${Date.now()}_assistant`})
       setIsStreaming(true)

       const payload: {type: string; prompt: string; sessionId?: string; model?: string; conversationId?: string} = {
         type: 'start',
         prompt,
       }
       if (currentSessionId) payload.sessionId = currentSessionId
       if (model) payload.model = model
       if (conversationIdRef.current) payload.conversationId = conversationIdRef.current
       ws.send(JSON.stringify(payload))
     },
     [currentSessionId, resetBuffer],
   )
   ```

5. Add a `loadConversation` function that dispatches LOAD_MESSAGES and sets the conversationId ref:
   ```typescript
   const loadConversation = useCallback((messages: ChatMessage[], conversationId: string) => {
     dispatch({type: 'LOAD_MESSAGES', messages})
     conversationIdRef.current = conversationId
     resetBuffer()
   }, [resetBuffer])
   ```

6. Modify `clearMessages` to also clear the conversationId ref:
   ```typescript
   const clearMessages = useCallback(() => {
     dispatch({type: 'CLEAR'})
     resetBuffer()
     conversationIdRef.current = null
   }, [resetBuffer])
   ```

7. Export `loadConversation` and `conversationId` (as a getter) from the hook return:
   ```typescript
   return {
     messages,
     isConnected: connectionStatus === 'connected',
     isStreaming,
     connectionStatus,
     currentSessionId,
     conversationId: conversationIdRef.current,

     sendMessage,
     sendFollowUp,
     interrupt,
     cancel,
     clearMessages,
     loadConversation,
   }
   ```

Note: `conversationIdRef.current` in the return won't cause re-renders but will be current when read. This is intentional -- the parent component manages the authoritative conversationId via URL params.
