```typescript
// Accumulate stream deltas in a mutable ref, flush to state on animation frames
function useStreamAccumulator() {
  const bufferRef = useRef('')
  const [displayText, setDisplayText] = useState('')
  const rafRef = useRef<number>()

  const appendDelta = useCallback((text: string) => {
    bufferRef.current += text
    if (!rafRef.current) {
      rafRef.current = requestAnimationFrame(() => {
        setDisplayText(bufferRef.current)
        rafRef.current = undefined
      })
    }
  }, [])

  const flush = useCallback(() => {
    setDisplayText(bufferRef.current)
  }, [])

  return { displayText, appendDelta, flush }
}
```

**Exported hook: `useAgentSocket()`**

Returns:
```typescript
{
  // State
  messages: ChatMessage[]        // Processed message list for rendering
  isConnected: boolean           // WebSocket connection state
  isStreaming: boolean           // Agent is actively generating
  connectionStatus: 'connected' | 'disconnected' | 'reconnecting'
  currentSessionId: string | null

  // Actions
  sendMessage: (prompt: string, model?: string) => void   // Starts new query
  sendFollowUp: (text: string) => void                    // Mid-conversation injection
  interrupt: () => void                                    // Abort current generation
  cancel: () => void                                      // Cancel entire session
  clearMessages: () => void                                // Reset message list
}
```

**ChatMessage type** (exported):
```typescript
export interface ChatMessage {
  id: string
  role: 'user' | 'assistant' | 'system'
  content: string
  toolCalls?: Array<{
    id: string
    name: string
    input: Record<string, unknown>
    status: 'running' | 'complete' | 'error'
    output?: string
  }>
  isStreaming?: boolean  // True while this message is being streamed
  timestamp: number
}
```

**Implementation details:**

1. **WebSocket URL construction** -- reuse pattern from trpc.ts:
   ```typescript
   const { protocol, hostname, port } = location
   const portPart = port ? `:${port}` : ''
   const wsProtocol = protocol === 'https:' ? 'wss:' : 'ws:'
   const wsUrl = `${wsProtocol}//${hostname}${portPart}/ws/agent`
   ```
   Append `?token={jwt}` from `JWT_LOCAL_STORAGE_KEY` in localStorage.

2. **Connection management** with `useRef<WebSocket | null>`:
   - Connect on mount, cleanup on unmount
   - Exponential backoff reconnection: initial 1s, max 30s, multiply by 2 on each failure
   - Reset backoff to 1s on successful connection
   - Track `connectionStatus` state: 'connected', 'disconnected', 'reconnecting'
   - Set `isConnected` from `ws.readyState === WebSocket.OPEN`
   - Handle `ws.onopen`, `ws.onclose`, `ws.onerror`, `ws.onmessage`

3. **Stream accumulation** using the batched state update pattern (v20-ARCHITECTURE.md Pattern 3):
   - `bufferRef = useRef('')` for mutable text accumulation
   - `rafRef = useRef<number>()` for animation frame batching
   - On `content_block_delta` with `text_delta`: append to bufferRef, schedule RAF to flush to state
   - On `content_block_stop` / `message_stop`: flush immediately
   - On `result` message: finalize the current assistant message, set `isStreaming = false`

4. **Message processing** in `ws.onmessage`:
   Parse JSON as `AgentWsMessage`. Switch on `msg.type`:

   - `'session_ready'`: Store sessionId, set `isStreaming = false` (ready for input)
   - `'error'`: Add error message to messages array, set `isStreaming = false`
   - `'sdk_message'`: Switch on `msg.data.type`:
     - `'system'`: Log init, ignore for display
     - `'assistant'`: Process complete assistant turn -- extract text blocks and tool_use blocks from `msg.data.message.content`. For each tool_use block, track it in the current message's toolCalls array with status 'running'. This handles the case where the SDK sends complete turns (non-streaming mode fallback).
     - `'stream_event'`: The main streaming path. Switch on `msg.data.event`:
       - `content_block_start`: If `content_block.type === 'tool_use'`, add a new toolCall entry with status 'running'
       - `content_block_delta`: If `delta.type === 'text_delta'`, accumulate `delta.text` via appendDelta. If `delta.type === 'input_json_delta'`, accumulate tool input JSON.
       - `content_block_stop`: Mark current tool call as 'complete' if applicable
       - `message_start`: Create a new assistant message entry if not already streaming
       - `message_stop`: Flush buffer, finalize current message
     - `'result'`: Flush buffer, finalize current message, set `isStreaming = false`. If `msg.data.subtype === 'error'`, append error info.

5. **Action implementations:**
   - `sendMessage(prompt, model?)`: Add user message to messages array, set `isStreaming = true`, send `{ type: 'start', prompt, model }` via ws.send()
   - `sendFollowUp(text)`: Add user message, send `{ type: 'message', text }`
   - `interrupt()`: Send `{ type: 'interrupt' }`, set `isStreaming = false`
   - `cancel()`: Send `{ type: 'cancel' }`, set `isStreaming = false`
   - `clearMessages()`: Reset messages to empty array

6. **State management** via `useReducer` for the messages array (avoids stale closure issues):
   - Actions: ADD_USER_MESSAGE, START_ASSISTANT_MESSAGE, UPDATE_STREAMING_CONTENT, ADD_TOOL_CALL, UPDATE_TOOL_CALL, FINALIZE_MESSAGE, ADD_ERROR, CLEAR

**Critical implementation notes:**
- Do NOT store raw SDKMessage objects in state (Anti-Pattern 4 from v20-ARCHITECTURE.md). Process stream events into ChatMessage format.
- Use `useRef` for the WebSocket instance, buffer, and reconnect timer to avoid stale closures in callbacks.
- The `sendMessage` function should check `ws.readyState === WebSocket.OPEN` before sending.
- On unmount (`useEffect` cleanup), close the WebSocket with code 1000 (normal closure) and clear reconnect timer.
