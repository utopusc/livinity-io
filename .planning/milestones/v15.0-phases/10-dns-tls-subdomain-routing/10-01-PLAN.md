**server.ts** -- Add ask endpoint for Caddy on-demand TLS validation:

In the `createRequestHandler` function, BEFORE the subdomain parsing logic, add a check:

```
if (req.url?.startsWith('/internal/ask')) {
  // Caddy on-demand TLS ask endpoint
  // Query: GET /internal/ask?domain={hostname}
  handleAskRequest(req, res, pool);
  return;
}
```

The `handleAskRequest` function (can be in server.ts or a new file):
1. Parse `domain` from query string: `new URL(req.url, 'http://localhost').searchParams.get('domain')`
2. Parse subdomain from the domain using `parseSubdomain(domain)`
3. If no username extracted → respond 404 (bare domain, no cert needed)
4. Query PostgreSQL: `SELECT 1 FROM users WHERE username = $1 LIMIT 1`
5. If user exists → respond 200 with `{ "allowed": true }`
6. If user doesn't exist → respond 404 with `{ "allowed": false }`

**IMPORTANT:** The ask endpoint needs access to the PostgreSQL pool. The `createRequestHandler` currently takes `(registry, redis)`. Add `pool: pg.Pool` as a third parameter. Update the call site in index.ts accordingly.

Also add a check: if the request path is `/internal/ask` but the request is NOT from localhost (check req.socket.remoteAddress), return 403. This prevents external abuse of the ask endpoint.

**Caddyfile** -- Create at `platform/relay/Caddyfile`:

```
{
    on_demand_tls {
        ask http://localhost:4000/internal/ask
    }
    servers {
        stream_close_delay 5m
    }
}

*.livinity.io, livinity.io {
    tls {
        on_demand
    }
    reverse_proxy localhost:4000 {
        header_up X-Forwarded-Proto {scheme}
        header_up X-Real-IP {remote_host}
    }
}
```

Notes:
- The single site block handles all subdomains AND the bare domain
- `on_demand` in the tls block tells Caddy to use on-demand TLS
- `reverse_proxy` handles both HTTP and WebSocket automatically
- Caddy auto-redirects HTTP (port 80) → HTTPS (port 443)
- `stream_close_delay 5m` prevents WS drops on config reload
