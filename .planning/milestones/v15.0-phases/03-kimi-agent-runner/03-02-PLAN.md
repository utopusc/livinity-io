Create `nexus/packages/core/src/kimi-agent-runner.ts` that mirrors SdkAgentRunner's public interface (extends EventEmitter, constructor takes AgentConfig, has `run(task): Promise<AgentResult>`) but uses Kimi CLI print mode internally.

**Architecture overview:**
```
KimiAgentRunner.run(task)
  |-> writes temp agent YAML + system prompt .md files
  |-> builds --mcp-config JSON from ToolRegistry + Chrome DevTools
  |-> spawns: kimi --print --output-format=stream-json --yolo --agent-file /tmp/nexus-agent-{id}.yaml --mcp-config '{...}' --max-steps-per-turn N --work-dir /opt/livos -p "task"
  |-> reads stdout line-by-line (JSONL)
  |-> parses each JSON line and maps to AgentEvent emissions
  |-> waits for process exit
  |-> cleans up temp files in finally block
  |-> returns AgentResult
```

**5. Helper: writeAgentFiles:**
Write temporary agent YAML and system prompt markdown files for each session. The Kimi CLI expects an agent YAML file with `system_prompt_path` pointing to a markdown file.

```typescript
const AGENT_TEMP_DIR = '/tmp/nexus-agents';

async function writeAgentFiles(sessionId: string, systemPrompt: string): Promise<{ yamlPath: string; mdPath: string }> {
  await mkdir(AGENT_TEMP_DIR, { recursive: true });

  const mdPath = join(AGENT_TEMP_DIR, `${sessionId}.md`);
  const yamlPath = join(AGENT_TEMP_DIR, `${sessionId}.yaml`);

  // Write system prompt markdown
  await writeFile(mdPath, systemPrompt, 'utf-8');

  // Write agent YAML pointing to the markdown file
  const yaml = `name: nexus-agent
system_prompt_path: ${mdPath}
`;
  await writeFile(yamlPath, yaml, 'utf-8');

  return { yamlPath, mdPath };
}
```

**6. Helper: cleanupAgentFiles:**
```typescript
async function cleanupAgentFiles(yamlPath: string, mdPath: string): Promise<void> {
  await unlink(yamlPath).catch(() => {});
  await unlink(mdPath).catch(() => {});
}
```

**8. The `run()` method -- full implementation:**

The run method should:

a) **Setup** -- extract config values (maxTurns, tier, nexusConfig), build system prompt (reuse the same default from SdkAgentRunner or use `config.systemPromptOverride`), prepend contextPrefix to task.

b) **Write agent files** -- call `writeAgentFiles()` with the system prompt.

c) **Build MCP config** -- call `buildMcpConfig()`, check CDP reachability for Chrome DevTools (same pattern as SdkAgentRunner lines 180-197), serialize to JSON string.

d) **Spawn kimi subprocess** -- build the args array:
```typescript
const args = [
  '--print',
  '--output-format=stream-json',
  '--yolo',
  '--agent-file', yamlPath,
  '--model', tierToModel(tier),
  '--max-steps-per-turn', String(maxTurns),
  '--work-dir', '/opt/livos',
  '--mcp-config', JSON.stringify(mcpConfigObj),
  '-p', taskWithContext,
];
const child = spawn('kimi', args, {
  stdio: ['pipe', 'pipe', 'pipe'],
  env: { ...process.env },
});
```

e) **Parse JSONL output** -- use readline on child.stdout:
```typescript
const rl = createInterface({ input: child.stdout });
```

Each line is a JSON object. Map according to the print mode event mapping from ARCHITECTURE.md:

- `{role: "assistant", content: "text"}` -> emit `{ type: 'chunk', turn, data: content }` and accumulate as `answer`
- `{role: "assistant", content: "text", tool_calls: [...]}` -> emit `{ type: 'tool_call', turn, data: { tool: tc.function.name, params: JSON.parse(tc.function.arguments) } }` for each tool call
- `{role: "tool", tool_call_id, content}` -> emit `{ type: 'observation', turn, data: { tool: tool_call_id, output: content } }`
- Messages with `usage` field -> extract `input_tokens` and `output_tokens` (or Kimi's equivalents: `input_other`, `output`, `input_cache_read`)

Track turn count by counting assistant messages that have tool_calls (each tool call round is a turn). The final assistant message without tool_calls is the final answer.

For TTFB: record `Date.now() - runStartTime` on the first content chunk.

f) **Handle process exit** -- wait for the child process to exit:
```typescript
const exitCode = await new Promise<number>((resolve) => {
  child.on('close', (code) => resolve(code ?? 1));
});
```

Also capture stderr for error reporting:
```typescript
let stderrBuffer = '';
child.stderr.on('data', (chunk: Buffer) => {
  stderrBuffer += chunk.toString();
});
```

g) **Build and return AgentResult** -- construct the result matching the AgentResult interface:
```typescript
return {
  success: exitCode === 0,
  answer: answer || stderrBuffer || 'Task completed (no response).',
  turns,
  totalInputTokens,
  totalOutputTokens,
  toolCalls,
  stoppedReason: exitCode === 0 ? 'complete' : 'error',
  ttfbMs,
  toolCallCount,
  durationMs: Date.now() - runStartTime,
};
```

h) **Cleanup** -- in a `finally` block, clean up temp agent files:
```typescript
finally {
  await cleanupAgentFiles(yamlPath, mdPath);
}
```

**9. Error handling:**

- If kimi is not found (ENOENT), emit error event and return AgentResult with `success: false, stoppedReason: 'error'`.
- If JSON.parse fails on a JSONL line, log the error and skip that line (don't crash the whole run).
- If the process times out, kill the child process and return with `stoppedReason: 'timeout'`.
- Implement timeout using `setTimeout` that calls `child.kill('SIGTERM')` after `config.timeoutMs`.

**10. onAction callback:**

Mirror SdkAgentRunner's pattern for the `onAction` callback (used by Daemon for WhatsApp/Telegram live updates):
- On text chunks: call `onAction({ type: 'thinking', thought: text, turn })`
- On final answer: call `onAction({ type: 'final_answer', turn, answer })`

**11. Token budget enforcement:**

Same as SdkAgentRunner: estimate tokens from content length (~4 chars per token). If `maxTokenBudget > 0` and estimated exceeds budget, kill the child process and set `stoppedReason: 'max_tokens'`.

IMPORTANT THINGS TO AVOID:
- Do NOT import or use `@moonshot-ai/kimi-agent-sdk` -- we are using print mode (child_process), not the SDK. The SDK is v0.1.5 and unstable (see PITFALLS.md P1).
- Do NOT try to create an in-process MCP server like SdkAgentRunner does with `createSdkMcpServer()` -- Kimi CLI does not support inline MCP servers. Use `--mcp-config` pointing to the existing `nexus-mcp` HTTP server on port 3100.
- Do NOT hardcode the system prompt in the class -- read from `config.systemPromptOverride` or use the same default prompt string as SdkAgentRunner.
- Do NOT write agent files to a location without ensuring the directory exists first (use `mkdir -p` pattern).
- Do NOT forget the `finally` block for cleanup -- temp files must be cleaned up even on error.
