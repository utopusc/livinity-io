Create an `OverviewTab` component in server-control/index.tsx and make it the default tab.

**OverviewTab component design:**

The component uses existing hooks for data. Important: the system resource hooks (useCpuForUi, useSystemMemoryForUi, useSystemDiskForUi) and chart data state are already called in the parent ServerControl component. To avoid duplicate API calls, OverviewTab should receive the existing data as props OR move those hooks into the OverviewTab and pass down to ResourceCards. The cleanest approach: OverviewTab calls its own hooks (the tRPC query caching in react-query means the same endpoint polled at the same interval will share the cache -- no duplicate network calls).

OverviewTab calls:
- `useCpuForUi({poll: true})` for CPU
- `useSystemMemoryForUi({poll: true})` for RAM
- `useSystemDiskForUi({poll: true})` for Disk
- `useCpuTemperature()` for temperature
- `useContainers()` for container summary (runningCount, totalCount)
- `usePM2()` for PM2 summary (onlineCount, totalCount)
- `useNetworkStats()` from use-monitoring.ts for network throughput

Layout (CSS grid, 3-column on large screens):

**Row 1: System Health Cards** (grid-cols-4 on lg, grid-cols-2 on sm)
Four mini health cards in a row:
- CPU: percentage value, mini sparkline (30-point AreaChart, height 40px), progress bar
- RAM: used/total, mini sparkline, progress bar
- Disk: used/total, progress bar (no sparkline -- disk doesn't change rapidly)
- Temp: temperature value with warning color indicator

Each card is a compact version of the existing ResourceCard. Reuse the existing ResourceCard component directly -- it already has the chart/progress layout.

**Row 2: Summary Cards** (grid-cols-3 on lg)
Three summary cards side by side:

Card 1 -- Docker Containers:
- Icon: IconBrandDocker
- Big number: `{runningCount} / {totalCount}`
- Label: "containers running"
- Status dot: green if all running, amber if some stopped, red if errors
- Not a button/link -- just informational (users can click the Containers tab)

Card 2 -- PM2 Processes:
- Icon: IconActivity (or IconServer)
- Big number: `{onlineCount} / {totalCount}`
- Label: "processes online"
- Status dot: green if all online, amber if some stopped

Card 3 -- Network Throughput:
- Icon: IconArrowsUpDown (or IconArrowUp + IconArrowDown stacked)
- Two lines: `{rxSpeed} in` and `{txSpeed} out`
- Use formatSpeed helper from MonitoringTab (Plan 01) or inline: convert bytes/sec to KB/s or MB/s
- Show "Calculating..." if data is null (first call)

**Styling:**
- Cards use `rounded-xl border border-border-default bg-surface-base p-4` (consistent with existing container table card style)
- Summary card big numbers: `text-2xl font-semibold text-text-primary`
- Labels: `text-xs text-text-secondary uppercase tracking-wider`
- Status dots: 8px circles with `bg-emerald-500` / `bg-amber-500` / `bg-red-500`
- Gap between rows: `gap-4`
- Padding: `p-4` for section spacing

**Tab reordering:**
1. Change `<Tabs defaultValue='containers'>` to `<Tabs defaultValue='overview'>`
2. Add Overview as the first TabsTrigger: `<TabsTrigger value='overview'>Overview</TabsTrigger>`
3. Add OverviewTab's TabsContent BEFORE the containers TabsContent:
```tsx
<TabsContent value='overview' className='flex-1 overflow-auto'>
  <OverviewTab />
</TabsContent>
```
4. Tab order: Overview, Containers, Images, Volumes, Networks, PM2, Monitoring

**Chart data for OverviewTab sparklines:**
OverviewTab needs its own cpuChartData and memoryChartData state arrays (30-point ring buffers). Use the same useState + useEffect pattern already in ServerControl. Since OverviewTab is a separate component, the chart data is isolated. The parent ServerControl's chart data is still used by the ResourceCard section above the tabs (which remains unchanged).

NOTE: The existing ResourceCard section at the top of ServerControl (the 3 cards above the tabs) should remain as-is. The OverviewTab provides a more detailed view inside the tab area. Do NOT remove the top resource cards.
