<objective>
Fix the v29.4 streaming regression at the deploy layer. Root-cause hypothesis from Phase 49 fixture: 1m 2s deploy duration was suspiciously short (vite alone is ~30-60s); UI build was either a phantom no-op (vite cache hit OR silent skip) or output was discarded. This produced a stale UI bundle in /opt/livos/livos/packages/ui/dist that the PWA service worker continued serving. New streaming code never reached the browser.

Defensive fix at the deploy layer with two changes to `update.sh`:
  1. `rm -rf dist` before `npm run build` — forces vite to regenerate from source. Eliminates phantom no-op builds.
  2. Move `verify_build` from BEFORE to AFTER `npm run build` — matches the function's documented "Call AFTER every build invocation" contract. The pre-build call was a no-op on existing installs and a hard-block on fresh installs.

Combined effect: every deploy starts UI with a clean dist, vite produces a new precache manifest hash, the PWA's existing `registerType: 'autoUpdate'` config picks up the new SW automatically and cleans up old caches. Browser auto-refreshes to fresh bundle on next visit.

NOT in this phase: sacred file edit for FR-MODEL-02 Branch N reversal (deferred per D-51-03 — needs live Mini PC verification which is currently blocked by fail2ban).
