1. Install vite-plugin-pwa as a dev dependency:

   ```bash
   cd livos/packages/ui && pnpm add -D vite-plugin-pwa
   ```

2. Delete `livos/packages/ui/public/site.webmanifest` -- it will be replaced by the vite-plugin-pwa generated manifest. The old manifest has wrong theme_color (#000000) and is missing start_url, scope, id.

3. Update `livos/packages/ui/vite.config.ts`:
   - Add import: `import { VitePWA } from 'vite-plugin-pwa'`
   - Add VitePWA to the plugins array AFTER react() and imagetools():

```ts
VitePWA({
  registerType: 'autoUpdate',
  includeAssets: ['favicon/favicon.ico', 'favicon/apple-touch-icon.png'],
  manifest: {
    name: 'Livinity',
    short_name: 'Livinity',
    description: 'Self-hosted AI server platform',
    id: '/',
    start_url: '/',
    scope: '/',
    theme_color: '#f8f9fc',
    background_color: '#f8f9fc',
    display: 'standalone',
    orientation: 'any',
    icons: [
      {
        src: '/favicon/android-chrome-192x192.png',
        sizes: '192x192',
        type: 'image/png',
      },
      {
        src: '/favicon/android-chrome-512x512.png',
        sizes: '512x512',
        type: 'image/png',
      },
      {
        src: '/favicon/android-chrome-512x512.png',
        sizes: '512x512',
        type: 'image/png',
        purpose: 'maskable',
      },
    ],
  },
  workbox: {
    globPatterns: ['**/*.{js,css,html,woff2}'],
    navigateFallback: '/index.html',
    navigateFallbackDenylist: [/^\/trpc/, /^\/api/, /^\/ws/],
    runtimeCaching: [
      {
        urlPattern: /\/wallpapers\/.*/,
        handler: 'CacheFirst',
        options: {
          cacheName: 'wallpapers',
          expiration: { maxEntries: 30, maxAgeSeconds: 30 * 24 * 60 * 60 },
        },
      },
      {
        urlPattern: /\/figma-exports\/.*/,
        handler: 'CacheFirst',
        options: {
          cacheName: 'app-icons',
          expiration: { maxEntries: 50, maxAgeSeconds: 30 * 24 * 60 * 60 },
        },
      },
    ],
  },
})
```

Key decisions:
- `registerType: 'autoUpdate'` -- auto-updates without user prompt (Livinity is a dashboard, not content app)
- `navigateFallbackDenylist` includes `/trpc`, `/api`, and `/ws` -- these MUST hit the network, never serve cached HTML
- `globPatterns` includes woff2 for fonts but NOT jpg/png (wallpapers cached via runtimeCaching instead)
- `theme_color: '#f8f9fc'` matches the existing meta tag and body background (fixes the old manifest's #000000)
- Maskable icon reuses 512x512 Android Chrome icon (existing asset)
- `id: '/'` ensures PWA identity consistency across installs

4. Add the `/dev-dist` directory to `.gitignore` if not already present (vite-plugin-pwa generates dev SW files there during dev mode). Add this line: `dev-dist`

    - livos/packages/ui/index.html
  </read_first>
  <action>
Update `livos/packages/ui/index.html` with these specific changes:

1. **Modify the viewport meta tag** -- add `viewport-fit=cover` to enable edge-to-edge rendering on notched iOS devices. Change:
   ```html
   <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1" />
   ```
   To:
   ```html
   <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, viewport-fit=cover" />
   ```

2. **Add Apple PWA meta tags** immediately after the existing `<meta name="theme-color">` line:
   ```html
   <!-- PWA: iOS standalone mode -->
   <meta name="apple-mobile-web-app-capable" content="yes" />
   <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />
   <meta name="apple-mobile-web-app-title" content="Livinity" />
   ```

3. **Remove the old manifest link** -- vite-plugin-pwa injects its own manifest link at build time. Remove this line:
   ```html
   <link rel="manifest" href="/site.webmanifest" />
   ```

The final `<head>` should contain (in order):
- charset meta
- viewport meta (with viewport-fit=cover)
- theme-color meta
- Apple PWA meta tags (3 new lines)
- robots, referrer metas
- apple-touch-icon link (keep existing)
- favicon links (keep existing)
- title

Do NOT add or remove any other tags. The apple-touch-icon link already exists and is correct (`/favicon/apple-touch-icon.png` at 180x180).
