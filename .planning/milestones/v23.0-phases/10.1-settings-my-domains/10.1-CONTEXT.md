# Phase 10.1: Settings My Domains ‚Äî Replace Domain & HTTPS with tunnel-synced domains

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove the old "Domain & HTTPS" section from Settings (manual domain setup, DNS verify, activate flow). Replace with "My Domains" section that shows domains synced from livinity.io via tunnel. Users manage domains on livinity.io dashboard, domains appear here automatically. Each domain can be mapped to a Caddy reverse proxy target.

</domain>

<decisions>
## Implementation Decisions

### What to Remove
- Settings menu item `{id: 'domain', label: 'Domain & HTTPS'}` ‚Äî replace with `{id: 'my-domains', label: 'My Domains'}`
- `DomainSection` component that renders `DomainSetupDialogContent` (the old wizard)
- Do NOT delete `domain/routes.ts` or `domain/caddy.ts` ‚Äî Caddy manager is still needed for generating Caddyfile

### What to Build
- New `MyDomainsSection` component in Settings
- Reads domains from Redis keys `livos:custom_domain:*` (populated by tunnel sync from Phase 09)
- Each domain card shows: domain name, status (synced/active/error), mapped app, Caddy config status
- "Configure" button per domain: opens dialog to map domain to an app (Docker container) or custom port
- When user maps a domain ‚Üí call existing `caddy.ts` `applyCaddyConfig()` to update Caddyfile with the custom domain block
- Domain list auto-refreshes (polling every 10s)

### Data Source
- Domains come from Redis `livos:custom_domain:{hostname}` keys (JSON with domain, status, userId, appMapping)
- These are populated by `tunnel-client.ts` `handleDomainSync` and `handleDomainListSync` (already built in Phase 09, reverted from production but code exists in git)
- Also stored in PostgreSQL `custom_domains` table (if available)
- For domains to appear, user must have added them on livinity.io dashboard first

### Caddy Integration
- When user maps domain to app, generate Caddy block: `mysite.com { reverse_proxy 127.0.0.1:{port} }`
- Use existing `generateFullCaddyfile()` from `caddy.ts` ‚Äî extend it to include custom domain blocks
- Caddy auto-provisions SSL via Let's Encrypt for custom domains

### tRPC Routes Needed
- `domain.myDomains.list` ‚Äî read all custom domains from Redis
- `domain.myDomains.configure` ‚Äî map domain to app/port, update Caddy
- `domain.myDomains.remove` ‚Äî remove domain mapping (domain stays synced, just unmapped from Caddy)

### Claude's Discretion
- UI component structure and styling
- Error state handling
- Empty state when no domains synced

</decisions>
