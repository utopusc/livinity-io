## 5. End-to-End Flows

**Status: COMPLETE.** Every step has verified evidence. No breaks.

### Flow 2: cmd+k -> result -> detail

5. **Detail panel renders** in target section with selection pre-applied

**Status: COMPLETE.** StatusBar Search button uses identical flow (wires to same usePaletteStore.openPalette).

### Flow 3: Add stack -> Git deploy -> webhook -> Schedules sync

Steps traced:

1. **User opens Stacks section** -> stacks/stack-section.tsx (441 lines) renders list via useStacks()

**Status: COMPLETE.** Full v27.0 GitOps + Scheduler infra is correctly surfaced through v28.0 UI restructure with no regression.
