**auth.ts** -- API key verification with Redis cache:
- Export `verifyApiKey(apiKey: string, pool: pg.Pool, redis: Redis): Promise<{ userId: string; username: string } | null>`
- Extract prefix from API key: `apiKey.substring(0, 14)` (e.g., "liv_k_testke" from "liv_k_testkey12345678")
- Check Redis cache first: `auth:cache:{prefix}` -- if exists, return cached user info (TTL from config.AUTH_CACHE_TTL_S)
- If not cached: query `api_keys JOIN users` by prefix, bcrypt.compare the full key against key_hash
- On success: cache in Redis with TTL, update `last_used_at` on the api_key row, return { userId, username }
- On failure: return null
- Use `bcryptjs` (NOT `bcrypt` -- the native module has build issues)

**subdomain-parser.ts** -- Extract username from Host header:
- Export `parseSubdomain(host: string | undefined): { username: string | null; appName: string | null }`
- Strip port from host if present
- Parse against config.RELAY_HOST:
  - `alice.livinity.io` -> { username: 'alice', appName: null }
  - `immich.alice.livinity.io` -> { username: 'alice', appName: 'immich' }
  - `livinity.io` or bare IP -> { username: null, appName: null }
- Handle edge case: `www.livinity.io` -> { username: 'www', appName: null } (this is fine, www won't be a registered user)

**health.ts** -- Simple health endpoint:
- Export `handleHealthRequest(res: http.ServerResponse, registry: TunnelRegistry): void`
- Return JSON: `{ status: 'ok', connections: registry.size, memory: process.memoryUsage(), uptime: process.uptime(), version: '0.1.0' }`
- Set Content-Type: application/json, status 200

**tunnel-registry.ts** -- TunnelConnection class and registry Map:

`TunnelConnection` class:
- Constructor takes: `{ username, userId, ws, sessionId }`
- Properties: username, userId, ws (WebSocket), sessionId, connectedAt (Date.now())
- Private Maps: `pendingRequests: Map<string, PendingRequest>` and `activeBrowserSockets: Map<string, WebSocket>` (for WS proxying, used in Plan 04)
- Start heartbeat in constructor: every config.HEARTBEAT_INTERVAL_MS, check `alive` flag. If not alive, call `destroy('ping timeout')`. If alive, set alive=false and send `ws.ping()`. On pong received, set alive=true.
- `destroy(reason?: string)` method:
  1. Clear heartbeat interval
  2. Fail all pending requests (clear timeout, write 502 "Tunnel disconnected" if headers not sent)
  3. Close all active browser WebSockets (code 1001)
  4. Clear both Maps
  5. Close tunnel WebSocket (code 1000)
  6. Remove all listeners from ws
- `addPendingRequest(id, res, timeout)` / `removePendingRequest(id)` / `getPendingRequest(id)` methods
- `addBrowserSocket(id, ws)` / `removeBrowserSocket(id)` / `getBrowserSocket(id)` methods
- Getter: `pendingCount` returns pendingRequests.size

`TunnelRegistry` class:
- Private Map: `tunnels: Map<string, TunnelConnection>`
- `register(username, connection)` -- if existing tunnel for username, destroy the old one first, then set new
- `unregister(username)` -- destroy and delete
- `get(username): TunnelConnection | undefined`
- `has(username): boolean`
- Getter: `size` returns tunnels.size
- `getBySessionId(sessionId): TunnelConnection | undefined` -- iterates to find by sessionId (small Map, fine for v8.0)

Export both classes.

**request-proxy.ts** -- HTTP request serialization and response handling:
- Export `proxyHttpRequest(tunnel: TunnelConnection, req: http.IncomingMessage, res: http.ServerResponse): void`
  1. Generate request ID with `nanoid()`
  2. Collect request body chunks into Buffer array
  3. On `req.end`, build `TunnelRequest` message: type 'http_request', id, method, path (req.url), headers (req.headers cast to Record), body (base64 or null), targetApp (from subdomain context -- pass as parameter or read from request)
  4. Set up timeout: setTimeout for config.REQUEST_TIMEOUT_MS, on expiry delete pending request and write 504 Gateway Timeout
  5. Add pending request to tunnel.addPendingRequest(id, res, timeout)
  6. Send JSON.stringify(message) through tunnel.ws
  7. Handle req.on('error') -- clean up pending request

- Export `handleTunnelResponse(tunnel: TunnelConnection, msg: TunnelResponse): void`
  1. Get pending request by msg.id from tunnel
  2. If not found, return silently (request already timed out)
  3. Clear timeout, remove from pending
  4. Decode body from base64 if present
  5. Write response: res.writeHead(msg.status, msg.headers), res.end(bodyBuffer)
  6. Handle multi-value headers (especially Set-Cookie): if msg.headers value is an array, pass through directly to writeHead

**server.ts** -- HTTP request handler:
- Export `createRequestHandler(registry: TunnelRegistry, redis: Redis): (req: http.IncomingMessage, res: http.ServerResponse) => void`
- Returns a function that:
  1. Parse host with parseSubdomain
  2. If path is `/health` and no subdomain -> handleHealthRequest(res, registry)
  3. If no username -> return 404 "Not Found"
  4. Get tunnel from registry by username
  5. If no tunnel or ws not OPEN -> return 503 with "Server offline" text (Plan 04 adds branded offline page here)
  6. Call proxyHttpRequest(tunnel, req, res)

**index.ts** -- Entry point that wires everything together:
- Import all modules
- Create `pg.Pool` from config.DATABASE_URL
- Create `Redis` from config.REDIS_URL (named import: `import { Redis } from 'ioredis'`)
- Apply schema: read schema.sql (use `fs.readFileSync` with `import.meta.url` resolution), execute against pool
- Create TunnelRegistry instance
- Create HTTP server with createRequestHandler
- Create WebSocketServer in noServer mode (for tunnel connections)
- Handle `server.on('upgrade')`:
  1. If req.url starts with `/tunnel/connect` -> `tunnelWss.handleUpgrade(req, socket, head, onTunnelConnect)`
  2. Otherwise -> for user subdomains, this will be handled in Plan 04 (WS proxy). For now, destroy the socket.
- `onTunnelConnect(ws)` function:
  1. Set auth timeout: if no message within config.AUTH_TIMEOUT_MS, close with 4001 "Auth timeout"
  2. On first message: parse JSON as TunnelAuth
  3. Call verifyApiKey(msg.apiKey, pool, redis)
  4. If invalid: send TunnelAuthError message, close with 4002 "Invalid API key"
  5. If valid: generate sessionId with `nanoid()`, create TunnelConnection, register in registry
  6. Send TunnelConnected message { type: 'connected', sessionId, assignedUrl: `${username}.${config.RELAY_HOST}` }
  7. Set up message handler on ws: parse JSON, switch on type:
     - 'http_response' -> handleTunnelResponse(tunnel, msg)
     - 'pong' -> (handled by TunnelConnection heartbeat internally via ws pong events)
     - 'ws_ready', 'ws_error', 'ws_frame', 'ws_close' -> no-op for now (Plan 04)
  8. On ws close/error: registry.unregister(username), log
- Set maxPayload on WebSocketServer: config.MAX_PAYLOAD_BYTES
- Listen on config.RELAY_PORT
- Log: `Relay server listening on port ${config.RELAY_PORT}`
- Handle SIGTERM/SIGINT: send TunnelRelayShutdown to all connected tunnels, close server, close pool, disconnect redis, exit

**Important implementation notes:**
- Use `import { nanoid } from 'nanoid'` (ESM named export)
- Use `import { Redis } from 'ioredis'` (named export, NOT default)
- Use `import pg from 'pg'` then `const pool = new pg.Pool(...)` (pg uses default export in ESM)
- WebSocket close codes: use 4001-4009 for application-specific errors (4001=auth timeout, 4002=invalid key, 4003=quota exceeded)
- The `handleUpgrade` pattern follows ws library's noServer mode exactly
