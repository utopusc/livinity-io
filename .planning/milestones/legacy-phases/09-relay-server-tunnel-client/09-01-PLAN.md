**protocol.ts** -- Define all 14 tunnel message types as TypeScript interfaces plus union types. Follow the RESEARCH.md specification exactly:

Relay-to-Client messages:
- `TunnelRequest` (type: 'http_request') -- id, method, path, headers (Record string to string|string[]), body (string|null), targetApp? (string)
- `TunnelWsUpgrade` (type: 'ws_upgrade') -- id, path, headers, targetApp?
- `TunnelPing` (type: 'ping') -- ts (number)
- `TunnelRelayShutdown` (type: 'relay_shutdown')
- `TunnelConnected` (type: 'connected') -- sessionId, assignedUrl
- `TunnelAuthError` (type: 'auth_error') -- error (string)
- `TunnelQuotaExceeded` (type: 'quota_exceeded') -- usedBytes, limitBytes, resetsAt (ISO string)

Client-to-Relay messages:
- `TunnelAuth` (type: 'auth') -- apiKey, sessionId? (optional, for reconnection)
- `TunnelResponse` (type: 'http_response') -- id, status, headers, body
- `TunnelWsReady` (type: 'ws_ready') -- id
- `TunnelWsError` (type: 'ws_error') -- id, error
- `TunnelPong` (type: 'pong') -- ts

Bidirectional messages:
- `TunnelWsFrame` (type: 'ws_frame') -- id, data (base64), binary (boolean)
- `TunnelWsClose` (type: 'ws_close') -- id, code?, reason?

Union types:
- `RelayToClientMessage` = TunnelRequest | TunnelWsUpgrade | TunnelPing | TunnelRelayShutdown | TunnelConnected | TunnelAuthError | TunnelQuotaExceeded
- `ClientToRelayMessage` = TunnelAuth | TunnelResponse | TunnelWsReady | TunnelWsError | TunnelPong
- `BidirectionalMessage` = TunnelWsFrame | TunnelWsClose
- `TunnelMessage` = RelayToClientMessage | ClientToRelayMessage | BidirectionalMessage

Export ALL interfaces and union types.
