  <action>
Create `nexus/packages/core/src/learning-engine.ts` with:

1. **LearningEngine class** accepting `{ redis: Redis }` in constructor.

2. **logToolCall(entry)** method — appends to Redis stream `nexus:tool_calls` using XADD with approximate trimming:
   ```
   XADD nexus:tool_calls MAXLEN ~10000 * tool {name} success {0|1} duration {ms} session {sessionId} type {capabilityType} timestamp {Date.now()}
   ```
   - Entry type: `{ tool: string; success: boolean; duration_ms: number; session_id: string; type?: string }`
   - Fire-and-forget (catch + log errors, don't throw)

3. **getCoOccurrences()** method — reads recent stream entries via XRANGE (last 5000), groups by session_id, builds co-occurrence matrix:
   - Returns `Array<{ toolA: string; toolB: string; count: number }>` sorted by count descending, top 20
   - Simple approach: for each session, for each pair of distinct tools used in that session, increment count
   - Cache result in a class-level variable with a 5-minute TTL to avoid re-scanning

4. **getSuggestions(currentToolNames: string[], allCapabilities: CapabilityManifest[])** method:
   - Calls getCoOccurrences() internally
   - For each tool in currentToolNames, find co-occurring tools not already in currentToolNames
   - Map those tool names back to capabilities via provides_tools matching
   - Returns `Array<{ capability: CapabilityManifest; reason: string; confidence: number }>` with confidence 0.25, max 3 suggestions
   - Reason format: `"commonly used with {toolName}"`

5. **getToolStats()** method — reads stream entries, aggregates per-tool: total calls, success count, avg duration. Returns `Array<{ tool: string; totalCalls: number; successRate: number; avgDuration: number }>`. Uses same cache as getCoOccurrences (refresh together).

Add `export { LearningEngine } from './learning-engine.js';` to `nexus/packages/core/src/lib.ts`.

   if (this.deps.learningEngine) {
     try {
       const selectedToolNames = this.getToolNamesFromCapabilities(selected);
