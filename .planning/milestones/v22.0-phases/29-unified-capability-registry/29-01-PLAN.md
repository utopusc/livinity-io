**4. Core methods:**

`async syncAll(): Promise<void>` — The main sync method. Calls four private sync methods in sequence, then persists all manifests to Redis. Steps:
  1. Clear the in-memory cache
  2. Call `this.syncTools()` — iterates `this.deps.toolRegistry.listAll()`, for each Tool creates a CapabilityManifest with:
     - `id: 'tool:' + tool.name`
     - `type: 'tool'`
     - `name: tool.name`
     - `description: tool.description`
     - `semantic_tags: []` (tools don't have tags)
     - `triggers: []`
     - `provides_tools: [tool.name]`
     - `requires: []`, `conflicts: []`
     - `context_cost`: Estimate by counting characters in `tool.description` + JSON.stringify of parameters, divide by 4 (rough tokens)
     - `tier: 'any'`
     - `source: tool.name.startsWith('mcp__') ? 'builtin' : 'system'` — Skip tools starting with `mcp__` (they'll be covered under MCP entries)
     - `status: 'active'`
     - `last_used_at: 0`, `registered_at: Date.now()`
     - Store in cache: `this.cache.set(manifest.id, manifest)`
  3. Call `this.syncSkills()` — iterates `this.deps.skillLoader.listSkills()`, for each skill:
     - `id: 'skill:' + skill.name`
     - `type: 'skill'`
     - `semantic_tags`: use skill manifest tags if available (access via getSkill), else `[]`
     - `triggers: skill.triggers`
     - `provides_tools: []` (skills consume tools, don't provide them)
     - `context_cost: 0` (skills don't add tool definitions to context)
     - `tier: 'any'` (skills have their own tier internally)
     - `source: skill.source || 'builtin'`
     - `status: 'active'`
  4. Call `this.syncMcps()` — gets configs from `this.deps.mcpConfigManager.getConfig()` and statuses from `this.deps.mcpClientManager.getAllStatuses()`. For each MCP server config:
     - `id: 'mcp:' + serverName`
     - `type: 'mcp'`
     - `description: config.description || serverName`
     - `semantic_tags: []`
     - `triggers: []`
     - `provides_tools`: from status.tools (the actual tool names registered)
     - `context_cost`: Sum the context cost of each provided tool (look up from cache by `tool:{toolName}`)
     - `source: config.installedFrom ? 'marketplace' : 'builtin'`
     - `status: status.running ? 'active' : (status.lastError ? 'error' : 'inactive')`
     - `last_error: status.lastError`
     - `metadata: { transport: config.transport, command: config.command, url: config.url }`
  5. Call `this.syncAgents()` — gets agent list from `this.deps.subagentManager.list()`. For each agent:
     - `id: 'agent:' + agent.id`
     - `type: 'agent'`
     - `name: agent.name`
     - `description: agent.description || ''`
     - `provides_tools: []`
     - `tier: (agent.tier as any) || 'any'`
     - `source: 'custom'`
     - `status: agent.status === 'active' ? 'active' : 'inactive'`
     - `last_used_at: agent.lastRunAt || 0`
     - `metadata: { schedule: agent.schedule, runCount: agent.runCount }`
  6. After all syncs, persist every manifest to Redis:
     ```typescript
     const pipeline = this.deps.redis.pipeline();
     for (const [id, manifest] of this.cache) {
       const [type, ...nameParts] = id.split(':');
       const name = nameParts.join(':');
       pipeline.set(`${CapabilityRegistry.REDIS_PREFIX}${type}:${name}`, JSON.stringify(manifest));
     }
     await pipeline.exec();
     ```
  7. Log: `logger.info('CapabilityRegistry: synced', { total: this.cache.size })`

**IMPORTANT for syncTools:** Skip tools whose name starts with `mcp__` — these are MCP-provided tools and will be captured under the MCP capability entry via `provides_tools`. This avoids double-counting.

`get(id: string): CapabilityManifest | undefined` — Lookup from in-memory cache.

`list(filter?: { type?: CapabilityType; status?: string }): CapabilityManifest[]` — Return all manifests from cache, optionally filtered by type and/or status. (per REG-01: user can view all capabilities in a single registry)

`search(query: { text?: string; tags?: string[]; type?: CapabilityType }): CapabilityManifest[]` — In-memory search (per user decision: in-memory filter, <200 entries). Logic:
  1. Start with all cached manifests
  2. If `query.type` provided, filter by type
  3. If `query.tags` provided, filter to entries where `semantic_tags` includes ANY of the query tags (OR match)
  4. If `query.text` provided, filter where name, description, or any semantic_tag includes the text (case-insensitive substring match)
  5. Return results sorted by name

`async start(): Promise<void>` — Subscribe to `nexus:config:updated` via Redis pub/sub. On message `mcp_config`, call `this.syncAll()`. Then do initial sync. (Per user decision: startup sync + event-driven)

`async stop(): Promise<void>` — Unsubscribe and clean up subscriber connection.

`get size(): number` — Return `this.cache.size`.
