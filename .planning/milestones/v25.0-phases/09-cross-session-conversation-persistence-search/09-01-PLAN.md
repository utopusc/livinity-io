1. **Create conversation_turns table + FTS5 virtual table:**
```sql
CREATE TABLE IF NOT EXISTS conversation_turns (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT NOT NULL,
  channel TEXT NOT NULL DEFAULT 'web',
  chat_id TEXT NOT NULL DEFAULT '',
  role TEXT NOT NULL CHECK(role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  metadata TEXT,
  created_at INTEGER NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_ct_user ON conversation_turns(user_id);
CREATE INDEX IF NOT EXISTS idx_ct_channel ON conversation_turns(channel);
CREATE INDEX IF NOT EXISTS idx_ct_created ON conversation_turns(created_at);

CREATE VIRTUAL TABLE IF NOT EXISTS conversation_turns_fts USING fts5(
  content,
  content='conversation_turns',
  content_rowid='id'
);

-- Triggers to keep FTS5 in sync
CREATE TRIGGER IF NOT EXISTS ct_ai AFTER INSERT ON conversation_turns BEGIN
  INSERT INTO conversation_turns_fts(rowid, content) VALUES (new.id, new.content);
END;
CREATE TRIGGER IF NOT EXISTS ct_ad AFTER DELETE ON conversation_turns BEGIN
  INSERT INTO conversation_turns_fts(conversation_turns_fts, rowid, content) VALUES('delete', old.id, old.content);
END;
```

2. **Add POST /archive endpoint** (after the existing `/add` endpoint, before `/search`):
- Accepts JSON body: `{ userId: string, channel: string, chatId: string, role: 'user' | 'assistant', content: string, metadata?: object }`
- Validates `userId` and `content` are present
- Inserts into `conversation_turns` table with `created_at: Date.now()`
- Returns `{ success: true, id: <lastInsertRowid> }`

- Accepts JSON body: `{ query: string, userId?: string, channel?: string, limit?: number, since?: number }`
- Uses FTS5 MATCH for the query: `SELECT ct.* FROM conversation_turns ct JOIN conversation_turns_fts fts ON ct.id = fts.rowid WHERE conversation_turns_fts MATCH ? ORDER BY rank LIMIT ?`
- Apply optional filters: if `userId` provided, add `AND ct.user_id = ?`; if `channel` provided, add `AND ct.channel = ?`; if `since` provided (unix ms), add `AND ct.created_at > ?`
- Default limit: 20
- Sanitize the FTS5 query: replace special chars that break FTS5 (`"`, `*`, `(`, `)`, `OR`, `AND`, `NOT`, `NEAR`) — wrap each word in double quotes for safety: `query.split(/\s+/).filter(Boolean).map(w => '"' + w.replace(/"/g, '') + '"').join(' ')`
- Return `{ results: Array<{ id, userId, channel, chatId, role, content, metadata, createdAt }> }`
- Parse `metadata` from JSON string if present
