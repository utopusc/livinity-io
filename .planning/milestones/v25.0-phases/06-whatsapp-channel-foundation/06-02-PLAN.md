**Key methods to implement:**

1. `setDmPairing(manager)` and `setApprovalManager(manager)` — same as TelegramProvider

2. `init(redis)` — Store redis, create WhatsAppAuthStore, call loadConfig()

3. `loadConfig()` — Read from `nexus:whatsapp:config` Redis key (same pattern as Telegram: `getRedisPrefix(this.id) + ':config'`)

4. `connect()` — Core Baileys connection logic:
   - If not enabled, set status disconnected and return early (same as Telegram)
   - Load auth state from WhatsAppAuthStore: `const { state, saveCreds } = await this.authStore!.loadState()`
   - Create socket:
     ```typescript
     this.sock = makeWASocket({
       auth: state,
       logger: baileysLogger as any,
       browser: ['LivOS', 'Desktop', '1.0.0'],
       markOnlineOnConnect: false,
       getMessage: async (key) => undefined,  // Minimal implementation for Phase 6
     });
     ```
   - Bind `creds.update` to saveCreds
   - Bind `connection.update` to handleConnectionUpdate
   - Bind `messages.upsert` to handleMessages

5. `handleConnectionUpdate(update)` — Connection lifecycle:
   - If `update.qr` present: Generate QR data URL via `import QRCode from 'qrcode'` and `QRCode.toDataURL(update.qr)`, store in Redis at `nexus:whatsapp:qr` with 60s TTL. Log QR generated.
   - If `update.connection === 'close'`: Extract statusCode from `(update.lastDisconnect?.error as Boom)?.output?.statusCode`. If statusCode !== `DisconnectReason.loggedOut` (408), schedule reconnect with 3-second delay by calling connect() again. If logged out, clear auth state via `authStore.clearAll()` and set status to disconnected.
   - If `update.connection === 'open'`: Set status connected, delete QR from Redis (`nexus:whatsapp:qr`), save status to Redis. Log connection success.

   **CRITICAL**: After QR scan, WhatsApp deliberately closes the connection once to provide auth credentials. This is NOT an error. The reconnect logic must handle this gracefully.

6. `handleMessages(upsert)` — Message reception with echo-loop guard:
   ```typescript
   private async handleMessages(upsert: { messages: any[]; type: string }) {
     if (upsert.type !== 'notify') return;  // Skip history sync messages
     
     for (const msg of upsert.messages) {
       if (!msg.message || msg.key.fromMe) continue;  // ECHO LOOP GUARD: skip own messages
       
       const text = msg.message.conversation
         || msg.message.extendedTextMessage?.text
         || '';
       if (!text.trim()) continue;
       
       // Deduplication via Redis
       if (this.redis) {
         const dedupKey = `nexus:whatsapp:dedup:${msg.key.id}`;
         const isNew = await this.redis.set(dedupKey, '1', 'EX', 86400, 'NX');
         if (!isNew) continue;
       }
       
       const jid = msg.key.remoteJid!;
       const isGroup = jid.endsWith('@g.us');
       const userId = msg.key.participant || jid;
       const userName = msg.pushName || 'Unknown';
       
       // DM Pairing check (non-group only)
       if (!isGroup && this.dmPairing) {
         const result = await this.dmPairing.checkAndInitiatePairing('whatsapp', userId, userName, jid);
         if (!result.allowed) {
           if (result.message) {
             await this.sock?.sendMessage(jid, { text: result.message }).catch(() => {});
           }
           continue;
         }
       }
       
       const incomingMsg: IncomingMessage = {
         channel: 'whatsapp',
         chatId: jid,
         userId,
         userName,
         text,
         timestamp: (msg.messageTimestamp as number) * 1000,
         isGroup,
       };
       
       if (this.messageHandler) {
         try {
           await this.messageHandler(incomingMsg);
         } catch (err: any) {
           logger.error('WhatsAppProvider: message handler error', { error: err.message });
         }
       }
     }
   }
   ```

7. `disconnect()` — Close socket, clear reconnect timer, save status

8. `getStatus()` — Return current status object (same as Telegram)

9. `sendMessage(chatId, text, replyTo?)` — Use `this.sock?.sendMessage(chatId, { text })` with chunkText for long messages. Return true on success, false on failure. Log errors.

10. `onMessage(handler)` — Store handler reference (same as Telegram)

11. `updateConfig(config)` — Merge config, save to Redis, reconnect if needed (same pattern as Telegram)

12. `testConnection()` — Return current connection state (connected = ok, not connected = error)

13. `saveStatus()` — Save status JSON to Redis at `nexus:whatsapp:status` (same pattern as Telegram)
