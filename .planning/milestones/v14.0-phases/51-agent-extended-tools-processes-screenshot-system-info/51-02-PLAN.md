  <action>
1. Run `cd agent && npm install node-screenshots` to add the native addon dependency.

2. Create `agent/src/tools/screenshot.ts`:
   - Use a lazy-load pattern for the native addon to handle graceful failure:
     ```typescript
     let Monitor: any = null;
     let loadError: string | null = null;
     try {
       const mod = await import('node-screenshots');
       Monitor = mod.Monitor;
     } catch (err: unknown) {
       loadError = err instanceof Error ? err.message : String(err);
     }
     ```
     Place this at module top level so the import attempt happens once on first load.

   - Export async function `executeScreenshot(params: Record<string, unknown>)` returning the tool result type including images.
   - Result type: `Promise<{ success: boolean; output: string; error?: string; data?: unknown; images?: Array<{ base64: string; mimeType: string }> }>`.
   - Parameters: `display` (optional number, default: primary display).
   - Implementation:
     a. If `loadError` is set, return `{ success: false, output: '', error: 'Screenshot capture unavailable: ' + loadError }`.
     b. Get all monitors: `const monitors = Monitor.all()`.
     c. If no monitors found, return error.
     d. Select monitor: if `display` param provided and valid index, use that; otherwise find primary (`isPrimary`) or fall back to first monitor.
     e. Capture: `const image = await monitor.captureImage()`.
     f. Encode JPEG: `const jpegBuffer = await image.toJpeg()` (node-screenshots has built-in JPEG encoding).
     g. Convert to base64: `const base64 = jpegBuffer.toString('base64')`.
     h. Return:
        ```
        {
          success: true,
          output: `Screenshot captured: ${image.width}x${image.height} (${jpegBuffer.length} bytes JPEG)`,
          data: { width: image.width, height: image.height, size: jpegBuffer.length },
          images: [{ base64, mimeType: 'image/jpeg' }]
        }
        ```
   - Wrap all capture logic in try/catch; on error return `{ success: false, output: '', error: 'Screenshot failed: ' + err.message }`.
   - IMPORTANT: Since this is a native addon and esbuild cannot bundle it, the import MUST use a dynamic import to avoid build-time resolution failures. Use the lazy-load pattern above.
  </action>
