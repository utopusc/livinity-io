**use-stacks.ts** -- Create new hook file following use-networks.ts pattern:

```typescript
import {useState} from 'react'
import {trpcReact} from '@/trpc/trpc'
```

Hook `useStacks()` should provide:

1. **`stacksQuery`** -- `trpcReact.docker.listStacks.useQuery(undefined, { retry: false, refetchInterval: 15000 })`. Expose as `stacks`, `isLoading`, `isError`, `error`, `refetch`.

2. **`deployStackMutation`** -- `trpcReact.docker.deployStack.useMutation()`. onSuccess: set actionResult success, refetch stacks. onError: set actionResult error. Clear actionResult after 3s/5s.

3. **`editStackMutation`** -- `trpcReact.docker.editStack.useMutation()`. Same success/error pattern.

4. **`controlStackMutation`** -- `trpcReact.docker.controlStack.useMutation()`. Same pattern.

5. **`removeStackMutation`** -- `trpcReact.docker.removeStack.useMutation()`. Same pattern.

6. Wrapper functions: `deployStack(input)`, `editStack(input)`, `controlStack(name, operation)`, `removeStack(name, removeVolumes)`.

7. `actionResult` state for toast messages.

8. Return all values: `stacks`, `isLoading`, `isError`, `error`, `refetch`, `deployStack`, `isDeploying`, `editStack`, `isEditing`, `controlStack`, `isControlling`, `removeStack`, `isRemoving`, `actionResult`.

**index.tsx** -- Add StacksTab and integrate into tabs:

1. **Add import** at top: `import {useStacks} from '@/hooks/use-stacks'` and add `IconStack2` (or `IconLayersLinked`) to the tabler icons import.

2. **Add StacksTab function component** (place it after NetworksTab, before PM2Tab definition, around line ~2250):

The StacksTab component:
- Call `useStacks()` hook.
- State: `expandedStack` (string | null), `deployFormOpen` (boolean), `editTarget` (string | null), `removeTarget` (string | null), `removeVolumes` (boolean).
- **Summary row**: `{stacks.length} stack(s)` count, "Deploy Stack" button (IconPlus).
- **Action result toast** (same as NetworksTab pattern): Show success/error banner if actionResult is set.
- **Stack table**: Table with columns: Name, Status, Containers, Actions.
  - Status column: badge with color -- green for 'running', red for 'stopped', yellow for 'partial'.
  - Actions column: Start button (when stopped/partial), Stop button (when running/partial), Restart button, Edit button (IconPencil), Remove button (IconTrash).
  - Row click toggles `expandedStack`.
  - Expanded row: show constituent containers as a sub-table with Name, Image, State columns (use Fragment pattern like image history).
- **Empty state**: "No stacks found. Deploy your first stack." when stacks array is empty.

3. **DeployStackForm** -- A full-page overlay (same pattern as ContainerCreateForm -- `AnimatePresence` + `motion.div` with fixed positioning). Contains:
  - Stack name input (text input, required, regex validated).
  - YAML editor: a `<textarea>` with monospace font (`font-family: 'JetBrains Mono', 'Fira Code', monospace; font-size: 13px`), dark background (`bg-neutral-900`), white text, min-height 400px, full width. Placeholder showing a sample docker-compose.yml snippet.
  - Environment Variables section: array of key-value rows with add/remove buttons (same pattern as container create form env vars).
  - "Deploy" button (or "Update" when editing).
  - "Cancel" button to close.
  - When `editTarget` is set: query `getStackCompose` and `getStackEnv` to prefill the form. Show "Edit Stack: {name}" as title. Name field is read-only in edit mode. Button says "Redeploy".

4. **RemoveStackDialog** -- Dialog with:
  - Title: "Remove Stack: {name}"
  - Description: "This will stop and remove all containers in this stack."
  - Checkbox: "Also remove associated volumes" (bound to removeVolumes state).
  - Cancel and Remove buttons.
  - On confirm: call `removeStack(name, removeVolumes)`, close dialog.

5. **Add Stacks tab trigger and content** in the tabs section:
  - Add `<TabsTrigger value='stacks'>Stacks</TabsTrigger>` between Networks and PM2 triggers (after line ~2592 `networks` trigger, before line ~2593 `pm2` trigger).
  - Add `<TabsContent value='stacks' className='flex-1 overflow-auto'><StacksTab /></TabsContent>` between networks and pm2 content sections.

6. **Add IconStack2 import** (or use `IconLayersLinked` if IconStack2 unavailable) to the tabler icons import at the top. If neither exists, use `IconBox` which is already imported.

For the YAML textarea, use simple monospace styling -- per Claude's Discretion in CONTEXT.md, a simple textarea with monospace font is fine (no syntax highlighting needed).
