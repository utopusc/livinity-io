**Part A: Create container-create-form.tsx**

Create a new file `livos/packages/ui/src/routes/server-control/container-create-form.tsx` that exports a `ContainerCreateForm` component.

Props:
```typescript
interface ContainerCreateFormProps {
  open: boolean
  onClose: () => void
  onSuccess: () => void  // called after successful creation to refetch container list
}
```

The component renders a full-page overlay (fixed position, z-50, bg-white, full width/height of the parent window) when `open` is true. NOT a Dialog -- a full-page overlay within the server-control window.

**Layout structure:**
- Header bar: "Create Container" title on left, "X" close button on right
- Tabbed form using shadcn Tabs with 6 tabs: General, Network, Volumes, Environment, Resources, Health Check
- Footer bar: "Create" button (primary, shows loading spinner during mutation) and "Cancel" button

**State management:**
Use React useState for form state. Initialize a form state object:

**Submit handler:**
On "Create" button click, build the mutation input from form state:
- Validate: name and image required (show inline error if empty)
- Split command string by spaces into string[] (filter empty)
- Split entrypoint string by spaces into string[] (filter empty)
- Convert ports array: parse hostPort and containerPort as numbers
- Convert volumes array: set hostPath or volumeName based on type
- Filter env/labels to remove empty-key entries
- Convert memoryLimitMB to bytes (parseFloat * 1024 * 1024), cpuLimit to number
- Wrap healthCheckTest in ["CMD-SHELL", value] if non-empty
- Convert interval/timeout/startPeriod from seconds to nanoseconds (parseFloat * 1e9)
- Split dns and extraHosts by comma, trim, filter empty
- Only include optional fields if they have values (don't send empty arrays or zero values)

Call `createMutation.mutate(input)`. Show `createMutation.error?.message` as an error banner above the footer if mutation fails.

- Dynamic row add buttons: `text-sm text-brand hover:text-brand/80` with a plus icon
