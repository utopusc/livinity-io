From livos/.../tunnel-client.ts (ReconnectionManager pattern to replicate):
```typescript
class ReconnectionManager {
  private attempt = 0
  private readonly baseDelay = 1000
  private readonly maxDelay = 60_000
  private readonly maxJitter = 1000

  getNextDelay(): number {
    const exponentialDelay = Math.min(this.baseDelay * 2 ** this.attempt, this.maxDelay)
    const jitter = Math.random() * Math.min(this.maxJitter, exponentialDelay)
    this.attempt++
    return exponentialDelay + jitter
  }

  reset(): void { this.attempt = 0 }
  get consecutiveFailures(): number { return this.attempt }
}
```

Relay onDeviceConnect auth sequence (what the agent must match):
1. Agent connects to wss://relay.livinity.io/device/connect
2. Agent sends DeviceAuth message as FIRST message within 5s
3. Relay verifies JWT, checks deviceId matches token claim
4. On success: relay sends DeviceConnected { type: 'device_connected', sessionId }
5. On failure: relay sends DeviceAuthError { type: 'device_auth_error', error } then closes
6. After auth: relay sends DevicePing every 30s, agent must respond with DevicePong

**agent/src/connection-manager.ts:**

Create `ReconnectionManager` class (replicate the pattern from tunnel-client.ts exactly):
```typescript
class ReconnectionManager {
  private attempt = 0;
  private readonly baseDelay: number;
  private readonly maxDelay: number;
  private readonly maxJitter: number;

  constructor() {
    this.baseDelay = RECONNECT_BASE_DELAY;  // 1000
    this.maxDelay = RECONNECT_MAX_DELAY;    // 60_000
    this.maxJitter = RECONNECT_MAX_JITTER;  // 1000
  }

  getNextDelay(): number {
    const exponentialDelay = Math.min(this.baseDelay * 2 ** this.attempt, this.maxDelay);
    const jitter = Math.random() * Math.min(this.maxJitter, exponentialDelay);
    this.attempt++;
    return exponentialDelay + jitter;
  }

  reset(): void { this.attempt = 0; }
  get consecutiveFailures(): number { return this.attempt; }
}
```

Create `ConnectionManager` class:
- Constructor takes `{ credentials: CredentialsData, onToolCall?: (call: DeviceToolCall) => DeviceToolResult }` where CredentialsData is the type from state.ts credentials
- Private fields: `ws: WebSocket | null`, `sessionId: string | null`, `reconnectTimer`, `reconnection: ReconnectionManager`, `status: 'idle' | 'connecting' | 'connected' | 'disconnected' | 'error'`, `destroyed: boolean`
- `connect()` method:
  1. Set status to 'connecting'
  2. Create `new WebSocket(credentials.relayUrl + '/device/connect')`
  3. On 'open': send DeviceAuth message with fields from credentials + `process.platform` + `AGENT_VERSION` + `TOOL_NAMES` array (import from tools.ts). The DeviceAuth message must match the interface exactly: `{ type: 'device_auth', deviceToken: credentials.deviceToken, deviceId: credentials.deviceId, deviceName: credentials.deviceName, platform: process.platform as 'win32' | 'darwin' | 'linux', agentVersion: AGENT_VERSION, tools: [...TOOL_NAMES] }`
  4. On 'message': parse JSON, route by `msg.type`:
     - `'device_connected'`: set status='connected', reset reconnection, store sessionId, call `writeState({ status: 'connected', connectedAt: new Date().toISOString(), relayUrl: credentials.relayUrl, deviceName: credentials.deviceName })`, log "Connected to relay (session: {sessionId})"
     - `'device_auth_error'`: set status='error', log error, close ws (do NOT reconnect on auth errors - same as TunnelClient)
     - `'device_ping'`: respond immediately with `{ type: 'device_pong', ts: msg.ts }`
     - `'device_tool_call'`: call `executeToolStub(msg.tool, msg.params)` and send back `{ type: 'device_tool_result', requestId: msg.requestId, result }`. (This will be upgraded in Phase 50-51)
  5. On 'close': if status was 'connected' or 'connecting' (and not 'error'), set status='disconnected', update state file, call scheduleReconnect()
  6. On 'error': log error (close handler fires next and triggers reconnect)
- `scheduleReconnect()` private method: if status is 'error' or destroyed, do NOT reconnect. Otherwise compute delay via `reconnection.getNextDelay()`, log "Reconnecting in {delay}ms (attempt {N})", set timer to call `connect()`
- `disconnect()` method: set destroyed=true, clear reconnect timer, close ws if open, remove all listeners, call `writeState({ status: 'disconnected', ... })`, call `removePid()`
- `getStatus()` method: returns `{ status, sessionId }`

**agent/src/cli.ts:**

Export async functions for each CLI command:

`setupCommand()` — Placeholder that prints "Run `livinity-agent setup` to authenticate. (Implemented in Plan 02)". This is a stub that Plan 02 will replace.

`startCommand()`:
1. Read credentials via `readCredentials()`. If null, print "No credentials found. Run `livinity-agent setup` first." and exit with code 1
2. Check if already running: `readPid()` — if PID exists and process is alive (use `process.kill(pid, 0)` in try/catch), print "Agent already running (PID {pid})" and exit
3. Write PID file: `writePid(process.pid)`
4. Create `ConnectionManager` with credentials
5. Call `manager.connect()`
6. Register signal handlers: on SIGINT and SIGTERM, call `manager.disconnect()` then `process.exit(0)`
7. Log "Agent started (PID {process.pid}). Press Ctrl+C to stop."

`stopCommand()`:
1. Read PID via `readPid()`. If null, print "Agent is not running." and exit
2. Try `process.kill(pid, 'SIGTERM')`. On success: print "Sent stop signal to agent (PID {pid})". On error (ESRCH): print "Agent process not found (stale PID file), cleaning up", call `removePid()`

`statusCommand()`:
1. Read state via `readState()`. If null, print "Agent has not been configured. Run `livinity-agent setup` first." and exit
2. Read PID via `readPid()`. Check if process alive.
3. Read credentials via `readCredentials()` for device name and relay URL.
4. Print status table:
   ```
   Livinity Agent Status
   ---------------------
   Status:      {state.status} (PID {pid} {alive ? 'running' : 'not running'})
   Device:      {credentials.deviceName}
   Relay:       {credentials.relayUrl}
   Connected:   {state.connectedAt || 'never'}
   ```
