<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser UI → tRPC apiKeys.create | Phase 59 backend is authoritative; UI just plumbs name → server-generated plaintext |
| Plaintext memory in component state | Once dismissed, must be cleared from React state to prevent post-close DevTools inspection |
| navigator.clipboard.writeText | Browser-level secret transit; same-origin only; no DOM serialization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-13 | I (Information disclosure) | plaintext leaks via console.log / telemetry | mitigate | Grep guard in done criteria; no console.log of plaintext anywhere; sonner toast carries only "copied" status, NOT the plaintext |
| T-62-14 | I | plaintext persists in React state after Close | mitigate | Explicit `setPlaintext(null)` on Close + cleanup useEffect on unmount |
| T-62-15 | T (Tampering) | older UI bundle calls older apiKeys.create that returns no plaintext | accept | Phase 59 contract is locked; if Phase 59 evolves, the create modal's optional-chaining handles missing fields gracefully |
| T-62-16 | R (Repudiation) | revoke confirmation accidentally clicked | mitigate | Two-step: open modal → click destructive button explicitly. Cannot revoke from one click. |
| T-62-17 | I | accessibility leak: screen reader announces plaintext after copy | accept | Visible plaintext is the user's intent (they need to read it); aria-live region not added; user closes modal when done |
</threat_model>
