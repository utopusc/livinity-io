ROADMAP §Phase 59 success criteria (verbatim, lines 116-120 of .planning/ROADMAP.md):
1. User creates a new API key → response shows the plaintext `liv_sk_<base62-32>` value exactly once with a copy-to-clipboard affordance; reloading the list shows only the prefix preview.
2. User invokes `curl -H "Authorization: Bearer liv_sk_..."` → request reaches their per-user broker context (their content, not another user's).
3. User revokes a key → immediately retries the same Bearer → HTTP 401 with body `{"error": {"type": "authentication_error", "message": "API key revoked"}}`.
4. User can audit `last_used_at` to see exactly when the key was last presented.
