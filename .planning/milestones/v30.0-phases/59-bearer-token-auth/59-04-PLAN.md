## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tRPC client → privateProcedure | `ctx.currentUser` enforced by `is-authenticated.ts` middleware; routes additionally throw `UNAUTHORIZED` defensively |
| tRPC client → adminProcedure (`apiKeys.listAll`) | `requireRole('admin')` middleware throws `FORBIDDEN` for non-admins |
| Audit log REUSE → `device_audit_log` | Append-only enforced by Phase 15 BEFORE UPDATE OR DELETE trigger; routes cannot mutate history |
| Cache invalidation cross-module | `apiKeys.revoke` is the only consumer of `cache.invalidate`; no external surface |

| T-59-16 | T (Tampering) | apiKeys.revoke user-scope | mitigate | Both the SELECT (pre-check) and the UPDATE filter on `user_id = ctx.currentUser.id` — a non-admin user cannot revoke another user's key. Test 3 in routes.test.ts asserts this. |
| T-59-17 | E (Elevation) | apiKeys.listAll | mitigate | `adminProcedure` enforces role hierarchy via `requireRole('admin')`; non-admin tRPC call gets `FORBIDDEN`. Test 6 (source-string assert) is defense-in-depth. |
