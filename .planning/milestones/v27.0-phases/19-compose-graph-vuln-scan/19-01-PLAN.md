    Create `livos/packages/ui/src/routes/server-control/compose-graph-viewer.tsx` (single self-contained module, no new shared utils):

    1. Imports: React (useMemo), `yaml` from 'js-yaml' (use named `load`), `ReactFlow`, `Background`, `Controls`, `MarkerType`, `Position`, type `Node`, type `Edge` from 'reactflow', `'reactflow/dist/style.css'` (side-effect import — required for sane default styling), `trpcReact` from '@/trpc/trpc', icons from `@tabler/icons-react`, `cn` helper from '@/shadcn-components/lib/utils'.

    2. Define a private parser `parseCompose(yamlSrc: string): {nodes: Node[]; edges: Edge[]; networks: string[]} | {error: string}`:
       - `yaml.load(yamlSrc)` → typed as `any` then narrowed: must be a record with a `services` object map; otherwise return `{error: 'No services found in compose file'}`. Wrap in try/catch — YAMLException → `{error: 'Invalid YAML: ' + e.message}`.
       - For each service entry `[name, svc]`:
         - Extract `image: string` (default '<no image>'), `ports: string[] | object[]` normalized to short-form display strings ("8080:80/tcp"). Handle both short form `"8080:80"` and long form `{published, target, protocol}`.
         - Extract `dependsOn: string[]` from `svc.depends_on` — can be `string[]` or `Record<string, {condition?: string}>` (object form, take keys).
         - Extract `networks: string[]` from `svc.networks` — can be `string[]` or `Record<string, {}>` (object form, take keys); default `['default']` per compose spec.
         - Build a Node: `{id: name, position: {x, y}, data: {label: name, image, ports, networks}, type: 'default', sourcePosition: Position.Right, targetPosition: Position.Left}`. Lay out as a simple grid: `x = column * 260`, `y = row * 140` where column/row are computed by topologically sorting services so dependees appear left of dependers (use Kahn's algorithm; if cyclic, fall back to insertion order).
         - data.label should be a JSX-friendly string OR build a custom node via the `data` payload — keep it simple by passing a multi-line string with newlines and styling via the Node `style` prop (white bg in light theme, surface-1 in dark via tailwind class on container). For minimum viable: render the rich content via a tiny `customNode` registered in `nodeTypes` prop that pulls from `data` and shows: bold service name, monospace image, then port pill row.
       - For each service `name` with `dependsOn`, build edges: `dependsOn.forEach(dep => edges.push({id: ${dep}->${name}, source: dep, target: name, markerEnd: {type: MarkerType.ArrowClosed}, animated: false, label: 'depends_on'}))`. Skip edges whose source isn't a known service.
       - Top-level `networks` keys (when present) collected into the returned `networks` array — used by the legend below the canvas.
       - Return `{nodes, edges, networks: collectedNetworkNames}`.

    3. Component `export function ComposeGraphViewer({stackName}: {stackName: string})`:
       - `const {data, isLoading, error} = trpcReact.docker.getStackCompose.useQuery({name: stackName}, {enabled: !!stackName, retry: false})`.
       - `const parsed = useMemo(() => data?.yaml ? parseCompose(data.yaml) : null, [data?.yaml])`.
       - Render states:
         - Loading: spinner placeholder ("Loading compose…", IconRefresh animate-spin), 320px tall.
         - tRPC error: red banner with `error.message`.
         - parse error: amber banner "Failed to parse compose: {parsed.error}" with the raw YAML in a collapsible `<details>` for debugging.
         - Success: 480px-tall container `className='relative h-[480px] w-full rounded-lg border border-border-default bg-surface-base'` containing `<ReactFlow nodes={parsed.nodes} edges={parsed.edges} nodeTypes={nodeTypes} fitView proOptions={{hideAttribution: true}}><Background /><Controls /></ReactFlow>` plus a legend strip below listing detected networks (`Networks: frontend, backend, default`).

    4. Define a `customNode` (a React.FC with the `NodeProps` type from reactflow) above the parser. It renders:
       ```tsx
       <div className='rounded-md border border-border-default bg-surface-1 px-3 py-2 shadow-sm min-w-[180px]'>
         <div className='font-semibold text-sm text-text-primary'>{data.label}</div>
         <div className='font-mono text-[11px] text-text-secondary truncate'>{data.image}</div>
         {data.ports?.length > 0 && (
           <div className='mt-1 flex flex-wrap gap-1'>
             {data.ports.map(p => <span key={p} className='inline-flex rounded bg-blue-500/15 px-1.5 py-0.5 text-[10px] font-mono text-blue-700'>{p}</span>)}
           </div>
         )}
       </div>
       ```
       Wrap inside `<Handle type='target' position={Position.Left} /> ... <Handle type='source' position={Position.Right} />` from reactflow so edges have anchor points.
       Define `const nodeTypes = {compose: customNode}` at module scope (outside component) so React Flow doesn't recreate the registry on every render — this is a documented reactflow gotcha. In `parseCompose`, set every node's `type: 'compose'`.

    5. Do NOT export anything except `ComposeGraphViewer`. Keep the file < 300 lines.

    Reference existing patterns:
    - tRPC client usage: see `livos/packages/ui/src/routes/server-control/index.tsx` line 2365 (`trpcReact.docker.getStackCompose.useQuery`).
    - Tailwind tokens: `bg-surface-base`, `border-border-default`, `text-text-primary` etc — already used throughout this file.
    - js-yaml import: `import yaml from 'js-yaml'` (default export, has `.load()` method) — same shape used in livinityd.

    Constraints:
    - NO Monaco / NO graph layout libs (dagre, elmjs) — Kahn's topo sort + grid is sufficient for typical home-server stacks (≤ 10 services). Future improvement deferred.
    - NO calls to docker.getStackCompose outside the component (so the query lifecycle ties to mount/unmount of the Graph tab).
    - All compose-spec parsing fallbacks (defaults to 'default' network when networks key missing, etc.) MUST match docker compose's actual behaviour — test with the canonical Nextcloud + Redis + MariaDB compose snippet documented in 19-CONTEXT.md AID-03.
