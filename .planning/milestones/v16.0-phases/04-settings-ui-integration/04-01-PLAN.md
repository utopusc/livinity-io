    **Claude Auth Routes (5 routes):**

    1. `getClaudeStatus` — privateProcedure.query. Fetch GET `${nexusUrl}/api/claude/status` with X-API-Key header. Return type: `{ authenticated: boolean; method?: string; provider: string }`. On error, return `{ authenticated: false, provider: 'claude' }`.

    2. `setClaudeApiKey` — privateProcedure.input(z.object({ apiKey: z.string().min(1) })).mutation. Fetch POST `${nexusUrl}/api/claude/set-api-key` with JSON body `{ apiKey: input.apiKey }` and headers Content-Type + X-API-Key. On non-ok response, parse error and throw TRPCError INTERNAL_SERVER_ERROR. Return `{ success: true }`.

    3. `claudeStartLogin` — privateProcedure.mutation. Fetch POST `${nexusUrl}/api/claude/start-login` with Content-Type + X-API-Key headers. Return the JSON response as-is (auth URL data for OAuth PKCE). On error throw TRPCError.

    4. `claudeSubmitCode` — privateProcedure.input(z.object({ code: z.string().min(1) })).mutation. Fetch POST `${nexusUrl}/api/claude/submit-code` with JSON body `{ code: input.code }` + headers. On error throw TRPCError. Return `{ success: true }`.

    5. `claudeLogout` — privateProcedure.mutation. Fetch POST `${nexusUrl}/api/claude/logout` with Content-Type + X-API-Key headers. On error throw TRPCError. Return `{ success: boolean }`.

    **Provider Management Routes (2 routes):**

    6. `getProviders` — privateProcedure.query. Fetch GET `${nexusUrl}/api/providers` with X-API-Key header. Return type: `{ providers: Array<{ name: string; available: boolean }>; primaryProvider: string; fallbackOrder: string[] }`. On error return `{ providers: [], primaryProvider: 'kimi', fallbackOrder: ['kimi'] }`.

    7. `setPrimaryProvider` — privateProcedure.input(z.object({ provider: z.enum(['claude', 'kimi']) })).mutation. Fetch PUT `${nexusUrl}/api/provider/primary` with JSON body `{ provider: input.provider }` + headers. On error throw TRPCError. Return the JSON response.

    All routes must:
    - Use `getNexusApiUrl()` for the base URL (existing helper)
    - Use `getErrorMessage()` for error extraction (existing helper)
    - Log errors via `ctx.livinityd.logger.error()`
    - Follow exact same try/catch/TRPCError pattern as existing Kimi routes
