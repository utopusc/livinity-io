## Executive Summary

v29.3 is **structurally complete locally** — the OAuth fallback is closed, per-user HOME isolation is wired through both AI Chat and the new broker, the broker speaks both Anthropic and OpenAI Chat Completions, marketplace apps with `requiresAiProvider: true` get the right env vars injected, and a dashboard surfaces token usage + rate-limit banners.

But **two requirements have integration-verified breakage** and **one requires operator-driven manifest registration** before they can be exercised:

1. **FR-DASH-03 (rate-limit 429 propagation + UI surface)** is structurally unreachable — the broker collapses upstream errors to HTTP 500 (`router.ts:159`) and drops the `Retry-After` header (`agent-runner-factory.ts:75-76`). Capture middleware's throttled-row insert never fires from live traffic. Phase 44 UI logic is correct given a throttled row, but no live row can be produced.
2. **FR-MARKET-02 (user installs MiroFish + chats end-to-end)** is mechanism-complete but the MiroFish manifest sits as a planning artifact only — it's not registered as a builtin app and the GHCR image isn't published. Operator-gated.

Plus three smaller integration gaps:
3. **OpenAI SSE → usage row gap** (FR-DASH-01/02): the OpenAI SSE adapter doesn't emit a `usage` chunk, so OpenAI-streaming marketplace traffic produces zero `broker_usage` rows. Phase 44 dashboard is incomplete for that traffic class.
4. **Five tRPC routes not on `httpOnlyPaths`**: `claudePerUserStartLogin` (sub) + `usage.getMine` (q) + `usage.getAll` (q) violate the "interactive / dashboard routes ride HTTP" project pattern. Likely UX hang under WS reconnect.
5. **Sacred file SHA drift**: integrity test asserts the Phase 40 SHA but the file has been modified since (presumably during v43.x model-bump commits per recent git history). Test will fail.

**Verification gap** (no per-phase VERIFICATION.md, no UAT executed) is not a blocker on its own — six UAT checklists are authored and ready — but it must be either closed by a deploy + UAT cycle OR explicitly accepted before milestone completion.
