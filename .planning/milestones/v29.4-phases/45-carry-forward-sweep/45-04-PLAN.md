1. **Wire-format compliance:** OpenAI Python SDK (and other strict spec consumers) expect `response.usage` to be populated when streaming. Without the chunk, `response.usage.total_tokens` is `0` or `undefined`.
2. **`broker_usage` rows missing for OpenAI streaming:** The capture middleware at `usage-tracking/parse-usage.ts:111-183` scans the SSE buffer for a chunk containing `usage.prompt_tokens` / `usage.completion_tokens`. Since none is emitted today, no row is ever written for streaming OpenAI traffic — Phase 44's usage dashboard shows zero OpenAI streaming usage forever.

There is a sub-issue surfaced by the pattern map (PATTERNS.md C4 section): even if the broker emits a `usage` chunk, the values would be `0/0` because `agent-runner-factory.ts:103-115` currently hardcodes `totalInputTokens: 0` and `totalOutputTokens: 0` on the AgentResult — there's an inline TODO comment stating "not surfaced by /api/agent/stream — Phase 44 may augment". Phase 45 C4's success criterion #5 demands NON-ZERO tokens, so this plan threads real token counts through the upstream stream into the wire chunk.

1. **`openai-sse-adapter.ts`:** Extend `OpenAIChatCompletionChunk` interface with optional `usage`. Update `makeChunk(delta, finishReason, usage?)` to attach `usage` only on the terminal chunk.
