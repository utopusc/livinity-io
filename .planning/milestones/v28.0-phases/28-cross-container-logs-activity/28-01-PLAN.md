    --- use-multiplexed-logs.ts ---

    Signature:
    ```ts
    export type LogLine = {
      id: string                  // monotonically increasing per-container id (e.g. `${name}#${counter++}`) — keys for React, not de-duplication
      containerName: string
      body: string                // raw text chunk (one Docker stream chunk); may contain multiple newlines
      receivedAt: number          // Date.now() at receipt
    }

    export type ConnectionState = 'idle' | 'connecting' | 'open' | 'closed' | 'error'

    export function useMultiplexedLogs(opts: {
      includedNames: string[]      // container names the user has checked in the sidebar
      envId: string                // selected env id (already resolved by useSelectedEnvironmentId)
      tail?: number                // default 500
    }): {
      lines: LogLine[]              // global aggregation: max 5000 per container * N containers
      states: Record<string, ConnectionState>
    }
    ```

    Implementation rules:
    - One `WebSocket` per `includedNames` entry. URL: `${wsProtocol}//${window.location.hostname}${port}/ws/docker/logs?container=<name>&envId=<envId>&tail=<n>&token=<jwt>` (mirror `container-detail-sheet.tsx` line 471-479 verbatim — include port logic, `localStorage.getItem('jwt')`).
    - Chunk -> lines: a single `ws.onmessage` chunk may contain multiple newlines. Split on `/\r?\n/`; drop trailing empty string from final newline; emit one `LogLine` per non-empty piece. Increment per-container counter for `id`.
    - State buffer: keep one `Map<containerName, LogLine[]>` in a ref. On each chunk, push bounded with `pushBounded(buf, line, MAX_LINES_PER_CONTAINER)` from Task 1. Then call `setLines((prev) => mergeAndCap(prev, newLines))` where `mergeAndCap` flattens all maps into a single array sorted by `receivedAt` ASC, capped at `MAX_LINES_PER_CONTAINER * includedNames.length`.
    - To avoid per-chunk renders dragging the React tree on a busy stream: throttle the React state setter via a 100ms `setTimeout` (`flushTimerRef`). Coalesce all lines received in that window into one setState call. Critical for >20 lines/sec streams.
    - On `includedNames` change: open a WS for any new name, close + drop buffer for any removed name. Do NOT clear the entire buffer (preserve already-fetched history of still-included containers).
    - On `envId` change: close ALL sockets, clear ALL buffers, reopen for current `includedNames`. Document this with an inline comment ("crossing env boundary — no carryover").
    - `states` returned for the LogsSidebar to show a small dot per row (open=green, connecting=yellow, error=red, idle=zinc).
    - Cleanup: useEffect return closes all sockets + clears flush timer + clears buffer ref. A `useRef` for the buffer Map ensures we don't lose state across renders.
    - Cap WS count proactively at 25 (defensive bound). If `includedNames.length > 25`, slice to first 25 and surface a warning via a `truncated: boolean` flag in the return value (UI shows a subtle banner "Showing first 25 of N selected containers" when true).
