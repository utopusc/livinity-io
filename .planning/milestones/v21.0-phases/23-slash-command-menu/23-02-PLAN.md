**Component props:**
```typescript
interface SlashCommandMenuProps {
  filter: string           // text after '/' for filtering
  selectedIndex: number    // currently highlighted item index
  onSelect: (command: SlashCommand) => void  // called on click
  onFilteredCountChange: (count: number) => void  // inform parent of filtered count for index clamping
}
```

**Component behavior:**
1. Fetch dynamic commands via `trpcReact.ai.listSlashCommands.useQuery(undefined, {staleTime: 30_000, refetchInterval: 60_000})`
2. Merge UI_COMMANDS + backend commands, deduplicate by name (UI_COMMANDS take priority)
3. Filter by `filter` using `cmd.name.slice(1).toLowerCase().includes(filter)` -- case-insensitive substring match
4. Call `onFilteredCountChange(filtered.length)` via useEffect when filtered list length changes
5. Auto-scroll selected item into view using `scrollIntoView({block: 'nearest'})` when `selectedIndex` changes
6. Use `onMouseDown` with `e.preventDefault()` on menu items to prevent focus stealing from textarea
7. Return `null` if filtered list is empty

**Styling:**
- Absolutely positioned: `absolute bottom-full left-0 right-12 mb-2`
- Max height with scroll: `max-h-64 overflow-y-auto`
- Dark surface: `rounded-lg border border-border-default bg-surface-base shadow-lg`
- Each item: `flex cursor-pointer items-center gap-3 px-3 py-2 text-sm`
- Selected item: `bg-surface-2 text-text-primary`
- Unselected: `text-text-secondary hover:bg-surface-1`
- Command name in mono: `font-mono text-brand`
- Description truncated: `truncate text-text-tertiary`
- Category badge (optional, use discretion): small uppercase label like `text-[10px] uppercase text-text-quaternary` showing 'tool' or 'skill' for non-builtin items

6. Modify `handleKeyDown` to intercept arrow/enter/escape when slash menu is open. The slash menu handling MUST come BEFORE the existing Enter-to-send logic:
```typescript
const handleKeyDown = (e: React.KeyboardEvent<HTMLTextAreaElement>) => {
  // Slash menu keyboard navigation takes priority
  if (showSlashMenu && filteredCount > 0) {
    if (e.key === 'ArrowUp') {
      e.preventDefault()
      setSelectedIndex((prev) => Math.max(0, prev - 1))
      return
    }
    if (e.key === 'ArrowDown') {
      e.preventDefault()
      setSelectedIndex((prev) => Math.min(filteredCount - 1, prev + 1))
      return
    }
    if (e.key === 'Enter') {
      e.preventDefault()
      // Selection is handled by SlashCommandMenu via the selectedIndex
      // We need to trigger selection -- fire a custom approach:
      // The menu renders items, and we need to tell the menu to select.
      // Instead: store filtered commands ref and select directly here.
      return  // prevent send; the menu's onSelect handles this via a ref approach
    }
    if (e.key === 'Escape') {
      e.preventDefault()
      onChange('')  // clear input to dismiss menu
      return
    }
  }
  // Existing Enter-to-send logic (only fires when slash menu is NOT open)
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault()
    if (value.trim()) {
      onSend()
    }
  }
}
```

**Important design note for Enter key:** The Enter key in the slash menu should trigger command selection. To make this work cleanly, add a `selectedCommandRef` approach:

**Recommended approach (simplest):** Have SlashCommandMenu accept an `onKeySelect` callback and call it when Enter is pressed. But since the keydown is on the textarea, not the menu, the simplest pattern is:
