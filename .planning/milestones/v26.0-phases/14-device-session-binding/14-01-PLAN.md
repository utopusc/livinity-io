<objective>
Bind every DeviceBridge WebSocket at the relay to a specific user session JWT at handshake time, and add a server-side expiry watchdog that closes bridges whose device token has expired. This plan establishes the session-binding half of SESS-01/SESS-02; Plan 14-02 completes SESS-03 by adding revocation pub/sub.

Purpose: Today the relay's /device/connect handshake validates only the device JWT (verifyDeviceToken — userId/deviceId/deviceName/platform claims). There is no tie-back to the specific user session that issued the token, no server-side expiry enforcement beyond jsonwebtoken's initial verify (an already-open connection with an expired token stays open indefinitely), and no defense against a replayed device token whose userId claim does not match the device row's owner. This plan fixes all three: (1) embed the approving user's sessions.id UUID into every new device token as a sessionId claim, (2) validate session existence/expiry AND cross-check tokenPayload.userId against devices.user_id at handshake, (3) add a 60s watchdog interval that closes any DeviceConnection whose tokenExpiresAt has passed.

Output:
- Migration 0008: device_grants.session_id column
- platform/web: approve route records the approving session's UUID; token route embeds it in JWT
- platform/relay: device-auth validates the new claim, handshake queries sessions + devices tables, DeviceConnection stores tokenExpiresAt, watchdog closes expired bridges with code 4401
</objective>
