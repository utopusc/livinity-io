Verifies pnpm install + every package's build produces a non-empty dist/ (Phase 31 BUILD-01 pattern). Bash artifact lint as bonus regression catch. |----|--------|------------|
| T-35-01 | Workflow YAML syntax error blocks ALL future PRs from running this check | Python yaml.safe_load validation in verify step + GitHub's own YAML linting on push |
| T-35-02 | Workflow times out beyond 15 min budget | `timeout-minutes: 15` hard cap; cache strategy keeps median ~5-8 min |
| T-35-03 | Workflow consumes excessive GitHub Actions minutes (cost) | `concurrency` block cancels superseded runs; trigger paths are narrow (only build-affecting paths) |
| T-35-04 | False positive failure (workflow fails on clean code) blocks legit PRs | First clean PR after merge will demonstrate green; if it fails, the workflow itself needs a fix-up commit |
| T-35-05 | False negative pass (workflow misses a real regression) | Mitigated by `verify_build`-style dist non-empty checks (Phase 31 BUILD-01 pattern); livinityd typecheck added as bonus catch |

</threat_model>
